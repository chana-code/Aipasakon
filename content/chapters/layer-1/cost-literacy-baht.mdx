# อ่านราคา AI ให้เป็นบาท — จ่ายตาม "token" ไม่ใช่ต่อคำถาม และทำไมบิลถึงพุ่งได้

คุณน่าจะเคยสะดุดกับตัวเลขนี้. ChatGPT, Claude, Gemini เวอร์ชันเสียเงิน ขึ้นราคามาว่า "$20 ต่อเดือน" — แล้วคุณก็คิดในใจว่าตกลงมันกี่บาท คุ้มไหม จ่ายแล้วได้อะไรเพิ่ม. หรือคุณอาจเคยเห็นคนบ่นในกระทู้ว่าลองต่อ AI ผ่าน "API" แล้วงานแค่ไม่กี่ชิ้น บิลดันเด้งจากหลักสตางค์ไปหลักร้อย ทั้งที่รู้สึกว่าตัวเองไม่ได้ใช้อะไรเยอะ.

และถ้าคุณเคยทำงานเอกสารภาษาไทยยาว ๆ กับ AI คุณอาจสังเกตว่ามันเหมือน "เปลือง" กว่าตอนทำงานภาษาอังกฤษ ทั้งที่เป็นเรื่องเดียวกัน. ตกลงราคาของ AI มันคิดยังไงกันแน่.

คำตอบของทั้งสามเรื่องเป็นเรื่องเดียวกัน. และพอคุณเข้าใจมัน คุณจะอ่านราคา AI ออกเป็นบาท ตัดสินใจได้ว่าจะใช้ฟรีหรือจ่าย และที่สำคัญที่สุด — มองออกล่วงหน้าว่าอะไรกำลังจะทำให้บิลพุ่ง ก่อนที่มันจะพุ่งจริง.

## AI ไม่ได้คิดเงินต่อ "คำถาม" — มันคิดตามปริมาณข้อความ

คนส่วนใหญ่เดาว่า AI คิดเงินแบบเดียวกับค่าโทรศัพท์สมัยก่อน — โทรหนึ่งครั้ง คิดหนึ่งครั้ง. ถามหนึ่งคำถาม คิดหนึ่งหน่วย. แต่มันไม่ใช่แบบนั้นเลย.

จำได้ไหมว่าโมเดลไม่ได้อ่านข้อความเป็นคำ ๆ แบบที่คนอ่าน แต่หั่นข้อความออกเป็นก้อนเล็ก ๆ ที่เรียกว่า **token** ก่อนเสมอ (เราเจาะเรื่องนี้ไปแล้วในบทเรื่องภาษาไทยกับ token). ปรากฏว่า token นี่แหละคือ *หน่วยที่ใช้คิดเงิน*. ทุกครั้งที่คุณคุยกับ AI มีข้อความวิ่งสองทาง: ข้อความที่ *เข้า* ไปให้โมเดลอ่าน (คำถามคุณ + ประวัติแชต + ไฟล์ที่แนบ + คำสั่งลับของระบบ) เรียกว่า input และข้อความที่โมเดล *คาย* ออกมาเป็นคำตอบ เรียกว่า output. ผู้ให้บริการนับ token ทั้งสองฝั่งรวมกัน แล้วคิดเงินตามนั้น.

นี่คือเหตุผลที่คำถามสั้น ๆ หนึ่งบรรทัดกับคำถามที่แนบเอกสาร 20 หน้าเข้าไป "ราคาไม่เท่ากัน" ทั้งที่นับเป็นคำถามเดียวเหมือนกัน. อย่างหลังป้อน token เข้าไปมากกว่าหลายเท่า. และคำตอบยาว ๆ ก็แพงกว่าคำตอบสั้น เพราะ output ก็นับ token เหมือนกัน. คุณไม่ได้จ่ายต่อ "ครั้งที่ถาม" — คุณจ่ายตาม "ปริมาณตัวอักษรที่ไหลผ่าน".

> 💡 **ใจความสำคัญ:** AI คิดเงินตาม token ที่ไหลเข้าและไหลออก ไม่ใช่ต่อจำนวนคำถาม. ยิ่งป้อนข้อความเข้ามาก ยิ่งให้ตอบยาว ราคายิ่งสูง — ถึงจะเป็นแค่ "คำถามเดียว" ก็ตาม.

## เทียบกับค่าน้ำค่าไฟ

วิธีที่ AI คิดเงินเหมือน *ค่าน้ำค่าไฟ* มากกว่าค่าโทรศัพท์เหมาจ่าย. การไฟฟ้าไม่ได้คิดเงินคุณต่อ "จำนวนครั้งที่เปิดสวิตช์" — มันคิดตามหน่วยไฟที่คุณใช้จริง. เปิดหลอดไฟดวงเดียวทิ้งไว้ห้านาที กับเปิดแอร์ทั้งบ้านทั้งคืน คนละราคากันคนละโลก ทั้งที่นับเป็น "การเปิด" เหมือนกัน.

AI ก็เหมือนกัน. token คือหน่วยไฟ. งานที่กินข้อความเข้า-ออกเยอะ = ใช้หน่วยเยอะ = จ่ายเยอะ. ส่วนงานที่ถามนิดเดียวตอบนิดเดียว = หน่วยน้อย = แทบไม่เสียอะไร.

จุดที่การเปรียบเทียบนี้พัง: มิเตอร์ค่าน้ำค่าไฟติดอยู่ที่บ้าน คุณเดินไปดูได้ว่าตอนนี้ใช้ไปกี่หน่วย. แต่มิเตอร์ token ของ AI คุณมองไม่เห็นตอนใช้ — มันนับเงียบ ๆ อยู่เบื้องหลัง. คุณจะรู้ตัวอีกทีตอนเห็นบิล. และนี่แหละคือต้นเหตุของเซอร์ไพรส์ราคาเกือบทั้งหมด — ไม่ใช่ว่าราคาแพงเกินจริง แต่เพราะหน่วยที่มันนับนั้น *มองไม่เห็น* คนเลยใช้เกินตัวโดยไม่รู้ตัว.

## "$20 ต่อเดือน" กับ "จ่ายตาม token" คือคนละมิเตอร์

ตรงนี้คือจุดที่หลายคนงงหนักที่สุด เพราะมันมีสองวิธีจ่ายเงินที่ฟังดูซ้ำกันแต่จริง ๆ เป็นคนละบริการ.

แบบแรกคือ **subscription** — ค่าสมาชิกรายเดือนแบบที่คุณเห็นว่า "$20 ต่อเดือน". นี่คือการจ่ายเหมาเพื่อใช้ *แอป* (เว็บ ChatGPT, แอป Claude) ได้ทั้งเดือน. ข้อดีคือมันมี *เพดาน* — ต่อให้คุณคุยทั้งวันทั้งคืน คุณก็จ่ายเท่าเดิม (มีลิมิตการใช้งานต่อช่วงเวลาอยู่บ้าง แต่ไม่มีบิลเซอร์ไพรส์). เหมาะกับคนทั่วไปที่นั่งพิมพ์คุยกับ AI ผ่านหน้าเว็บเป็นหลัก.

แบบที่สองคือ **API** — การต่อตรงเข้าไปที่ตัวโมเดล เพื่อเอาไปประกอบกับโปรแกรมหรือเครื่องมืออื่น (เราเล่าความต่างระหว่างแอปกับ API ไว้แล้วในบทเรื่องนี้). แบบนี้จ่าย *ตาม token จริง* ที่ใช้ และจุดสำคัญคือ — มัน *ไม่มีเพดาน*. ใช้มากจ่ายมาก ใช้น้อยจ่ายน้อย และถ้ามีอะไรทำให้มันใช้เยอะแบบที่คุณไม่ได้ตั้งใจ บิลก็พุ่งได้จริง ๆ.

คำถามที่คนไทยถามกันบ่อยในกระทู้คือ "ถ้าจ่ายค่าสมาชิกแล้ว ยังต้องจ่ายค่า API อีกเหรอ จ่ายซ้ำหรือเปล่า". คำตอบคือไม่ใช่การจ่ายซ้ำ — มันคือสองมิเตอร์ของสองการใช้งานคนละแบบ. ค่าสมาชิกคือค่าใช้แอป. ค่า API คือค่าต่อโมเดลตรงเข้าโปรแกรมของคุณเอง. คนส่วนใหญ่ใช้แค่แอป ก็จ่ายแค่ค่าสมาชิกพอ ไม่ต้องไปแตะ API เลย.

## ทำไมงานภาษาไทยถึงแพงกว่า

ตรงนี้คือจุดที่เรื่องราคากระทบคนไทยมากกว่าฝรั่งโดยตรง. จำได้ไหมว่าภาษาไทยถูกหั่นเป็น token ละเอียดกว่าภาษาอังกฤษราว 3–5 เท่า สำหรับเนื้อหาที่มีความหมายเท่ากัน (เราอธิบายว่าทำไมไว้แล้วในบทเรื่องภาษาไทยกับ token).

พอ token คือหน่วยที่ใช้คิดเงิน เรื่องนี้ก็แปลตรง ๆ เป็นบิล: งานชิ้นเดียวกันเป๊ะ ๆ ทำเป็นภาษาไทยอาจกินค่าใช้จ่ายราว 3–5 เท่าของภาษาอังกฤษ. ไม่ใช่เพราะ AI คิดแพงกับคนไทย แต่เพราะข้อความไทยเดียวกัน "นับเป็น token ได้มากกว่า" — ทั้งฝั่ง input ที่คุณป้อนเข้า และฝั่ง output ที่มันตอบกลับมาเป็นไทย.

นี่คือเหตุผลที่การเขียนคำสั่งภาษาไทยให้กระชับ ไม่เล่ายืดเยื้อ และไม่แนบเอกสารไทยทั้งกองทั้งที่ใช้แค่บางส่วน — มันไม่ใช่แค่เรื่องความเรียบร้อย แต่ช่วยประหยัดเงินจริง โดยเฉพาะเวลาทำงานเป็นภาษาไทย.

## ตัวร้ายที่ทำบิลพุ่งจริง ๆ: ลูปที่วนซ้ำโดยไม่มีใครเฝ้า

ถึงตรงนี้ราคายังดูจัดการได้ — ใช้เยอะจ่ายเยอะ ก็ฟังดูยุติธรรมดี. แต่มีหนึ่ง pattern ที่ทำให้คนเจอบิลเซอร์ไพรส์หนักที่สุด และมันมาจากสิ่งที่เราเรียนกันมาตั้งแต่ต้นว่า AI ที่ทำงานอัตโนมัติทำงานเป็น *วงจรที่เรียกโมเดลซ้ำ ๆ* (the loop).

ลองนึกภาพ: คุณตั้งให้ AI ทำงานชิ้นหนึ่งแบบอัตโนมัติ. การเรียกโมเดลหนึ่งครั้งอาจถูกแค่หลักสตางค์ — ดูเหมือนไม่มีอะไร. แต่ถ้างานนั้นวนเป็นลูป และในลูปมีอะไรพัง — มันลองทำ ไม่สำเร็จ ลองใหม่ ไม่สำเร็จ ลองใหม่อีก — แต่ละรอบมันป้อนข้อความเดิมทั้งกองเข้าไปใหม่ บวกของที่สะสมมา แล้วเรียกโมเดลอีกครั้ง. การกระทำที่ "ถูกแค่หลักสตางค์" พอวนหลายร้อยรอบโดยไม่มีใครเฝ้า ก็กลายเป็นบิลจริงที่พุ่งจากระดับไม่กี่บาทไปหลักร้อยได้ในเวลาไม่นาน. มีคนในวงการเล่าถึงงานที่ควรจะจ่ายไม่ถึงห้าบาท แต่จบลงด้วยหลักร้อย เพราะลูปที่พังแล้วลองซ้ำเงียบ ๆ ทั้งคืน.

นี่คือจุดที่การเปรียบเทียบกับค่าน้ำกลับมาเจ็บ: ลูปที่พังแล้ววนซ้ำ ก็เหมือนก๊อกน้ำที่เปิดทิ้งไว้ทั้งคืนโดยที่คุณไม่รู้ตัว. ทีละหยดดูไม่มีอะไร แต่พอเช้ามาน้ำท่วม. และเพราะมิเตอร์ token มองไม่เห็น คุณเลยไม่มีทางรู้จนกว่าจะเห็นบิล.

เรื่องนี้แทบไม่กระทบคนที่ใช้แค่แอปแบบ subscription เลย เพราะมีเพดานคุมไว้. มันเป็นความเสี่ยงของฝั่ง API ที่ไม่มีเพดาน — โดยเฉพาะเวลาคุณเริ่มเอา AI ไปต่อกับงานอัตโนมัติ.

## สุขลักษณะเรื่องค่าใช้จ่าย — ทำอะไรได้บ้าง

พอคุณเห็นว่าราคามาจากปริมาณ token ที่ไหลผ่าน คุณก็คุมมันได้ตรงจุด โดยไม่ต้องท่องตารางราคา:

- **อย่าใช้โมเดลตัวใหญ่สุดกับทุกงาน.** โมเดลตัวใหญ่คิดค่าต่อ token แพงกว่า. งานง่าย ๆ อย่างสรุปสั้นหรือร่างข้อความ ใช้ตัวเล็กก็พอและถูกกว่ามาก (เราเจาะวิธีเลือกโมเดลให้เหมาะกับงานไว้ในบทเรื่องการเลือกโมเดล).
- **เขียนให้กระชับ โดยเฉพาะภาษาไทย.** input ที่สั้นลง = token น้อยลง = ถูกลง. ไม่ต้องแนบเอกสารทั้งกองถ้าใช้แค่สองหน้า.
- **ระวังงานที่วนเป็นลูป.** ถ้าคุณเริ่มทำงานอัตโนมัติผ่าน API ให้ตั้งลิมิตการใช้จ่ายไว้เสมอ และอย่าปล่อยลูปทำงานทิ้งไว้โดยไม่มีใครดู.
- **ถ้าใช้แค่พิมพ์คุย ก็อยู่กับ subscription พอ.** มันมีเพดาน คุณนอนหลับสบายไม่ต้องกลัวบิลเด้ง. ไม่จำเป็นต้องแตะ API ถ้างานคุณไม่ได้ต้องการมัน.

### ลองทำดู

เปิดแชตใหม่กับ AI แล้วลองถามคำถามสั้น ๆ หนึ่งบรรทัด เช่น "แนะนำชื่อร้านกาแฟหน่อย". จากนั้นในแชตใหม่อีกอัน วางข้อความยาว ๆ สัก 2–3 หน้าเข้าไป แล้วสั่งว่า "สรุปให้หน่อย". สังเกตด้วยตัวคุณเองว่า งานที่สองมันป้อนข้อความเข้าไป *เยอะกว่ามาก* ทั้งที่คุณรู้สึกว่ามันก็ "ถามคำถามเดียว" เหมือนกัน. นั่นคือภาพของ token ที่ไหลเข้า — สิ่งที่มิเตอร์กำลังนับ. แค่เห็นความต่างของปริมาณข้อความสองอันนี้ คุณก็เริ่มมองออกแล้วว่างานแบบไหนกินเงิน งานแบบไหนแทบไม่เสียอะไร.

(ตัวเลขราคาต่อ token เปลี่ยนบ่อยมากและแต่ละโมเดลไม่เท่ากัน เราเลยไม่ปักตัวเลขไว้ในบทนี้ — สิ่งที่ไม่เปลี่ยนคือ *หน่วย* ที่ใช้คิด คือ token. พอคุณอ่านหน่วยออก ตัวเลขจริงคุณไปดูในหน้าราคาของแต่ละเจ้าได้เอง.)

## กลับมาที่ความสับสนตอนต้น

"$20 คือกี่บาท" — คือค่าเหมารายเดือนเพื่อใช้แอป มีเพดาน คุณคำนวณเป็นบาทตามเรตวันนั้นได้เลย และมันจะไม่เด้งเกินนั้น. "token ละกี่ตังค์" — น้อยมากต่อหนึ่ง token แต่มันคือหน่วยที่ *ทบกัน* ตามปริมาณข้อความเข้า-ออก เลยต้องมองที่ปริมาณรวม ไม่ใช่ราคาต่อหน่วย. "ใช้ ๆ ไปบิลจะพุ่งไหม" — ถ้าคุณใช้แค่แอปแบบ subscription ไม่พุ่งเพราะมีเพดาน. ที่พุ่งได้คือฝั่ง API ที่ไม่มีเพดาน โดยเฉพาะเวลามีลูปวนซ้ำโดยไม่มีคนเฝ้า.

ตอนนี้คุณอ่านราคา AI ออกเป็นบาทแล้ว และมองเห็นมิเตอร์ที่เคยมองไม่เห็น. แต่การรู้ว่ามันราคาเท่าไหร่ เป็นคนละเรื่องกับการรู้ว่า *เมื่อไหร่ที่ไม่ควรเอามันมาทำงานตั้งแต่แรก* — บางงานต่อให้ AI ทำได้และไม่แพง ก็ยังเป็นเครื่องมือที่ผิดสำหรับงานนั้น. บทต่อไปจะปิดท้ายด้วยเส้นที่ซื่อสัตย์ที่สุด: เมื่อไหร่ที่คำตอบที่ดีที่สุดคือ "อย่าใช้ AI กับงานนี้".
