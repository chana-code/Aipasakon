# มี prompt วิเศษไหม — ทำไมพิมพ์เหมือนกัน บางทีดี บางทีพัง

คุณน่าจะเคยเจอสองอย่างนี้. อย่างแรก — คุณเห็นใครสักคนในเน็ตโพสต์ "prompt ลับที่ทำให้ ChatGPT เก่งขึ้น 10 เท่า" คุณก๊อปมาลองแล้วก็... เฉย ๆ ไม่ได้วิเศษอย่างที่โฆษณา. อย่างที่สอง — คุณพิมพ์คำสั่งเดิมเป๊ะ ๆ สองครั้ง ครั้งแรกได้งานที่ใช้ได้เลย ครั้งที่สองได้อะไรไม่รู้ที่ต้องแก้ทั้งยวง.

ตรงนี้มันชวนงง. ถ้ามันเป็นเครื่องจักร พิมพ์เหมือนกันก็ควรได้เหมือนกันสิ. และถ้ามี "คำวิเศษ" จริง ทำไมคำวิเศษของคนอื่นใช้กับงานคุณไม่ขึ้น.

คำตอบของทั้งสองเรื่องเป็นเรื่องเดียวกัน. และพอคุณเข้าใจมัน คุณจะเลิกไล่หา prompt วิเศษ เลิกกดส่งซ้ำหวังดวง แล้วหันมาเขียน prompt แบบที่ *รู้ว่าตัวไหนกำลังขยับผลลัพธ์* — ซึ่งเป็นคนละเรื่องกับการท่องสูตร.

## ทำไม "ช่วยเขียนอีเมลหน่อย" ถึงพังบ่อย

ย้อนกลับไปที่หัวใจก่อน. โมเดลภาษาทำได้อย่างเดียวจริง ๆ คือ *เดาคำถัดไป* จากข้อความที่เห็นตรงหน้า — เราเจาะเรื่องนี้ไปแล้วในบทแรก ๆ ของหลักสูตร. มันไม่ได้ "เข้าใจ" คำสั่งคุณแบบที่คนเข้าใจ มันแค่ดูข้อความที่คุณป้อน แล้วเดาว่าข้อความแบบไหนน่าจะตามมา.

ทีนี้ลองคิดว่าคุณพิมพ์แค่ "ช่วยเขียนอีเมลหน่อย". สำหรับโมเดล ข้อความสั้น ๆ แค่นี้เปิดทางให้คำตอบที่ "น่าจะตามมา" ได้กว้างมหาศาล. อีเมลขอลางาน อีเมลทวงหนี้ อีเมลขายของ อีเมลขอบคุณลูกค้า — ทั้งหมดนี้คือคำตอบที่ "เป็นไปได้" จากคำสั่งกว้าง ๆ ของคุณ. โมเดลก็เลือกหยิบเส้นทางใดเส้นทางหนึ่งมา และแต่ละครั้งมันไม่จำเป็นต้องหยิบเส้นเดิม.

นี่แหละคือสาเหตุที่คำสั่งเดียวกัน บางทีได้ดีบางทีพัง. ไม่ใช่เพราะดวง ไม่ใช่เพราะ AI วันนี้อารมณ์ไม่ดี. แต่เพราะคำสั่งกว้างของคุณเปิดความเป็นไปได้ไว้กว้างเกินไป แล้วโมเดลก็มีอิสระจะเดินไปทางไหนก็ได้ในความกว้างนั้น.

## prompt ที่ดีไม่ได้ "สั่ง" — มัน"จำกัดสิ่งที่น่าจะตามมา"

พอมองแบบนี้ หน้าที่จริงของ prompt ก็เปลี่ยนไป. prompt ที่ดีไม่ได้ทำงานเหมือนคาถาที่ปลุกพลังลับของ AI. มันทำงานด้วยการ *หุบความกว้าง* ของคำตอบที่น่าจะตามมาให้แคบลง จนเหลือแต่เส้นทางที่คุณอยากได้.

ยิ่งคุณใส่รายละเอียดที่บีบความเป็นไปได้ลง — ใครเป็นคนเขียน เขียนถึงใคร น้ำเสียงแบบไหน หน้าตางานที่อยากได้เป็นยังไง — เส้นทางที่ "น่าจะตามมา" ก็ยิ่งแคบลง จนโมเดลแทบไม่มีทางเดินไปทางอื่นได้. นั่นคือเหตุผลที่ prompt ละเอียดให้ผลที่นิ่งและตรงกว่า prompt กว้าง ๆ. ไม่ใช่เพราะถ้อยคำขลังกว่า แต่เพราะมันเหลือทางให้เดาน้อยลง.

> 💡 **ใจความสำคัญ:** ไม่มีคำวิเศษ — โมเดลยังเดาคำถัดไปอยู่เสมอ. prompt ที่ดีคือการ *จำกัดสิ่งที่น่าจะตามมา* ให้แคบลงจนเหลือแต่สิ่งที่คุณต้องการ. ทุกบรรทัดที่คุณเพิ่มเข้าไปคือคันโยกบนความน่าจะเป็น ไม่ใช่คาถา.

## คันโยก 4 ตัวที่ขยับผลลัพธ์ได้จริง

ถ้า prompt คือการบีบความเป็นไปได้ให้แคบลง คำถามต่อไปคือ — บีบยังไงให้ได้ผล. มีคันโยกอยู่ 4 ตัวที่ขยับผลลัพธ์ได้แรงที่สุด. ทั้งสี่ตัวทำงานเรื่องเดียวกันหมด คือบอกโมเดลให้ชัดขึ้นว่า "สิ่งที่น่าจะตามมาควรหน้าตาแบบไหน".

**1. บอกบทบาท (role).** บอกโมเดลว่าให้สวมบทเป็นใคร — เช่น "คุณคือนักการตลาดสายประกันที่เขียนภาษาเป็นกันเองแต่น่าเชื่อถือ". การระบุบทบาทช่วยดึงคำตอบให้เข้าไปอยู่ในโซนของข้อความแบบที่คนในบทบาทนั้นเขียน. ข้อความของ "นักการตลาดประกัน" หน้าตาต่างจากข้อความของ "อาจารย์มหาวิทยาลัย" ชัดเจน และพอคุณระบุบทบาท คุณก็ตัดอีกหลายเส้นทางทิ้งไปในประโยคเดียว.

**2. ให้ดูตัวอย่าง (few-shot).** แทนที่จะอธิบายว่าอยากได้แบบไหน ให้แปะตัวอย่างงานที่ดีไป 1-2 ชิ้นเลย — เช่น "นี่คือแคปชันเก่าที่ลูกค้าชอบ เขียนใหม่ให้โทนเดียวกัน" แล้ววางแคปชันตัวอย่างไว้. โมเดลเก่งเรื่องเลียนแบบรูปแบบจากตัวอย่างที่เห็นมาก. การให้ดูของจริงหนึ่งชิ้น มักได้ผลดีกว่าการพรรณนาเป็นย่อหน้า. ในวงการเรียกเทคนิคให้ดูตัวอย่างแบบนี้ว่า **few-shot** (แปลคร่าว ๆ ว่า "ให้ดูสักสองสามตัวอย่าง").

**3. กำหนดโครงสร้างและรูปแบบ.** บอกไปตรง ๆ ว่าอยากได้คำตอบหน้าตายังไง — "ตอบเป็นข้อ 5 ข้อ ข้อละไม่เกินสองบรรทัด" หรือ "ทำเป็นตาราง 3 คอลัมน์: ปัญหา / สาเหตุ / วิธีแก้". การกำหนดรูปร่างของคำตอบเป็นการบีบความเป็นไปได้ที่ตรงและแรงมาก เพราะมันล็อกทั้งโครงไว้เลย ไม่ใช่แค่เนื้อหา.

**4. สั่งให้คิดเป็นขั้น (chain-of-thought).** สำหรับงานที่ต้องใช้เหตุผลหลายชั้น — เช่นคำนวณ วิเคราะห์ เปรียบเทียบเงื่อนไข — ลองเติมประโยคแบบ "ค่อย ๆ คิดทีละขั้นก่อนสรุป". พอคุณสั่งให้มันเขียนขั้นตอนความคิดออกมา แต่ละขั้นที่มันพิมพ์จะกลายเป็นข้อความที่อยู่ตรงหน้าให้ขั้นต่อไปเดาต่อ ทำให้คำตอบสุดท้ายมีฐานให้ยืนมากกว่าการกระโดดตอบเลย. เทคนิคนี้เรียกว่า **chain-of-thought** (ย่อว่า CoT — แปลว่า "ห่วงโซ่ความคิด").

## ของจริงคือเอาคันโยกมาซ้อนกัน

จุดที่คนเข้าใจผิดบ่อยคือคิดว่าต้องหา prompt เดียวที่สมบูรณ์แบบ. จริง ๆ แล้ว prompt ที่ได้ผลดีมักเป็นการเอาคันโยกหลายตัวมาซ้อนกัน. ลองดูความต่าง.

prompt กว้าง: "ช่วยเขียนอีเมลแจ้งลูกค้าเรื่องเลื่อนนัด"

prompt ที่ซ้อนคันโยก: "คุณคือเจ้าหน้าที่ดูแลลูกค้าของบริษัทประกัน [บทบาท] ช่วยเขียนอีเมลแจ้งลูกค้าว่าต้องเลื่อนนัดตรวจสุขภาพจากวันศุกร์เป็นวันจันทร์ น้ำเสียงสุภาพแต่ไม่เป็นทางการจนแข็ง [บทบาท+โทน] ความยาวไม่เกิน 4 ประโยค ขึ้นต้นด้วยการขอโทษ ลงท้ายด้วยการเสนอช่องทางติดต่อกลับ [โครงสร้าง]"

อันที่สองไม่ได้ "ขลัง" กว่า. มันแค่ปิดเส้นทางที่ไม่ต้องการทิ้งไปทีละเส้น จนสิ่งที่เหลือให้โมเดลเดามีแค่อีเมลแบบที่คุณอยากได้จริง ๆ. นี่คือสาเหตุที่มันให้ผลนิ่งกว่าและตรงกว่า — ไม่ใช่เพราะถ้อยคำวิเศษ แต่เพราะมันเหลือทางพังให้น้อยลง.

## เทียบให้เห็นภาพ — สั่งงานน้องใหม่

ลองคิดว่าการเขียน prompt เหมือนสั่งงานน้องใหม่ที่เพิ่งเข้าทีมวันแรก. ถ้าคุณบอกแค่ "ช่วยทำสไลด์ให้หน่อย" น้องก็ต้องเดาเอาเองว่าสไลด์เรื่องอะไร ให้ใครดู ยาวกี่หน้า โทนทางการแค่ไหน — แล้วโอกาสที่จะออกมาตรงใจตั้งแต่รอบแรกก็น้อย. แต่ถ้าคุณบอกบริบทครบ ยื่นตัวอย่างสไลด์เก่าที่หัวหน้าชอบให้ดู บอกจำนวนหน้าและโครงที่อยากได้ — งานก็ออกมาตรงขึ้นเยอะ. ยิ่งบอกชัด ยิ่งตรง.

จุดที่การเปรียบเทียบนี้พัง: น้องใหม่ตัวจริง พอทำงานกับคุณไปสักพักก็จะ *จำ* ได้เองว่าคุณชอบอะไร ครั้งต่อไปไม่ต้องบอกซ้ำ. แต่ AI จำไม่ได้ — มันเริ่มจากศูนย์ทุกครั้งที่ตอบ และสิ่งที่มันรู้ในแต่ละตามีแค่ข้อความที่ถูกป้อนเข้าไปในตานั้น ๆ เท่านั้น. เพราะฉะนั้นบริบทที่น้องจริงจำได้เอง คุณต้อง *เขียนป้อนให้ AI ใหม่ทุกครั้ง*. (เรื่องนี้คือหัวใจของบท "AI ไม่ได้จำคุณ" — ถ้ายังไม่ได้อ่าน แนะนำให้ย้อนไปดู เพราะมันคือเหตุผลที่คนใช้ AI เก่ง ๆ เก็บบริบทไว้ในไฟล์แล้วยื่นให้อ่านทุกครั้ง.)

## แล้ว prompt ที่พังล่ะ — แก้ยังไง

นี่คือจุดที่เปลี่ยนคนใช้ AI ทั่วไปให้กลายเป็นคนที่ใช้ได้จริง. เวลา prompt ออกมาไม่ดี คนส่วนใหญ่ทำสองอย่าง: กดส่งซ้ำหวังว่ารอบหน้าจะดีขึ้น หรือรื้อ prompt ใหม่ทั้งก้อน. ทั้งสองอย่างคือการเดาสุ่ม.

ทางที่ได้ผลกว่าคือ *เปลี่ยนทีละคันโยก แล้วอ่านว่าผลขยับยังไง*. คำตอบยาวเกิน — เติมคันโยกโครงสร้าง ("ไม่เกิน 3 ข้อ"). โทนไม่ใช่ — ปรับคันโยกบทบาท. หน้าตาไม่ตรงที่ต้องการ — ยื่นตัวอย่างให้ดู. พอคุณเปลี่ยนทีละอย่าง คุณจะเห็นว่าคันโยกตัวไหนทำให้ผลขยับไปทางไหน และครั้งต่อไปคุณก็จะรู้ว่าต้องดึงตัวไหน. การกดส่งซ้ำเฉย ๆ ไม่เคยสอนอะไรคุณเลย เพราะคุณไม่ได้เปลี่ยนอะไร — คุณแค่ขอให้มันเดาใหม่ในความกว้างเดิม.

## เรื่องของคนทำงานภาษาไทย

มีอีกมุมที่คนไทยควรรู้. ภาษาไทยกินจำนวน token มากกว่าภาษาอังกฤษราว 3-5 เท่า (เราเจาะเรื่อง token ไว้แล้วในบทต้น ๆ). แปลว่า prompt ภาษาไทยยาว ๆ ไม่ได้แค่เปลืองพื้นที่ context เร็วกว่า แต่ยัง *แพงกว่า* ด้วยถ้าคุณจ่ายตามปริมาณการใช้ (เรื่องค่าใช้จ่ายมีบทเฉพาะใน Act นี้).

ข่าวดีคือคันโยกทั้งสี่ตัวไม่ได้แปลว่าต้องเขียนยาว. prompt ที่ดีคือ prompt ที่ *บีบความเป็นไปได้ได้ตรงจุด* ไม่ใช่ prompt ที่ยาวที่สุด. การระบุบทบาทหนึ่งบรรทัด ยื่นตัวอย่างหนึ่งชิ้น และกำหนดโครงสามข้อ อาจบีบผลลัพธ์ได้แม่นกว่าการเขียนพรรณนาเป็นย่อหน้า — และกินที่น้อยกว่า. กระชับและตรงประเด็นคือเพื่อนของคุณ ทั้งในแง่คุณภาพและในแง่บิล.

### ลองทำดู

หยิบงานที่คุณเคยให้ AI ทำแล้วผลออกมางั้น ๆ มาหนึ่งงาน. รอบแรก พิมพ์คำสั่งกว้าง ๆ แบบที่เคยพิมพ์ เช่น "ช่วยสรุปประชุมนี้ให้หน่อย" แล้วดูผล. จากนั้นเปิดแชตใหม่ แล้วเติมคันโยกเข้าไปทีละตัว: เพิ่มบทบาท ("คุณคือผู้ช่วยที่สรุปให้ผู้บริหารที่ไม่มีเวลาอ่านยาว") เพิ่มโครงสร้าง ("3 หัวข้อ: ตัดสินใจอะไรไปแล้ว / ใครต้องทำอะไรต่อ / เรื่องที่ยังค้าง") แล้วดูว่าผลขยับไปยังไง.

สังเกตสองอย่าง — หนึ่ง คำตอบรอบหลังตรงกว่าและนิ่งกว่าชัดเจน. สอง คุณรู้แล้วว่า *คันโยกตัวไหน* ทำให้มันดีขึ้น. นั่นคือสิ่งที่การกดส่งซ้ำหวังดวงไม่เคยให้คุณ.

## คลายปมแล้ว และก้าวต่อไป

ย้อนกลับไปที่ความสับสนตอนต้น. prompt วิเศษไม่มีจริง เพราะไม่มีถ้อยคำไหนปลุกพลังลับของโมเดลได้ — มันยังเดาคำถัดไปอยู่เสมอ. และคำสั่งเดียวกันบางทีดีบางทีพัง เพราะคำสั่งกว้างเปิดทางให้คำตอบที่เป็นไปได้กว้างเกินไป โมเดลเลยเดินไปคนละทางได้ในแต่ละครั้ง. งานของคุณไม่ใช่หาคาถา แต่คือบีบความกว้างนั้นด้วยคันโยกที่ขยับผลได้จริง แล้วแก้ทีละคันโยกเมื่อมันพัง.

ถ้าคุณอยากได้คลังรูปแบบ prompt ที่ละเอียดกว่านี้ — เทคนิคเฉพาะทางหลายแบบที่หยิบไปใช้ได้เลย — เราเก็บไว้เป็นรายการอ้างอิงแยกต่างหากให้เปิดดูได้ตอนต้องการ. แต่คุณไม่จำเป็นต้องท่องมัน เพราะพอคุณเข้าใจว่าทุกเทคนิคคือคันโยกบนความน่าจะเป็น คุณก็อ่านมันออกเองทุกอัน.

แต่การเขียน prompt เก่งยังตอบไม่ได้ทุกเรื่อง. เพราะต่อให้ prompt ดีแค่ไหน AI ก็ยังตอบด้วยน้ำเสียงลื่น ๆ มั่นใจ ๆ ทั้งที่บางทีมันมั่ว. คำถามต่อไปคือ — แล้วคุณจะรู้ได้ยังไงว่าคำตอบที่ได้ "ดีพอ" และไม่ได้กำลังหลอกคุณอยู่. บทต่อไปจะสอนวิธีเช็คงานของ AI โดยไม่ต้องเชื่อคำพูดลื่น ๆ ของมัน.
