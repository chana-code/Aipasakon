# แผนที่ศัพท์ AI ในรูปเดียว — มันคือชั้น ไม่ใช่บันได

ลองนึกถึงเวลาคุณพยายามตามข่าว AI หรือดูคลิปสอนใช้ AI สักคลิป. ภายในห้านาที คุณจะโดนคำพวกนี้ถล่มใส่ — agent, sub-agent, skill, MCP, CLI, API, loop, hook. แต่ละคำคนพูดเหมือนคุณควรจะรู้อยู่แล้ว และไม่มีใครหยุดอธิบายว่ามันต่างกันยังไง.

แล้วคำถามที่ค้างอยู่ในหัวคุณก็คือ — *จะเริ่มเรียนตัวไหนก่อน?* เหมือนมันมีลำดับ มี "ขั้นที่ 1" ที่ต้องผ่านก่อนถึงจะขยับไป "ขั้นที่ 2" ได้. และพอเห็นคำว่า MCP โผล่มาคู่กับ API บ่อย ๆ คุณก็เริ่มสงสัยว่า — มันก็แค่ API ที่เปลี่ยนชื่อให้ดูเท่ขึ้นหรือเปล่า?

ปัญหาไม่ได้อยู่ที่คุณตามไม่ทัน. ปัญหาอยู่ที่ภาพในหัวคุณผิดตั้งแต่ต้น. คุณกำลังมองคำพวกนี้เป็น *บันได* — ขั้นความยากที่ต้องไต่ทีละขั้น. แต่จริง ๆ แล้วมันไม่ใช่บันได.

## ความเข้าใจผิดเรื่อง "บันได"

ถ้าคุณมองคำพวกนี้เป็นบันได คุณจะคิดประมาณว่า — เริ่มจาก API ก่อน พอเก่งแล้วค่อยขยับขึ้นไป CLI แล้วค่อยไป MCP แล้วสุดท้ายถึงจะ "ปลดล็อก" agent ซึ่งเป็นขั้นสูงสุด. เหมือนเล่นเกมที่ต้องผ่านด่านตามลำดับ.

ความเชื่อนี้ทำให้คนติดอยู่กับที่. บางคนไม่กล้าลองทำอะไรเลย เพราะคิดว่า "ฉันยังไม่ถึงขั้นนั้น". บางคนไปนั่งเรียน MCP ทั้งที่งานของเขาไม่ต้องใช้มันเลย เพราะคิดว่ามันเป็น "ขั้นที่ต้องผ่าน". ทั้งคู่เสียเวลากับลำดับที่ไม่มีอยู่จริง.

เพราะคำพวกนี้ไม่ใช่ระดับความยากที่เรียงกัน. มันคือ *ชื่อของส่วนต่าง ๆ* ในระบบเดียวกัน — ระบบที่เราแกะดูมาตั้งแต่ต้นหมวดนี้แล้ว.

## กลับไปที่ harness — ระบบที่ห่อโมเดลไว้

จำภาพจากบทแรกของหมวดนี้ได้ไหม. สิ่งที่คุณใช้ทุกวันไม่ใช่ตัวโมเดลเปล่า ๆ แต่เป็น *ผลิตภัณฑ์* ที่มีชั้นซอฟต์แวร์ห่อโมเดลไว้อีกที — ชั้นที่ฝรั่งเรียกว่า **harness**. ตัวโมเดลทำได้อย่างเดียวคือเดาคำถัดไป. ส่วน harness คือทุกอย่างที่อยู่รอบ ๆ ที่ทำให้มัน "ทำสิ่งต่าง ๆ ได้" — คำสั่งลับ, เครื่องมือ, ความจำ, และวงจรที่เรียกโมเดลซ้ำ. (รายละเอียดเต็มอยู่ในบท "โมเดลไม่ใช่ผลิตภัณฑ์".)

นี่คือกุญแจของทั้งบทนี้: **คำศัพท์ที่ทำให้คุณงงทั้งหมด — agent, skill, MCP, CLI, API, loop, hook — แทบทุกคำคือชื่อของส่วนใดส่วนหนึ่งใน harness นี้ หรือชื่อของวิธีตั้งค่ามัน.** ไม่มีคำไหนเป็น "ขั้นที่สูงกว่า" คำอื่น. มันเป็นแค่ส่วนประกอบคนละชิ้นของเครื่องเดียวกัน.

พอคุณเห็นว่ามันเป็นชั้นของระบบเดียว ไม่ใช่ขั้นบันได คำถาม "เริ่มตัวไหนก่อน" ก็หายไปเอง — เพราะคุณไม่ได้ "ไต่" มัน คุณแค่หยิบชิ้นที่งานของคุณต้องใช้.

## กางทุกคำในรูปเดียว

ทีนี้มาวางแต่ละคำลงบนแผนผังเดียวกัน. อ่านแต่ละอันแค่ให้พอเห็นว่ามันคือ *ชั้นไหน* ของระบบ — ไม่ต้องจำให้ขึ้นใจ เพราะจุดประสงค์คือให้คุณเห็นภาพรวม ไม่ใช่ท่องนิยาม.

- **API** — ท่อตรงที่ต่อถึงโมเดลเปล่า ๆ ไม่มี harness ห่อ. มันคือ "ประตูหลัง" ที่โปรแกรมเมอร์ใช้เรียกโมเดลมาประกอบเองในแอปของตัวเอง. (เราพูดเรื่องนี้ไปแล้วในบท "แอปกับ API ทำไมตอบไม่เหมือนกัน".)
- **CLI** — คือ harness ที่คุณ *ขับจากหน้าจอพิมพ์คำสั่ง* (terminal) แทนที่จะเป็นหน้าแชตสวย ๆ. มันไม่ได้ "สูงกว่า" แอปแชต — มันแค่เป็นหน้าตาอีกแบบของ harness เดียวกัน.
- **loop** — วงจรที่เรียกโมเดลซ้ำ ๆ: เรียกโมเดล เห็นว่ามันอยากทำอะไร ทำให้ แล้วเรียกอีกที วนจนเสร็จ. นี่คือกาวที่ยึด harness ไว้ด้วยกัน (อธิบายไว้แล้วในบท "โมเดลไม่ใช่ผลิตภัณฑ์").
- **tool** — ความสามารถที่ harness "ยื่นให้" โมเดลใช้ เช่น ค้นเว็บ รันโค้ด อ่านไฟล์. โมเดลไม่ได้ลงมือทำเอง มันแค่ *ขอ* แล้ว harness ทำให้ (เจาะลึกไว้ในบท "เครื่องเดาคำใช้เครื่องมือได้ยังไง").
- **skill** — ชุดคำสั่ง/สูตรสำเร็จที่เขียนเก็บไว้ล่วงหน้า แล้วเรียกใช้ซ้ำได้ เหมือนสูตรอาหารในแฟ้มที่หยิบมาทำตามได้เมื่อต้องการ.
- **MCP** — ปลั๊กมาตรฐานที่ทำให้เครื่องมือจากเจ้าไหนก็ได้ เสียบเข้ากับ harness ด้วยวิธีเดียวกัน. (อันนี้คนเข้าใจผิดบ่อยสุด — เดี๋ยวเราเจาะต่อข้างล่าง.)
- **hook** — กฎที่ตั้งไว้ให้ "ยิงอัตโนมัติ" เมื่อถึงจุดใดจุดหนึ่งใน loop เช่น "ทุกครั้งก่อนรันโค้ด ให้เซฟไฟล์ก่อน". เป็นการแทรกกฎของคุณเข้าไปในวงจร.
- **agent** — harness ที่ถูกตั้งให้ *วน loop ไปเองจนกว่าจะถึงเป้าหมาย* โดยคุณไม่ต้องกดสั่งทุกขั้น. มันไม่ใช่ AI พันธุ์พิเศษ มันคือ harness ที่ปล่อยให้เดินเองได้ไกลขึ้น (เจาะลึกในบท "ทีม AI แบบจาร์วิส").
- **sub-agent** — agent ตัวเล็กที่ agent ตัวหลัก "เรียกมาช่วย" งานย่อย แล้วเอาผลกลับมา (เรื่องนี้มีรายละเอียดในบท "Sub-agent ตัวช่วยที่ย้อนแย้ง").

สังเกตไหมว่าไม่มีคำไหนเลยที่ "อยู่สูงกว่า" คำอื่น. agent ไม่ได้ยากกว่า API. MCP ไม่ได้เป็นด่านสุดท้ายหลัง CLI. มันคือชิ้นส่วนคนละชิ้นของเครื่องเดียวกัน — บางชิ้นซ้อนทับกันด้วยซ้ำ (agent ตัวหนึ่ง *ใช้* tool ที่เสียบผ่าน MCP ได้ในเวลาเดียวกัน).

## ฆ่าความเข้าใจผิด: "MCP ก็แค่ API เปลี่ยนชื่อ"

อันนี้คือจุดที่คนสับสนมากที่สุด เพราะ MCP กับ API ฟังดูเป็นเรื่องเทคนิคพอ ๆ กัน เลยดูเหมือนของอย่างเดียวกัน. แต่จริง ๆ มันอยู่กันคนละชั้น.

**API** คือ *ปลายทางของตัวโมเดล* — ท่อที่ต่อเข้าหาเครื่องเดาคำโดยตรง. ส่วน **MCP** คือ *ปลั๊กมาตรฐานสำหรับต่อเครื่องมือเข้ากับ harness*. พูดง่าย ๆ — API ตอบคำถามว่า "จะคุยกับโมเดลยังไง" ส่วน MCP ตอบคำถามว่า "จะเสียบเครื่องมือใหม่ ๆ เข้าให้ harness ใช้ยังไงให้เป็นแบบเดียวกันหมด".

กลับไปที่ภาพร้านอาหารก็เห็นชัด — API คือท่อที่ต่อตรงเข้าหาแม่ครัวในครัว. ส่วน MCP คือ *รูปลั๊กมาตรฐานหลังครัว* ที่ทำให้เครื่องครัวยี่ห้อไหนก็เสียบเข้าใช้งานได้แบบเดียวกัน. คนละบทบาทกันโดยสิ้นเชิง. MCP ไม่ใช่ของวิเศษ และไม่ใช่ API ที่เปลี่ยนชื่อ — มันแค่เป็น *ข้อตกลงร่วม* (convention) ว่าจะต่อเครื่องมือเข้ากันยังไงให้ไม่ต้องประดิษฐ์วิธีใหม่ทุกครั้ง.

> 💡 **ใจความสำคัญ:** คำศัพท์ AI ที่ทำให้คุณงง ไม่ใช่ขั้นบันไดความยากที่ต้องไต่ทีละขั้น — มันคือชื่อของชั้นต่าง ๆ ใน harness ตัวเดียวกัน. คุณไม่ต้อง "เรียนตามลำดับ" คุณแค่หยิบชิ้นที่งานของคุณต้องใช้.

## เทียบให้เห็นภาพ: ส่วนต่าง ๆ ของร้านอาหาร

ตลอดหมวดนี้เราใช้ภาพร้านอาหารมาตลอด — โมเดลคือแม่ครัวที่ปรุงได้ทีละจานจากของตรงหน้าแล้วลืม ส่วน harness คือทั้งร้านที่ทำให้ครัวนั้นเสิร์ฟลูกค้าได้จริง. ลองวางคำศัพท์พวกนี้ลงในภาพร้านเดิม:

แม่ครัว คือ *โมเดล*. ระบบรับออร์เดอร์แล้ววิ่งงานไปจนจบ คือ *loop* และเมื่อปล่อยให้มันเดินเองจนเสร็จงานก็คือ *agent*. รูปลั๊กมาตรฐานหลังครัวที่เสียบเครื่องครัวยี่ห้อไหนก็ได้ คือ *MCP*. สูตรอาหารในแฟ้มที่หยิบมาทำตามได้ คือ *skill*. ผู้ช่วยที่ผู้จัดการเรียกมาเสริมเฉพาะกิจ คือ *sub-agent*. ประเด็นคือ — สิ่งเหล่านี้ไม่ใช่ "เลเวลของพ่อครัว" ที่ต้องเลื่อนขั้น. มันคือ *ส่วนต่าง ๆ ของร้าน* ที่ทำงานพร้อมกัน.

จุดที่การเปรียบเทียบนี้พัง: ของในร้านจริงแยกชิ้นชัดเจน — ครัวก็คือครัว แฟ้มสูตรก็คือแฟ้มสูตร. แต่คำศัพท์ AI พวกนี้ *ซ้อนทับกันได้* มากกว่าของในร้าน. agent ตัวหนึ่งอาจใช้ tool หลายตัวที่เสียบผ่าน MCP พร้อมกัน ในขณะที่ทำงานตาม skill อยู่. เพราะฉะนั้นให้มองแผนผังนี้เป็น *ภาพจัดวางคร่าว ๆ ว่าอะไรเป็นชั้นไหน* ไม่ใช่ช่องแยกที่ตายตัว.

## แล้วคนทั่วไปต้องเรียนตัวไหนบ้าง

คำตอบที่ทำให้คุณโล่งใจ — สำหรับการใช้ AI ทำงานในชีวิตประจำวัน คุณ *แทบไม่ต้องแตะ* คำพวกนี้เลยด้วยซ้ำ. คุณไม่ต้องตั้งค่า MCP คุณไม่ต้องใช้ CLI คุณไม่ต้องประกอบ agent เอง. สิ่งเหล่านี้เป็นเครื่องมือสำหรับคนที่ต้องการ *สร้าง* ระบบ AI ของตัวเอง — ซึ่งเราจะลงมือทำจริงในโมดูลทีหลัง ถ้าคุณอยากไปถึงตรงนั้น.

สิ่งที่คุณต้องมีจริง ๆ ตอนนี้ ไม่ใช่การจำว่า hook ต่างจาก skill ยังไง. มันคือการ *เห็นภาพรวมว่าคำพวกนี้คือชั้นของระบบเดียวกัน* — เพื่อว่าครั้งหน้าที่เจอมันในข่าวหรือในคลิป คุณจะไม่ตื่นตระหนกว่าตัวเองตามไม่ทัน. คุณจะรู้ว่า "อ๋อ อันนี้แค่ชื่อของชั้นหนึ่งใน harness" แล้วเลือกได้เองว่าอันไหนเกี่ยวกับงานคุณ อันไหนปล่อยผ่านได้.

### ลองทำดู

เปิดข่าว AI หรือคลิปสอนใช้ AI ที่คุณเคยอ่านแล้วงง ๆ ขึ้นมาอีกรอบ. คราวนี้ทุกครั้งที่เจอคำพวก agent, skill, MCP, CLI, API ให้หยุดแล้วถามตัวเองคำเดียว — "คำนี้มันคือชั้นไหนของ harness?" เป็นท่อถึงโมเดล (API)? เป็นหน้าตาอีกแบบ (CLI)? เป็นเครื่องมือที่ยื่นให้ (tool/MCP)? หรือเป็นทั้งระบบที่เดินเอง (agent)?

คุณจะรู้สึกได้ทันทีว่าบทความเดิมที่เคยอ่านแล้วเบลอ กลายเป็นอ่านรู้เรื่องขึ้นเยอะ — ไม่ใช่เพราะคุณไปเรียนแต่ละคำมาแบบลึก แต่เพราะคุณมี *แผนที่* ให้วางคำพวกนั้นลงไปแล้ว.

## ความสับสนเรื่องศัพท์หายไปแล้ว — ทีนี้ไปต่อ

กลับไปที่ความรู้สึกตอนเปิดบท — คำพวก agent, skill, MCP, CLI, API ที่เคยปนกันจนไม่รู้จะเริ่มตรงไหน. ตอนนี้คุณเห็นแล้วว่ามันไม่ใช่บันไดที่ต้องไต่ มันคือชั้นต่าง ๆ ของ harness ตัวเดียวที่เราแกะดูมาตั้งแต่ต้นหมวดนี้. ไม่มี "ขั้นที่ 1" ที่ต้องผ่านก่อน. และ MCP ก็ไม่ใช่ API เปลี่ยนชื่อ — มันคือปลั๊กมาตรฐาน คนละชั้นกัน.

มีคำหนึ่งในแผนที่นี้ที่กำลังเป็นกระแสแรงที่สุด และก็เข้าใจผิดกันมากที่สุดด้วย นั่นคือ **agent** — โดยเฉพาะไอเดียที่ว่า "ถ้า agent ตัวเดียวเก่ง งั้นเอาหลาย ๆ ตัวมาประกอบเป็นทีมแบบจาร์วิส มันต้องเก่งกว่าสิ". บทต่อไปจะพาดูว่าทำไมทีม AI หลายตัวถึงมัก *ช้ากว่า โง่กว่า แพงกว่า* การใช้ตัวเดียวที่ตั้งมาดี — และเมื่อไหร่ที่มันคุ้มจริง ๆ.
