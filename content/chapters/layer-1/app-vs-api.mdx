# โมเดลเดียวกัน ทำไมในแอปกับใน API ตอบไม่เหมือนกัน

สมมติคุณเริ่มสงสัยว่า ChatGPT ที่ใช้ในเว็บทุกวัน กับ "API" ที่เพื่อนสายเทคชอบพูดถึง มันต่างกันยังไง. คุณเลยลองส่งคำถามเดียวกันเป๊ะ ๆ เข้าไปทั้งสองทาง — ตั้งให้เป็นโมเดลตัวเดียวกันด้วย เช่น GPT รุ่นเดียวกัน.

แล้วคุณก็เจอเรื่องแปลก. คำตอบมันคนละเรื่อง. ในเว็บมันตอบยาว สุภาพ ค้นข่าววันนี้มาให้ด้วย. ส่วนทาง API มันตอบสั้น ห้วน และพอถามข่าววันนี้ มันกลับบอกว่า "ผมไม่มีข้อมูลหลังปีที่ฝึกเสร็จ".

โมเดลเดียวกันแท้ ๆ — แล้วทำไมตอบไม่เหมือนกัน. อันไหนกันแน่ที่ "พัง" ? และถ้ามันคนละอย่างกัน แปลว่าคุณต้องจ่ายเงินสองต่อหรือเปล่า — ทั้งค่าสมาชิกรายเดือน ทั้งค่า API?

## ไม่มีอันไหนพัง — มันคือสองสภาพของของชิ้นเดียวกัน

คำตอบสั้น ๆ คือ ไม่มีอันไหนพังเลย. สิ่งที่คุณเพิ่งเห็นคือโมเดลตัวเดียวกัน ในสองสภาพที่ต่างกันมาก.

จำเรื่องจากบทก่อนได้ไหม — ที่เราบอกว่าสิ่งที่คุณใช้ในเว็บที่ชื่อ ChatGPT จริง ๆ ไม่ใช่ตัวโมเดล แต่เป็น *ผลิตภัณฑ์ที่ห่อโมเดลไว้อีกชั้น* และชั้นที่ห่อนั้นเราเรียกว่า **harness** (โครงที่ครอบโมเดล — คำสั่งลับที่ตั้งไว้ก่อน, เครื่องมือ, ความจำ, และวงจรที่เรียกโมเดลซ้ำ). ถ้ายังไม่ชัด ย้อนกลับไปอ่านบท "โมเดลไม่ใช่ผลิตภัณฑ์" ก่อนได้ บทนี้จะต่อยอดจากตรงนั้น.

ความต่างทั้งหมดที่คุณเห็น มาจากเรื่องเดียว: **เว็บกับ API ห่อ harness รอบโมเดลไม่เท่ากัน**.

- เวลาคุณใช้ ChatGPT ในเว็บ คุณได้ทั้งคัน — โมเดลบวก harness ที่ OpenAI ประกอบมาให้เสร็จสรรพ.
- เวลาคุณเรียกผ่าน API คุณได้แค่เครื่องยนต์เปล่า ๆ — โมเดลตัวเดียวกันแหละ แต่ไม่มี harness มาให้. ถ้าอยากได้ harness คุณต้องสร้างเอง.

## แอปแอบเติมอะไรให้บ้าง ที่ API ไม่มี

ลองดูทีละอย่างว่า ตอนคุณพิมพ์ในเว็บ มีอะไรถูกแอบเติมเข้าไปก่อนถึงโมเดล ที่ฝั่ง API ไม่มีให้:

**คำสั่งลับที่ตั้งไว้ก่อน (system prompt).** ในเว็บ ก่อนข้อความของคุณจะถึงโมเดล จะมีข้อความชุดหนึ่งถูกแปะนำหน้าเสมอ — ประมาณว่า "คุณคือผู้ช่วยที่สุภาพ วันนี้คือวันที่... ตอบให้ละเอียดและเป็นมิตร". คุณไม่เคยเห็นมัน แต่มันทำให้คำตอบในเว็บออกมายาวและสุภาพ. ส่วน API ไม่มีคำสั่งลับนี้เลย เว้นแต่คุณจะเขียนใส่เอง — เพราะงั้นมันเลยตอบห้วน ๆ ตรง ๆ ตามค่าตั้งต้น.

**เครื่องมือ เช่น ค้นเว็บ.** นี่คือเหตุผลที่ในเว็บมันค้นข่าววันนี้ให้ได้ แต่ API บอกว่าไม่รู้. ตัวโมเดลเองความรู้หยุดอยู่ที่วันฝึกเสร็จทั้งสองทาง — แต่เว็บมีเครื่องมือค้นเว็บต่อไว้ให้ พอถามข่าว มันก็วิ่งไปค้นมาป้อนให้โมเดลอ่าน. ส่วน API เปล่า ๆ ไม่มีเครื่องมือต่อไว้ โมเดลเลยทำได้แค่บอกตามตรงว่ามันไม่รู้.

**ความจำและประวัติการคุย.** ในเว็บ แอปจะคอยมัดบทสนทนาเก่ากับสิ่งที่มันจดเกี่ยวกับคุณ ส่งกลับเข้าไปให้โมเดลทุกครั้ง. ส่วน API ไม่จำอะไรให้เลย — ทุกครั้งที่เรียก มันเริ่มจากศูนย์ ถ้าอยากให้มันรู้บทสนทนาก่อนหน้า คุณต้องแนบเองทั้งหมด.

**ค่าตั้งต้นต่าง ๆ ที่ตั้งมาให้แล้ว.** เว็บปรับค่าหลายอย่างไว้ให้เหมาะกับคนทั่วไป — เช่นความยาวคำตอบ หรือระดับความสร้างสรรค์ในการเลือกคำ (เรื่องว่าทำไมถามซ้ำแล้วได้คำตอบต่างกัน เราอธิบายไว้ในบทก่อน ๆ แล้ว). ส่วน API ส่งค่าดิบ ๆ มา ให้คุณตั้งเองทุกตัว.

> 💡 **ใจความสำคัญ:** เว็บกับ API ใช้โมเดลตัวเดียวกันได้ก็จริง แต่เว็บส่ง "ทั้งคันที่ประกอบเสร็จแล้ว" มาให้ ส่วน API ส่ง "เครื่องยนต์เปล่า ๆ" มาให้คุณประกอบเอง. คำตอบที่ต่างกันไม่ได้แปลว่าตัวไหนพัง — มันแปลว่า harness รอบโมเดลไม่เท่ากัน.

## เทียบให้เห็นภาพ: นั่งกินในร้าน กับ เข้าครัวเปล่า ๆ

ลองคิดว่าโมเดลคือ *แม่ครัวฝีมือดี* คนหนึ่ง — คนเดียวกันทั้งสองกรณี.

**ใช้ในเว็บ = นั่งกินในร้านอาหาร.** คุณบอกว่าอยากกินอะไร เดี๋ยวมีคนจัดโต๊ะให้ เสิร์ฟน้ำให้ จัดจานสวย ๆ มาเสร็จ. คุณไม่ต้องรู้ว่าในครัวเกิดอะไร แค่สั่งแล้วรอกิน. สะดวก ครบ จบในที่เดียว.

**ใช้ผ่าน API = เดินเข้าไปในครัว ขอคุยกับแม่ครัวตรง ๆ.** แม่ครัวคนเดิม ฝีมือเท่าเดิม. แต่ในครัวไม่มีพนักงานเสิร์ฟ ไม่มีคนจัดโต๊ะ ไม่มีน้ำมาเสิร์ฟ. ถ้าคุณอยากได้น้ำ คุณต้องบอกเอง อยากได้จานสวยก็ต้องจัดเอง. แลกกับความยุ่งยากนั้น คุณคุมได้ทุกอย่าง — สั่งให้ทำเผ็ดพิเศษ ใส่วัตถุดิบที่คุณเอามาเอง ทำซ้ำ 500 จานแบบเป๊ะ ๆ ก็ได้.

จุดที่การเปรียบเทียบนี้พัง: คำว่า "เข้าครัวเปล่า ๆ" อาจฟังเหมือน API เป็นของด้อยกว่าหรือสำหรับมือใหม่ที่ยังไม่มีร้าน. จริง ๆ ตรงข้าม — API คือสิ่งที่คนสร้างผลิตภัณฑ์ AI ทั้งหลาย (รวมถึงตัวเว็บ ChatGPT เอง) ใช้เป็นวัตถุดิบตั้งต้น. มันไม่ได้ด้อยกว่า แค่ "ดิบกว่า" และสำหรับงานบางอย่าง การคุมเองทุกขั้นนั่นแหละคือข้อดี ไม่ใช่ข้อเสีย.

### ลองทำดู

ถ้าคุณมีสิทธิ์เข้า Playground หรือ API console ของผู้ให้บริการ (ถ้าไม่มี ข้ามไปอ่านการทดลองแบบที่สองได้) ลองทำแบบนี้: ส่งคำถามที่ต้องใช้ข้อมูลวันนี้ เช่น "ราคาทองวันนี้เท่าไหร่" เข้าไปทาง API แบบเปล่า ๆ. คุณจะเห็นมันบอกว่าไม่มีข้อมูลล่าสุด. จากนั้นเปิดถามคำถามเดียวกันในเว็บ ChatGPT แล้วสังเกตคำว่า "กำลังค้นหา" ที่โผล่มา. นั่นคือคุณเห็นกับตาว่า ความต่างไม่ได้อยู่ที่โมเดล แต่อยู่ที่เครื่องมือที่ห่อรอบมัน.

ถ้าไม่มี API ก็ทำแบบนี้แทน: ในเว็บ ChatGPT ลองสร้าง Project หรือเขียน Custom Instructions ว่า "ตอบสั้น ๆ เป็น bullet เท่านั้น". แล้วถามคำถามเดิมในแชตปกติที่ไม่ได้ตั้งค่านี้ เทียบกับในแชตที่ตั้งค่าไว้. คุณจะเห็นว่าคำสั่งลับที่คุณเพิ่งใส่ ทำให้โมเดลตัวเดิมตอบคนละสไตล์ — นี่คือตัวอย่างย่อ ๆ ของสิ่งที่ harness ทำ.

## แล้วเรื่องเงิน — ต้องจ่ายสองต่อไหม

นี่คือจุดที่หลายคนสับสนหนักที่สุด และมันเป็นคนละเรื่องกับเรื่องคุณภาพคำตอบเลย.

ค่าสมาชิกรายเดือน (เช่น ChatGPT Plus) กับค่า API เป็น *คนละบริการกัน* คุณไม่ได้จ่ายซ้ำซ้อนกัน:

- **ค่าสมาชิกรายเดือน** คือค่า "นั่งกินในร้าน" — จ่ายก้อนเดียวต่อเดือน ใช้เว็บ/แอปได้เต็มที่ พร้อม harness ที่เขาประกอบให้ ไม่ว่าจะถามกี่ครั้งก็ราคาเดียว.
- **ค่า API** คือค่า "วัตถุดิบที่ตักไปใช้ในครัวคุณเอง" — จ่ายตามปริมาณที่ใช้จริง นับเป็น token (หน่วยที่โมเดลใช้วัดข้อความ — เราอธิบายไว้แล้วในบทเรื่อง token). ยิ่งส่งข้อความเข้า-ออกมาก ยิ่งจ่ายมาก. ไม่ใช้ ไม่จ่าย.

คุณ *ไม่จำเป็น* ต้องมีทั้งสองอย่าง. คนทั่วไปที่แค่อยากแชตกับ AI จ่ายค่าสมาชิกรายเดือนอย่างเดียวก็พอ ไม่ต้องแตะ API เลย. ส่วน API มีไว้สำหรับตอนที่คุณอยากให้ AI ไปทำงานในโปรแกรมของคุณเอง หรือทำงานซ้ำ ๆ เป็นพัน ๆ ครั้งแบบอัตโนมัติ — งานที่นั่งแชตในเว็บทีละข้อความไม่ไหว.

สำหรับคนไทยมีเรื่องที่ควรรู้เพิ่มอีกนิด. การจ่ายค่า API มักคิดเป็นเงินดอลลาร์ และผูกกับบัตรเครดิตที่จ่ายต่างประเทศได้ — ซึ่งบางคนเจอปัญหาบัตรไม่ผ่านหรือโดนคิดค่าธรรมเนียมแปลงสกุลเงิน. และเพราะภาษาไทยกินจำนวน token มากกว่าภาษาอังกฤษหลายเท่า (เรื่องนี้เราเจาะไว้ในบทเรื่อง token ภาษาไทย) ค่า API เวลาทำงานภาษาไทยจึงแพงกว่าที่หลายคนคาดไว้. ส่วนค่าใช้จ่ายเชิงลึก — คิดยังไง คุมยังไงให้ไม่บานปลาย — เราจะลงรายละเอียดในบทเรื่องต้นทุนทีหลัง บทนี้ขอแค่ให้คุณเลิกงงว่า "ตกลงจ่ายซ้ำหรือเปล่า" ก่อน.

## สรุปความสับสนที่เปิดบทมา

กลับไปที่คำถามแรก. prompt เดียวกัน โมเดลเดียวกัน แต่เว็บกับ API ตอบไม่เหมือนกัน — ไม่ใช่เพราะตัวไหนพัง. เว็บส่งทั้งคันที่ประกอบ harness เสร็จแล้วมาให้ ส่วน API ส่งเครื่องยนต์เปล่ามาให้คุณประกอบเอง. และค่าสมาชิกรายเดือนกับค่า API ก็เป็นคนละบริการ ไม่ใช่การจ่ายซ้ำ.

พอเห็นภาพนี้ คุณจะเข้าใจประโยคที่เคยฟังแล้วงงทันที — อย่างที่ว่า "เราสร้างผลิตภัณฑ์นี้บน API ของ OpenAI". มันแปลว่าเขาเอาเครื่องยนต์เปล่ามา แล้ว *สร้าง harness รอบมันเอง* — คำสั่งลับ เครื่องมือ ความจำ ทุกอย่างที่เว็บ ChatGPT มีให้ฟรี เขาต้องลงมือต่อเองหมด. นั่นคือสิ่งที่คุณจะได้ลองทำจริงในโมดูลสร้างเครื่องมือทีหลัง.

แต่ก่อนจะไปถึงตรงนั้น มีคำถามหนึ่งที่ค้างคามาตั้งแต่ต้น Act — ในเมื่อโมเดลทำได้แค่ "เดาคำถัดไป" แล้วไอ้ "เครื่องมือค้นเว็บ" ที่เราพูดถึงตลอดบทนี้ มันทำงานยังไง โมเดลที่ทำได้แค่พิมพ์ข้อความ มันไปกดค้นเว็บเองได้ยังไง. บทต่อไปจะเปิดฝาดูตรงนั้นพอดี.

---

*อ่านต่อ: เครื่องเดาคำ "ใช้เครื่องมือ" ได้ยังไง — ในเมื่อมันทำได้แค่พิมพ์ข้อความ*
