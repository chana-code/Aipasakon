# Neural Network คืออะไร — เครื่องจักรที่เรียนรู้จากตัวเลข

ทุกครั้งที่คุณพิมพ์ข้อความใน ChatGPT แล้วได้คำตอบกลับมาภายในวินาที สิ่งที่อยู่หลังจอนั้นไม่ใช่คนพิมพ์ตอบ ไม่ใช่ฐานข้อมูลที่ค้นหาคำตอบสำเร็จรูป และไม่ใช่โปรแกรมที่มีคนเขียนกฎเอาไว้ทุกข้อ. ถ้าเป็นกฎที่เขียนเอง ต้องเขียนกี่ล้านกฎถึงจะครอบคลุมภาษาทุกภาษา ความรู้ทุกเรื่อง และบริบทที่แตกต่างกันนับไม่ถ้วน?

สิ่งที่อยู่ข้างในคือ **neural network** — โครงสร้างทางคณิตศาสตร์ที่ประกอบจากหน่วยเล็ก ๆ นับพันล้านหน่วย แต่ละหน่วยทำงานง่ายมาก คูณกับบวก. แค่นั้นจริง ๆ. แต่เมื่อเอามาซ้อนกันหลายร้อยชั้นและ "สอน" ด้วยข้อความจากอินเทอร์เน็ตเกือบทั้งหมด มันกลายเป็นสิ่งที่ตอบคำถาม แปลภาษา เขียนโค้ด และวิเคราะห์เอกสารได้.

บทนี้จะเปิดกล่องให้ดูว่า neural network ประกอบจากอะไร ทำงานอย่างไรในระดับโครงสร้าง และทำไมมันถึงทรงพลังแบบที่เห็น. พอเข้าใจบทนี้แล้ว บท [LLM คืออะไร](/foundations/what-is-an-llm) ที่จะมาถึง จะต่อยอดโดยอธิบายว่า neural network ชนิดพิเศษที่เรียกว่า LLM ทำงานอย่างไร.

เมื่อเข้าใจบทนี้แล้ว คุณจะ "อ่าน" ข่าว AI ที่เห็นทุกวันได้อีกแบบ — ตัวเลขอย่าง "175 billion parameters" หรือ "96 transformer layers" จะไม่ใช่ตัวเลขลึกลับอีกต่อไป แต่คือตัวเลขที่บอกขนาดและความลึกของสมการที่กำลังทำงานทุกครั้งที่คุณกดส่ง.

บทนี้กับบทเรื่อง [LLM คืออะไร](/foundations/what-is-an-llm) เชื่อมกันโดยตรง — บทนี้อธิบายโครงสร้างพื้นฐาน ส่วนบท LLM ที่จะมาถึงจะนำโครงสร้างนั้นไปประกอบเป็น LLM ที่คุณใช้ทุกวัน.

## เริ่มจากหน่วยเล็กที่สุด — neuron คืออะไร

ชื่อ "neural network" (โครงข่ายประสาทเทียม) ฟังดูหรูหรา แต่หน่วยพื้นฐานของมันทำงานง่ายมาก.

ลองนึกถึงการตัดสินใจง่าย ๆ เช่น "ควรพกร่มออกไปไหมวันนี้?" คุณดูหลายปัจจัยพร้อมกัน: ท้องฟ้าครึ้มไหม, อากาศชื้นไหม, พยากรณ์อากาศบอกว่าอะไร. แล้วในหัวคุณ "ชั่งน้ำหนัก" แต่ละปัจจัย — ถ้าท้องฟ้าดำมากมันมีผลต่อการตัดสินใจมากกว่าความชื้นเล็กน้อย. รวมทุกอย่างแล้วได้คำตอบ "พกหรือไม่พก."

**artificial neuron** ทำสิ่งเดียวกันนั้น — แต่ทำด้วยตัวเลขล้วน ๆ:

1. รับตัวเลขหลายตัวเข้ามาเป็น input (เช่น x1, x2, x3)
2. คูณแต่ละตัวด้วยค่าที่เรียกว่า **weight** (น้ำหนัก) ของมันเอง
3. บวกผลรวมทั้งหมดเข้าด้วยกัน แล้วบวกค่าเสริมที่เรียกว่า **bias**
4. ส่งผลลัพธ์ผ่านฟังก์ชันที่ทำให้กราฟไม่เป็นเส้นตรง ที่เรียกว่า **activation function**
5. ส่งตัวเลขออกมาหนึ่งตัวเป็น output

ในสมการเขียนว่า:

```
output = activation(w1×x1 + w2×x2 + ... + wn×xn + b)
```

weight และ bias รวมกันเรียกว่า **parameter** ของโมเดล — และทั้งหมดที่ neural network "เรียนรู้" คือการหาค่า parameter ที่ดีที่สุดสำหรับงานที่ต้องการ.

weight แต่ละตัวบอกว่า input ตัวนั้นสำคัญแค่ไหนสำหรับ neuron ตัวนี้. weight สูง = input นั้นสำคัญมาก ส่งผลต่อ output มาก. weight ต่ำ (ใกล้ 0) = input นั้นแทบไม่มีผล. weight ติดลบ = input นั้น "ต้านทาน" — ยิ่งสูงยิ่งทำให้ neuron ไม่ติด.

bias เป็นตัวเลขที่ "เลื่อน" ผลลัพธ์ก่อนผ่าน activation function. ถ้าไม่มี bias neuron ทุกตัวจะต้องผ่านจุดศูนย์เสมอ ซึ่งจำกัดความยืดหยุ่น. bias ทำให้ neuron "เปิดหรือปิดโดย default" ได้อิสระขึ้น.

ในโมเดลขนาดใหญ่อย่าง GPT-3 มี parameter ราว 175,000 ล้านตัว. ทุกตัวถูกตั้งค่าเป็นตัวเลขสุ่มตอนเริ่มต้น แล้วค่อย ๆ ปรับผ่านกระบวนการ training จนได้ตัวเลขที่ทำให้โมเดลตอบได้ดี.

> 💡 **Key insight:** neuron เดี่ยว ๆ ตัวเดียวไม่ฉลาดอะไรเลย. มันแค่คูณและบวกตัวเลข. พลังของ neural network มาจากการเอา neuron หลายล้านตัวมาต่อกันเป็นชั้น ๆ — ไม่ใช่จากความฉลาดของแต่ละ neuron.

## ทำไมต้อง "activation function" — เรื่องสำคัญที่คนมักข้ามไป

ถ้าตัด activation function ออก แล้วให้ neuron แค่คูณและบวกตัวเลขล้วน ๆ จะเกิดอะไรขึ้น?

คำตอบคือ: ไม่ว่าจะซ้อน neuron กี่ชั้น ผลลัพธ์สุดท้ายก็ยังเป็นสมการเส้นตรงอยู่ดี. นี่เป็นข้อเท็จจริงทางคณิตศาสตร์ — ถ้าเอาฟังก์ชันเส้นตรงมาต่อกันสิบชั้น ผลลัพธ์ก็ยังเป็นเส้นตรงอยู่. network ที่ลึก 100 ชั้นแต่ไม่มี activation function ก็เทียบเท่า network 1 ชั้น.

ปัญหาของสมการเส้นตรงคือมันแทน pattern ซับซ้อนไม่ได้. ลองนึกถึงข้อมูลที่ไม่ได้แยกจากกันด้วยเส้นตรง เช่น รู้จำหน้าคน จำแนกว่าประโยคนี้มีเจตนาแบบไหน ทำนายว่าลูกค้าจะซื้อหรือไม่. โลกจริงเต็มไปด้วย pattern ที่ไม่เป็นเส้นตรง.

**activation function** คือฟังก์ชันที่เพิ่มความ "งอ" ให้กับ output ของแต่ละ neuron ก่อนส่งต่อ ทำให้เมื่อซ้อนหลาย ๆ ชั้นเข้าด้วยกัน network สามารถแทน pattern ที่ซับซ้อนได้.

activation function มีหลายชนิด ที่ใช้งานมาตลอดประวัติศาสตร์ของ neural network:

**Sigmoid** คือรุ่นเก่าที่สุด บีบตัวเลขทุกค่าให้อยู่ในช่วง 0 ถึง 1 เหมือนการแปลงให้เป็น probability. ใช้กันมากในยุคแรก แต่มีปัญหาที่เรียกว่า "vanishing gradient" — เมื่อ network ลึกมาก ๆ gradient จะเล็กมากจนชั้นแรก ๆ ไม่ได้รับสัญญาณ training ทำให้ train ยาก.

**Tanh** คล้าย sigmoid แต่บีบค่าให้อยู่ในช่วง -1 ถึง 1 แทน. คลุกรวมที่ 0 ทำให้ training เสถียรขึ้นเล็กน้อย แต่ยังมีปัญหา vanishing gradient เมื่อ network ลึกมาก.

**ReLU** (Rectified Linear Unit) คือตัวที่เปลี่ยนวงการ ทำงานง่ายมาก:
- ถ้าตัวเลขเป็นลบ ให้ output = 0
- ถ้าตัวเลขเป็นบวก ให้ output = ตัวเลขนั้นเลย

```
ReLU(x) = max(0, x)
```

ฟังดูง่ายมากเกินไปไหม? แต่นี่คือฟังก์ชันที่อยู่เบื้องหลัง AI ทุกตัวในปัจจุบัน. ทำไม ReLU ถึงดีกว่า Sigmoid?

เพราะ ReLU คำนวณเร็วมาก — แค่เปรียบเทียบกับ 0. และสำคัญกว่านั้น gradient ของ ReLU ในฝั่งบวกคือ 1 เสมอ ไม่เคยหายไปเหมือน sigmoid ทำให้ network ที่ลึกมาก ๆ ยังคง train ได้.

ในปี 2026 LLM ส่วนใหญ่ใช้ ReLU รุ่นปรับปรุงที่เรียกว่า **GELU** (Gaussian Error Linear Unit) หรือ **SwiGLU** ซึ่งนุ่มนวลกว่าเล็กน้อยและทำให้ training stable ขึ้นในโมเดลขนาดใหญ่. แต่หลักการเดียวกัน — เพิ่มความไม่เป็นเส้นตรงเพื่อให้ network เรียนรู้ pattern ซับซ้อนได้.

มีทฤษฎีบทสำคัญที่เรียกว่า **Universal Approximation Theorem** บอกว่า: neural network ที่มี hidden layer หนึ่งชั้นพร้อม activation function ที่เหมาะสม และมี neuron มากพอ สามารถแทนฟังก์ชันต่อเนื่องใด ๆ ก็ได้. ในทางทฤษฎีหมายความว่า neural network สามารถเรียนรู้ได้ "ทุกอย่าง" ที่เป็น pattern ต่อเนื่อง — แต่ทฤษฎีบทนี้บอกแค่ว่า "มีอยู่" ไม่ได้บอกว่า training จะหาค่านั้นเจอในทางปฏิบัติ. ความต่างระหว่างทฤษฎีและความเป็นจริงของ neural network คือสิ่งที่ทำให้งานวิจัยด้านนี้ยังเดินหน้าต่อไม่หยุด.

> 💡 **Key insight:** ไม่มี activation function = ไม่ว่าจะมีกี่ layer ก็เทียบเท่า layer เดียว. มี activation function = network สามารถเรียนรู้ pattern ซับซ้อนได้ตามทฤษฎีแทบไม่จำกัด. ความต่างนี้ไม่ใช่เชิงปริมาณ แต่เชิงคุณภาพสิ้นเชิง.

## จาก neuron หนึ่งตัวสู่ network — ชั้นและความลึก

neuron เดี่ยวทำได้น้อยมาก. พลังเริ่มเกิดขึ้นเมื่อเอา neuron มาจัดเป็นโครงสร้าง.

ลองนึกถึงโรงงานผลิตสินค้า ที่มีสายพานหลายสาย เรียงกันเป็นสถานี. สถานีที่ 1 รับวัตถุดิบแล้วส่งต่อ สถานีที่ 2 รับสิ่งที่สถานี 1 ส่งมาแล้วแปลงต่อ และต่อไปเรื่อย ๆ จนถึงสถานีสุดท้ายที่ได้ผลิตภัณฑ์สำเร็จรูป. ไม่มีสถานีไหนที่ทำทุกอย่างคนเดียว — ความซับซ้อนเกิดจากการทำงานร่วมกัน.

สิ่งที่ต้องเน้นคือทิศทางของข้อมูลนั้นไหลทางเดียวเสมอ — จาก input layer → hidden layers → output layer. ไม่มีการไหลย้อนกลับตอนใช้งาน. (มีเฉพาะตอน training ที่ gradient ไหลย้อนกลับ แต่นั่นเป็นคนละขั้นตอน.)

สิ่งที่ทำให้ neural network น่าทึ่งกว่าโรงงานคือ: โรงงานมีคนออกแบบว่าแต่ละสถานีต้องทำอะไร. แต่ neural network ไม่มีใครออกแบบ "สูตร" ให้แต่ละ layer — มันค้นพบเองว่าจะทำอะไรจากการดูตัวอย่างหลายล้านชิ้น. สิ่งที่มนุษย์ออกแบบคือโครงสร้าง (กี่ layer กี่ neuron) ส่วนเนื้อหาใน weight นั้นเกิดขึ้นเองจาก training.

Neural network มีโครงสร้างแบบเดียวกัน:

**Layer** คือกลุ่มของ neuron ที่ทำงานพร้อมกันในระดับเดียวกัน แต่ละ neuron ในชั้นเดียวกันรับ input ชุดเดียวกัน แต่มี weight ของตัวเอง. ผลรวมของ layer คือชุดตัวเลขชุดใหม่ที่มีขนาดเท่ากับจำนวน neuron ในชั้นนั้น.

**Network** คือลำดับของ layer ต่อกัน output ของ layer หนึ่งกลายเป็น input ของ layer ถัดไป. ข้อมูลไหลจากซ้ายไปขวาตลอดเวลา — ไม่มีการย้อนกลับตอนใช้งาน.

โครงสร้างพื้นฐานมีสามส่วน:

```
Input layer  →  Hidden layer (หลาย ๆ ชั้น)  →  Output layer
(รับข้อมูลดิบ)    (แปลงข้อมูลไปเรื่อย ๆ)        (ให้ผลลัพธ์)
```

**Input layer** รับข้อมูลดิบเข้ามา เช่น ค่าพิกเซลของรูปภาพ หรือ embedding ของคำ. ชั้นนี้ไม่ทำการคำนวณอะไร แค่รับข้อมูลและส่งต่อ.

**Hidden layer** คือส่วนที่ทำงานจริง แต่ละชั้นรับตัวเลขจากชั้นก่อนหน้า คูณกับ weight บวก bias ผ่าน activation ส่งต่อชั้นถัดไป. ชื่อ "hidden" มาจากที่มันซ่อนอยู่ระหว่าง input กับ output — ไม่มีใครเห็นตรง ๆ ว่ามันทำอะไร ต้องวิเคราะห์หาเอา.

**Output layer** ให้ผลลัพธ์ตามงานที่ต้องการ เช่น probability ว่ารูปนี้คืออะไร หรือ probability distribution ของคำถัดไปใน LLM.

ส่วนที่น่าสนใจที่สุดคือ hidden layer ตรงกลาง. ใน LLM ที่คุณใช้อยู่ทุกวัน มี hidden layer ราว 80-120 ชั้น. ใน GPT-4 ที่คาดว่าใหญ่กว่านั้นอาจมีมากกว่า.

ลองนึกถึงขนาดของงาน: ถ้า network มี 100 layer แต่ละ layer มี neuron 10,000 ตัว และแต่ละ neuron รับ input จาก neuron ทุกตัวใน layer ก่อนหน้า (fully connected) นั่นคือ 10,000 × 10,000 = 100 ล้าน weight แค่ระหว่างสอง layer เท่านั้น. ทั้ง network มี parameter เป็นพันล้านตัวอย่างรวดเร็ว. LLM จริง ๆ มีขนาดใหญ่กว่านี้อีกมาก.

(เปรียบเทียบโรงงานใช้ไม่ได้ตรงไหน: โรงงานจริงมีคนออกแบบแต่ละสถานีว่าต้องทำอะไร. ใน neural network ไม่มีใครออกแบบว่า hidden layer แต่ละชั้นต้องจับ pattern อะไร — network "ค้นพบ" เองตอน training ว่าแบ่งงานกันยังไงถึงจะได้ผลลัพธ์ดีที่สุด.)

## เรขาคณิตของ neuron — มันทำอะไรในแง่รูปทรง

ถ้าคุณชอบคิดเป็นภาพ มีวิธีมองการทำงานของ neuron ที่เข้าใจง่ายมาก.

สมการ `w1×x1 + w2×x2 + ... + wn×xn + b` ในทางเรขาคณิตคือสมการของระนาบ (hyperplane) ในพื้นที่ n มิติ. neuron หนึ่งตัวจึงทำหน้าที่ "ตัด" พื้นที่ของ input ออกเป็นสองฝั่ง — ฝั่งที่ผ่าน activation และฝั่งที่ไม่ผ่าน.

พูดง่าย ๆ คือ neuron ตัวเดียวเป็น "ผู้ตัดสินใจแบบใช่/ไม่ใช่" ที่ขอบเขตเป็นเส้นตรง. มันถามว่า "input ของฉันอยู่ฝั่งไหนของเส้นนี้?"

ปัญหาคือโลกจริงไม่ได้แบ่งด้วยเส้นตรงเสมอไป. ลองนึกถึงการจำแนกรูปภาพว่า "แมว" หรือ "ไม่ใช่แมว" — boundary ระหว่างสองอย่างนี้ในพื้นที่พิกเซลซับซ้อนมาก ไม่ใช่เส้นตรง. หรือลองนึกถึงการตัดสินว่าประโยคนี้เป็น "คำถาม" หรือ "ข้อความ" — มีหลายแบบที่ไม่ชัดเจนในทันที.

นั่นคือเหตุผลที่ต้องมีหลาย layer และ non-linear activation function. แต่ละชั้น "บิด" และ "พับ" พื้นที่ของข้อมูลนิดหน่อย จนในที่สุด pattern ที่ซับซ้อนในพื้นที่ original กลายเป็น pattern ที่ง่ายขึ้นในพื้นที่ใหม่ ที่ layer สุดท้ายสามารถตัดสินใจได้.

นี่คือสาเหตุที่นักวิจัยด้าน interpretability มักพูดถึง "representation learning" — neural network ไม่แค่เรียนรู้คำตอบ แต่เรียนรู้ **วิธีมองข้อมูล** ใหม่ให้คำตอบกลายเป็นเรื่องง่าย.

## neural network เกิดขึ้นมาได้ยังไง — เส้นทาง 80 ปี

ถ้าคิดว่า neural network เป็นเรื่องใหม่ที่เพิ่งเกิดขึ้นในยุค ChatGPT คุณอาจแปลกใจ. แนวคิดนี้เก่ากว่าที่คิดมาก.

**ปี 1943** — McCulloch และ Pitts เสนอแบบจำลองทางคณิตศาสตร์ของเซลล์ประสาทเทียมตัวแรก. ยังไม่มีคอมพิวเตอร์ใน sense ที่เราเข้าใจกัน แต่แนวคิดว่า "เซลล์ประสาทคำนวณได้ด้วยตรรกะ" ถูกวางไว้แล้ว. งานนี้เป็น paper ที่ถูกอ้างอิงมากที่สุดชิ้นหนึ่งในประวัติศาสตร์วิทยาการคอมพิวเตอร์.

**ปี 1958** — Frank Rosenblatt เสนอ "Perceptron" — neural network ตัวแรกที่เรียนรู้ได้จริง. มันรู้จำตัวอักษรและรูปทรงง่าย ๆ ได้. สื่อในยุคนั้นตื่นเต้นมาก บางหัวข้อเขียนว่า "เครื่องจักรที่จะเรียนรู้ได้เหมือนมนุษย์."

**ปี 1969** — Minsky และ Papert พิสูจน์ว่า perceptron มีขีดจำกัด (เรื่อง XOR ที่พูดไปข้างต้น). ทุนวิจัยหายไปเกือบหมด. ช่วงนี้เรียกว่า "AI winter."

**ปี 1986** — Rumelhart, Hinton, และ Williams ทำให้ backpropagation เป็นที่รู้จักกว้างขวาง. นี่คืออัลกอริทึมที่ทำให้ neural network เรียนรู้ได้จริงจาก data โดยการคำนวณ gradient ย้อนกลับ. ประตูเปิดอีกครั้ง. Hinton ได้รับ Nobel Prize สาขาฟิสิกส์ปี 2024 จากงานนี้ร่วมกับ Hopfield.

**ปี 2012** — AlexNet ของ Krizhevsky, Sutskever, และ Hinton ชนะการแข่งขัน ImageNet ด้วยอัตราผิดพลาดที่ต่ำกว่าคู่แข่งถัดมาอย่างถล่มทลาย (error 15.3% เทียบกับ 26.2% ของอันดับสอง). ทั้งวงการตื่น. ยุค deep learning เริ่มต้น. ทุนวิจัย AI หลั่งไหลกลับมา.

**ปี 2017** — paper "Attention Is All You Need" ของทีม Google Brain เสนอ Transformer architecture.

**ปี 2017** — Transformer ถูกเสนอใน paper "Attention Is All You Need" ของทีม Google Brain. architecture นี้เป็นรากฐานของ GPT, Claude, Gemini ทุกตัวในปัจจุบัน. ชื่อ paper แสดงให้เห็นว่า attention mechanism เป็นสิ่งที่ทำให้มันต่างออกไปจาก network ก่อนหน้า.

**ปี 2022-ปัจจุบัน** — ChatGPT เปิดตัวในพฤศจิกายน 2022. ใน 5 วันมีผู้ใช้ 1 ล้านคน ใน 2 เดือนมี 100 ล้านคน. AI กลายเป็นเรื่องของทุกคน ไม่ใช่แค่นักวิจัย.

บทเรียนจากประวัติศาสตร์: แนวคิดพื้นฐานมีมา 80 ปี แต่ต้องรอ GPU ราคาถูก ข้อมูลจากอินเทอร์เน็ต และเทคนิค training ที่ดีขึ้น จึงกลายเป็นสิ่งที่เห็นในปัจจุบัน. ไม่มีการ "ค้นพบ" ครั้งเดียวที่เปลี่ยนทุกอย่าง — แต่เป็นการสะสมของหลายทศวรรษ.

สิ่งที่เปลี่ยนหลังปี 2012 มีสามอย่าง: ข้อมูลจาก internet ที่ใหญ่มาก, GPU ที่ทำ matrix multiplication ได้เร็ว และเทคนิค training ที่ทำให้ network ลึก ๆ stable มากพอที่จะ train ได้. ถ้าขาดอย่างใดอย่างหนึ่ง AI ก็จะยังอยู่ในห้องแล็บ.

> 💡 **Key insight:** เหตุผลที่ AI ก้าวกระโดดอย่างรวดเร็วในช่วง 10 ปีที่ผ่านมาไม่ใช่เพราะแนวคิดใหม่ — แต่เพราะ scale: ข้อมูลมากขึ้น, compute เร็วขึ้น, ราคาถูกลง. neural network เป็นเทคนิคเดิมที่ทำงานได้ดีกว่าอย่างมหาศาลเมื่อ scale ใหญ่พอ.

## ดูการทำงานจริง — ตัวเลขไหลผ่าน network อย่างไร

ลองดูตัวอย่างที่จับต้องได้ เพื่อให้เห็นว่าสิ่งที่ฟังดูเป็นนามธรรมนั้นเกิดขึ้นจริงอย่างไร.

สมมติมี neuron ตัวหนึ่งในระบบรู้จำภาพ รับ input 3 ค่าจาก layer ก่อนหน้า:

- weight ของ neuron นี้: w1 = 0.5, w2 = -1.2, w3 = 0.8
- bias: b = 0.3
- input ที่ได้รับในรอบนี้: x1 = 2, x2 = 1, x3 = -1

**ขั้นที่ 1 — คำนวณผลรวมถ่วงน้ำหนัก:**

```
z = (0.5 × 2) + (-1.2 × 1) + (0.8 × -1) + 0.3
z = 1.0 - 1.2 - 0.8 + 0.3
z = -0.7
```

**ขั้นที่ 2 — ผ่าน ReLU:**

```
output = ReLU(-0.7) = max(0, -0.7) = 0
```

neuron นี้ส่ง 0 ออกไปให้ layer ถัดไป — มันถูก "ดับ" ในรอบนี้. เหมือนปุ่มที่ถูกกด "off" เพราะ input ที่รับมาไม่ผ่าน threshold.

แต่ถ้า input ชุดอื่นทำให้ z เป็นบวก เช่น x1 = 3, x2 = 0.5, x3 = 1 คำนวณใหม่ได้:

```
z = (0.5 × 3) + (-1.2 × 0.5) + (0.8 × 1) + 0.3
z = 1.5 - 0.6 + 0.8 + 0.3
z = 2.0
output = ReLU(2.0) = 2.0
```

ตอนนี้ neuron ส่งค่า 2.0 ออกไปให้ layer ถัดไป.

ความแตกต่างระหว่างสองรอบนี้คือทั้งหมด — neuron บางตัวจะ "ติด" กับบาง input และ "ดับ" กับ input อื่น และรูปแบบว่า neuron ตัวไหนติดตัวไหนดับนั้นคือสิ่งที่ encode ความรู้ของโมเดลเอาไว้.

ลองนึกภาพ network ที่มี neuron 1,000 ตัวใน layer เดียว. บาง input อาจทำให้ neuron 400 ตัวติดและ 600 ตัวดับ. Input อื่นอาจทำให้ neuron 300 ตัวติดและ 700 ตัวดับ. แต่ละ "ลาย" ของ neuron ที่ติดและดับเป็นการแทนความหมายของ input นั้น ๆ. ทุก layer ต่อกันทำให้ลายเหล่านั้นถูก transform และรวมกันจนได้ output ที่ต้องการ.

กระบวนการที่ข้อมูลไหลจาก input ผ่านทุก layer ออกมาเป็น output โดยไม่มีการปรับ weight เรียกว่า **forward pass** — เกิดขึ้นทุกครั้งที่คุณส่งข้อความหา ChatGPT. เวลาคุณรอคำตอบ 1-2 วินาที forward pass กำลังเกิดขึ้นซ้ำแล้วซ้ำเล่า ทีละ token.

ในเชิง engineering forward pass คือชุดของ matrix multiplication ขนาดใหญ่ที่ทำซ้ำตาม layer จำนวนมาก. GPU เก่งเรื่องนี้มากเพราะทำ parallel computation ได้พร้อมกันหลายพันหน่วย — เป็นเหตุผลที่ training และ inference ของ neural network ใช้ GPU ไม่ใช่ CPU.

สำหรับ LLM การ forward pass หนึ่งครั้งให้ probability ของ token ถัดไปหนึ่งตัว. ถ้าคำตอบที่ต้องการยาว 200 คำ (ราว 300 token ในภาษาไทย) หมายความว่า forward pass ต้องเกิดขึ้น 300 ครั้ง — แต่ละครั้งผ่าน 100 layer ของ transformer. นั่นคือ 30,000 ครั้งของการส่งตัวเลขผ่านชั้น parameter หลายพันล้านตัว เพื่อตอบคำถามหนึ่งข้อ. ตัวเลขนี้คือเหตุผลที่ค่าใช้จ่ายในการรัน LLM ขนาดใหญ่ไม่ใช่เรื่องเล็กน้อย.

### ลองทำดู

เปิด ChatGPT หรือ Claude แล้วลองส่ง prompt สั้น ๆ ว่า "ท้องฟ้าสีอะไร" แล้วสังเกตว่าคำตอบออกมาทีละคำ ไม่ได้โผล่ออกมาพร้อมกันทั้งประโยค. ในระหว่างที่คุณรอนั้น forward pass กำลังเกิดขึ้น — ข้อความของคุณถูกแปลงเป็นตัวเลข ไหลผ่าน layer ราว 100 ชั้น แต่ละชั้นทำ matrix multiplication นับล้านครั้ง แล้วออกมาเป็น probability ของคำแรกในคำตอบ. จากนั้นวนซ้ำทีละคำจนประโยคเสร็จ. ทั้งหมดเกิดในเวลาไม่ถึงวินาทีต่อคำ.

## training คือการปรับตัวเลขพันล้านตัวพร้อมกัน

ถึงตอนนี้คุณเข้าใจแล้วว่า network ทำงานอย่างไรตอน inference (ตอบคำถาม). แต่มันเรียนรู้มาจากไหน?

คำตอบคือจากกระบวนการที่เรียกว่า **training** — การปรับค่า weight และ bias ทุกตัวในทุก neuron ให้ output ของ network ใกล้เคียงกับคำตอบที่ถูกต้องมากขึ้นเรื่อย ๆ.

กลไกของ training เราเจาะลึกไปแล้วในบท [โมเดลเรียนรู้ยังไง](/foundations/how-models-learn). ทบทวนสั้น ๆ: โมเดลเดาคำตอบ (forward pass), วัดว่าผิดแค่ไหน (**loss**), คำนวณย้อนกลับว่าต้องขยับ parameter แต่ละตัวไปทางไหน (**backpropagation**), แล้วขยับทีละนิดในทิศที่ลด loss (**gradient descent**) — วนซ้ำหลายล้านล้านรอบ. สำหรับ neural network จุดสำคัญที่ต้องเข้าใจคือ training นี้ปรับ weight และ bias ของ neuron ทุกตัวพร้อมกัน. สำหรับ LLM ขนาดใหญ่ training อาจใช้เวลาหลายเดือนบน GPU cluster หลายพัน-หมื่นตัว ใช้งบหลักร้อยล้านดอลลาร์.

> 💡 **Key insight:** โครงสร้างของ network (จำนวน layer, จำนวน neuron, ชนิด activation) เป็นสิ่งที่ **มนุษย์ออกแบบ**. แต่ค่าของ weight และ bias ทุกตัวเป็นสิ่งที่ **เรียนรู้จาก data** — ไม่มีใครเขียนว่า "weight ตัวนี้ควรเป็น 0.37" มันค้นพบเองตอน training. โครงสร้างกับ weight เป็นคนละสิ่งที่แยกจากกัน และ training คือสะพานที่เชื่อมโครงสร้างว่างเปล่ากับโมเดลที่ตอบได้.

## Neural network ไม่ใช่สมองเทียม — analogy นี้พังตรงไหน

ชื่อ "neural network" ทำให้หลายคนคิดว่า AI กำลังทำงาน "เหมือนสมองมนุษย์." เป็นความเข้าใจที่แพร่หลายมาก — และผิดพอสมควร.

ความคล้ายกันมีอยู่จริงในระดับ surface: เซลล์ประสาทจริงในสมองก็รับ signal จากหลายทาง รวมสัญญาณ แล้วยิง signal ออกไปถ้าเกิน threshold. artificial neuron ก็รับ input หลายตัว คูณและบวก แล้วส่ง output ออกไป. โครงสร้างกว้าง ๆ คล้ายกัน — และนั่นคือที่มาของชื่อ.

แต่หลังจากนั้นก็แตกต่างกันทุกอย่าง:

เซลล์ประสาทจริงยิงสัญญาณแบบ "on/off" เป็น pulse ในเวลา ไม่ใช่ตัวเลขทศนิยมต่อเนื่อง. เซลล์ประสาทจริงมี dendrite ที่ทำการประมวลผลซับซ้อนในตัวมันเอง ไม่ใช่แค่บวกและคูณ. สมองมนุษย์มีเซลล์ประสาทราว 86,000 ล้านเซลล์ แต่ละเซลล์เชื่อมกับเซลล์อื่นอีกราว 7,000 เส้นทาง — และทำงานด้วยพลังงานราว 20 วัตต์ ขณะที่การรัน LLM ใหญ่ ๆ ใช้พลังงานหลายร้อยกิโลวัตต์.

ที่สำคัญที่สุด: สมองเรียนรู้ผ่านกลไกที่นักวิทยาศาสตร์ยังไม่เข้าใจชัดเจน และแน่นอนว่าไม่ใช่ backpropagation. สมองสามารถเรียนรู้จากตัวอย่างน้อย ๆ ได้ดีมาก ขณะที่ neural network ต้องการข้อมูลมหาศาล.

นักวิจัยรุ่นใหม่หลายคนเลือกเรียกว่า "deep network" หรือเรียกตามสถาปัตยกรรม เช่น "transformer" มากกว่าจะเน้นคำว่า "neural" เพราะไม่อยากให้ภาพลวงตาเรื่องสมองมาครอบงำการทำความเข้าใจ.

> 💡 **Key insight:** neural network เป็น function approximator ทางคณิตศาสตร์ที่เรียนรู้จากข้อมูล — ไม่ใช่แบบจำลองของสมอง ไม่มี consciousness ไม่ได้ "เข้าใจ" อะไร. ทุกพฤติกรรมที่เห็นอธิบายได้ด้วยคณิตศาสตร์เชิงสถิติในที่สุด. ชื่อ "neural" เป็น metaphor จากปี 1943 ที่ยังค้างมาถึงปัจจุบัน.

## ชั้นต่าง ๆ เรียนรู้อะไร — จากง่ายสู่ซับซ้อน

หนึ่งในสิ่งที่น่าทึ่งที่สุดของ neural network คือแต่ละชั้นเรียนรู้ feature ในระดับที่ต่างกัน โดยไม่มีใครบอก.

ลองนึกถึง CNN (Convolutional Neural Network) ที่ใช้รู้จำรูปภาพ นักวิจัยได้ทำการ "intepretability" — เปิดดูว่า neuron แต่ละตัวตอบสนองกับอะไร — และพบว่า:

- **ชั้นแรก ๆ** จับสิ่งง่ายที่สุด: เส้น, ขอบ, และสีพื้นฐาน. neuron บางตัวจะ "ติด" เมื่อเห็นเส้นแนวนอน บางตัวติดเมื่อเห็นเส้นแนวตั้ง บางตัวติดเมื่อเห็นสีแดง.
- **ชั้นกลาง** รวมขอบเหล่านั้นเป็น texture และ pattern เล็ก เช่น ลายเส้น, พื้นผิว, และส่วนของวัตถุอย่างตา หรือหนวด.
- **ชั้นลึก ๆ** รวม pattern เล็กเป็นส่วนของวัตถุที่ใหญ่ขึ้น เช่น หน้าแมว หรือขาแมว.
- **ชั้นสุดท้าย** ตัดสินใจจาก feature ทั้งหมดว่า "นี่คือแมว" ด้วย probability 0.92.

ไม่มีใครเขียนกฎเหล่านี้. network ค้นพบเองว่าการแบ่งงานแบบนี้ทำให้ทายได้ถูกต้องที่สุด. มันเหมือนมีพนักงานใหม่ที่ไม่ได้รับ job description แต่ค้นพบด้วยตัวเองว่าควรทำอะไรจากการดูผลลัพธ์ที่ถูกกับผิด.

ใน LLM ก็เหมือนกัน นักวิจัยพบว่า:

- **ชั้นแรก ๆ** จับ pattern ง่าย ๆ ของภาษา เช่น "คำนี้เป็นคำนาม" หรือ "สองคำนี้มักอยู่ด้วยกัน" หรือ "จุดนี้น่าจะเป็นจุดจบประโยค"
- **ชั้นกลาง ๆ** จับบริบทที่ซับซ้อนขึ้น เช่น "ประโยคนี้กำลังถามคำถาม" หรือ "น้ำเสียงนี้เป็นทางการ" หรือ "บริบทกำลังพูดถึงอาหาร"
- **ชั้นท้าย ๆ** จับความสัมพันธ์ระดับสูง เช่น "จากทั้งหมดที่อ่านมา คำตอบของคำถามนี้น่าจะเป็นชื่อสถานที่" หรือ "tone ของบทสนทนานี้ต้องการคำตอบที่เป็นทางการ"

นี่คือเหตุผลที่เรียกว่า **deep** learning — ความลึกของ layer คือสิ่งที่ทำให้ network เรียนรู้ abstraction ซับซ้อนขึ้นเรื่อย ๆ จากสิ่งที่ง่ายที่สุดไปสู่สิ่งที่ซับซ้อนที่สุด.

ลองนึกถึงตัวอย่างของ LLM กับประโยค "แมวที่นั่งอยู่บนโต๊ะกินปลา":

- **ชั้นแรก:** รู้จัก token แต่ละตัว — "แมว" คือ noun, "กิน" คือ verb, "ปลา" คือ noun
- **ชั้นกลาง:** จับความสัมพันธ์ — "แมว" เป็น subject ของประโยค ไม่ใช่ "โต๊ะ"
- **ชั้นลึก:** เข้าใจว่าถ้าต้องบอกว่าใครกินอะไร — แมวกินปลา ไม่ใช่โต๊ะกินปลา
- **ชั้นสุดท้าย:** ถ้าถูกถามว่า "ใครกิน?" — ตอบ "แมว" ได้ถูกต้อง

การที่ชั้นกลางสามารถจับได้ว่า "แมว" เป็น subject แม้มี "โต๊ะ" คั่นอยู่ในประโยค คือสิ่งที่ทำให้ transformer เหนือกว่า RNN — กลไก attention ทำให้ทุก token "มองเห็น" กันได้ทั้งหมดพร้อมกัน.

## ความเข้าใจผิดที่พบบ่อย — และคำตอบที่ตรง

ก่อนจะไปดู architecture ต่าง ๆ มีความเข้าใจผิดบางอย่างที่พบบ่อยมากจนควรพูดตรง ๆ เลย.

**"ยิ่งมี layer เยอะยิ่งดีเสมอ"** — ไม่ใช่. depth ช่วยได้แน่ แต่ถึงจุดหนึ่งก็ saturate. network ที่ลึกเกินไปมีปัญหา vanishing gradient (gradient ที่ต้องไหลย้อนกลับหลายชั้นเริ่มเล็กจนเกือบหายไป ทำให้ชั้นแรก ๆ ไม่ได้รับสัญญาณ training) training instability และอาจ overfit กับข้อมูล training มากเกินไปจนทำงานกับข้อมูลใหม่ได้แย่. เทคนิคอย่าง residual connection (ที่ ResNet ใช้) และ layer normalization เป็นสิ่งที่ทำให้ network ลึก ๆ train ได้จริงในทางปฏิบัติ.

**"weight เก็บข้อเท็จจริงเป็นชิ้น ๆ"** — ไม่ใช่. ความรู้ถูกกระจายอยู่ทั่วทั้ง network แบบ distributed. ไม่มี neuron ตัวไหนที่ "รู้เรื่องกรุงเทพ" หรือ "รู้เรื่องสูตรทำอาหาร" โดยเฉพาะ. ถ้าลบ neuron 100 ตัวออก คุณจะไม่ทำให้โมเดลลืมข้อเท็จจริงข้อเดียว — แต่จะทำให้คุณภาพโดยรวมลดลงเล็กน้อยทุกเรื่อง.

**"neural network เพิ่งถูกคิดค้น"** — ไม่ใช่. แนวคิด McCulloch-Pitts neuron มีมาตั้งแต่ปี 1943. perceptron เสนอในปี 1958. backpropagation ถูกทำให้เป็นที่นิยมในปี 1986. สิ่งที่เปลี่ยนหลังปี 2012 คือ scale ของข้อมูลและ compute (GPU) บวกกับเทคนิค training ที่ทำให้ network ลึก ๆ train ได้ — ไม่ใช่แนวคิดพื้นฐาน.

**"activation function ที่เอาออกไปแค่ทำให้ network เรียบง่ายลง"** — ไม่ใช่ เป็นความเสียหายเชิงคุณภาพ. ไม่มี activation = network ทั้งหมดกลายเป็นสมการเส้นตรงเดียวไม่ว่าจะมีกี่ layer. ความสามารถลดจาก "แทนฟังก์ชันต่อเนื่องใดก็ได้" เหลือ "แทน linear function ได้เท่านั้น."

## ทำไม XOR ทำลาย neural network ไป 20 ปี

มีเรื่องหนึ่งในประวัติศาสตร์ AI ที่แสดงให้เห็นได้ชัดที่สุดว่า activation function และ hidden layer สำคัญแค่ไหน.

ปี 1969 นักวิจัยสองคนชื่อ Minsky และ Papert เขียนหนังสือพิสูจน์ว่า perceptron (neural network รุ่นดั้งเดิมที่ไม่มี hidden layer และไม่มี non-linear activation) ไม่สามารถแก้ปัญหา XOR ได้.

XOR คือฟังก์ชันง่าย ๆ ที่บอกว่า input สองตัว "ต่างกัน" หรือ "เหมือนกัน":

| input A | input B | output XOR |
|---------|---------|-----------|
| 0 | 0 | 0 (เหมือนกัน) |
| 0 | 1 | 1 (ต่างกัน) |
| 1 | 0 | 1 (ต่างกัน) |
| 1 | 1 | 0 (เหมือนกัน) |

ลองวาด 4 จุดนี้บนกระดาษ — ไม่มีเส้นตรงเส้นเดียวที่แบ่ง "0" ออกจาก "1" ได้. network ที่ทำแค่สมการเส้นตรง (ไม่มี non-linear activation) จึงแก้ไม่ได้ เพราะมันทำได้แค่ "ตัด" space ด้วยเส้นตรง.

การพิสูจน์นี้ทำให้แหล่งทุนวิจัยหายไปเกือบหมด และวงการ neural network "จำศีล" ไปยี่สิบปีเต็ม — ช่วงที่เรียกว่า "AI winter."

แต่สิ่งที่ Minsky และ Papert พิสูจน์คือ perceptron **ที่ไม่มี hidden layer และ activation function** ทำไม่ได้. เมื่อเพิ่ม hidden layer พร้อม non-linear activation function เข้าไป network สามารถ "บิด" space ของ input ให้ XOR กลายเป็นปัญหาที่แก้ได้ง่าย ๆ.

นึกภาพแบบนี้: ถ้า 4 จุดของ XOR อยู่บนกระดาษแผ่นเดียว คุณแบ่งด้วยเส้นตรงไม่ได้. แต่ถ้าคุณยก "บิด" กระดาษขึ้นมาในสามมิติ สองจุดของ class เดียวกันอาจจะอยู่ใกล้กัน และคุณแบ่งได้ด้วยระนาบ. hidden layer ทำหน้าที่นั้น — "บิด" space ของข้อมูลให้ปัญหาที่ซับซ้อนกลายเป็นง่าย.

เรื่อง XOR ยังมีบทเรียนอีกอย่างที่ลึกกว่า: ปัญหาหลาย ๆ อย่างในโลกจริงที่ดูยากในมิติปกติ จะกลายเป็นง่ายมากในมิติที่ต่างออกไป. neural network เรียนรู้ "มิติใหม่" ที่เหมาะสำหรับแต่ละปัญหาเองผ่าน training นี่คือสาระสำคัญของ representation learning.

กรณี XOR ยังสำคัญเชิงประวัติศาสตร์อีกมิติหนึ่ง: มันแสดงให้เห็นว่า "พิสูจน์ว่า A ทำไม่ได้" ไม่ใช่เรื่องเดียวกับ "พิสูจน์ว่า B ทำไม่ได้" ถ้า A และ B ต่างกัน. การที่ perceptron แบบเก่าทำ XOR ไม่ได้ ไม่ได้หมายความว่า neural network ทุกแบบทำไม่ได้ — แต่เราใช้เวลา 20 ปีกว่าจะค้นพบสิ่งนั้น.

บทเรียนนี้สำคัญมาก: **non-linearity + depth คือสิ่งที่ปลดล็อกพลังของ neural network**.

### ลองทำดู

เปิด ChatGPT แล้วถามว่า "จงบอก output ของ XOR gate สำหรับ input (0,1) และ (1,1)" — มันตอบได้ถูกต้อง: (0,1) ได้ 1 และ (1,1) ได้ 0. คนที่อยู่เบื้องหลังคำตอบนั้นคือ transformer ที่มี hidden layer หลายสิบชั้นพร้อม non-linear activation — ถ้าไม่มีสิ่งเหล่านั้น มันตอบไม่ได้เลย. ลองต่อด้วยคำถามว่า "ทำไม neural network ถึงแก้ XOR ได้ถึงแม้ perceptron เดี่ยวทำไม่ได้?" แล้วดูว่ามันอธิบายอะไร.

## Architecture หลักที่ควรรู้จัก

neural network ไม่ได้มีแบบเดียว. นักวิจัยออกแบบ architecture ต่าง ๆ ที่เหมาะกับงานต่าง ๆ และชนิดข้อมูลต่าง ๆ.

**MLP (Multi-Layer Perceptron)** คือ architecture พื้นฐานที่สุด ทุก neuron ใน layer หนึ่งเชื่อมกับทุก neuron ใน layer ถัดไป เรียกว่า "fully connected" หรือ "dense." ใช้ดีกับข้อมูลตาราง เช่น ราคาบ้าน, อายุลูกค้า, ยอดขายรายเดือน. ข้อเสียของ MLP คือถ้าใช้กับรูปภาพจะมี parameter มหาศาลเพราะต้องเชื่อมทุกพิกเซลกับทุก neuron ในชั้นถัดไป — จึงเกิด architecture เฉพาะทางที่ดีกว่าสำหรับแต่ละชนิดข้อมูล.

**CNN (Convolutional Neural Network)** ออกแบบมาสำหรับรูปภาพโดยเฉพาะ โดยอาศัยหลักการที่ว่าพิกเซลที่อยู่ใกล้กันมีความสัมพันธ์กัน. แทนที่จะเชื่อมทุก neuron กับทุก neuron มันใช้ "filter" ขนาดเล็ก (เช่น 3×3 พิกเซล) ที่เลื่อนผ่านรูปภาพทีละส่วน. filter เดียวกันใช้ซ้ำกับทุกส่วนของรูป ทำให้ประหยัด parameter มากและจับ feature เชิงพื้นที่ได้ดี. AlexNet ปี 2012 ที่ทำให้ deep learning โด่งดังคือ CNN.

**RNN (Recurrent Neural Network)** มี "ความจำ" โดยส่ง output ของ step ก่อนหน้ากลับเข้ามาเป็น input ของ step ถัดไป เหมาะกับข้อมูลที่เป็นลำดับ เช่น ข้อความ หรือ time series. เคยเป็น architecture หลักของงานภาษาก่อนปี 2017 แต่มีปัญหาสำคัญ: มันประมวลผลทีละคำเป็นลำดับ ทำให้รอได้ยาก และ "ความจำ" ของมันเลือนหายตามระยะทาง — คำที่ห่างออกไปมากในประโยคจะถูก "ลืม" ไปบ้าง.

**Transformer** คือ architecture ที่เปลี่ยนวงการ AI ในปี 2017 และเป็น architecture ที่ LLM ทุกตัวในปัจจุบันใช้ รวมถึง GPT, Claude, Gemini, และ Llama. แทนที่จะประมวลผลข้อความทีละคำแบบ RNN มันใช้กลไก **attention** ที่ทำให้ทุก token "มองเห็น" ทุก token อื่น ๆ พร้อมกันในครั้งเดียว. ทำให้เข้าใจความสัมพันธ์ระหว่างคำที่อยู่ห่างกันมากได้ดีกว่า RNN อย่างมาก และทำ parallel computation ได้ — เร็วกว่า RNN อย่างมหาศาล.

ในปี 2026 transformer ยังครองตลาด แต่มี architecture ใหม่อย่าง Mamba (state space model) กำลังถูก track ว่าอาจประหยัด memory กว่าสำหรับบาง use case. landscape ยังเปลี่ยนอยู่.

> 💡 **Key insight:** ChatGPT, Claude, Gemini ที่คุณใช้ทุกวัน ล้วนเป็น transformer ทั้งนั้น. ความแตกต่างระหว่างโมเดลเหล่านี้อยู่ที่ขนาด วิธี training และข้อมูลที่ใช้ — ไม่ใช่ architecture พื้นฐาน ซึ่งเหมือนกันทุกตัว.

## ขนาดของ neural network — ตัวเลขที่ทำให้ตาค้าง

จำนวน parameter (weight + bias รวมกัน) คือตัววัด "ขนาด" ของโมเดล ยิ่งใหญ่ยิ่งจับ pattern ละเอียดได้มากขึ้น:

| โมเดล | จำนวน parameter |
|-------|-----------------|
| network ฝึกหัดในห้องเรียน | 1,000–10,000 |
| ResNet-50 (รู้จำรูปภาพ) | 25 ล้าน |
| BERT-base (งานภาษา) | 110 ล้าน |
| GPT-3 (2020) | 175,000 ล้าน |
| GPT-4 / Claude รุ่นใหม่ | คาดว่าระดับ trillion |

ตัวเลข 175,000 ล้านหมายความว่าอะไรในชีวิตจริง? ถ้าคุณปรับ parameter 1 ตัวต่อวินาที ไม่หยุดพักเลย คุณจะปรับครบทุกตัวใน GPT-3 ใน 5,500 ปี.

scale ขนาดนี้คือเหตุผลที่การ train โมเดลใหม่ตั้งแต่ต้นใช้เงินราว $100 ล้านดอลลาร์ขึ้นไปต่อรอบ ใช้ไฟฟ้าเท่าเมืองขนาดกลาง และต้องการ GPU หลายพันตัวทำงานพร้อมกันเป็นเดือน. เป็นเหตุผลที่บริษัทที่มี frontier model อยู่แค่ไม่กี่บริษัทในโลก ได้แก่ OpenAI, Anthropic, Google, Meta, และ xAI.

แต่ scale ไม่ใช่ทุกอย่าง. โมเดลเล็กที่ฝึกกับข้อมูลดีและ fine-tune สำหรับงานเฉพาะ อาจทำงานเฉพาะอย่างได้ดีกว่าโมเดลใหญ่ที่ train แบบ general. ในปี 2026 มีโมเดลเล็กหลายตัว (Llama 3.1 8B, Mistral 7B, Phi-3) ที่ทำงานได้น่าทึ่งในคอมพิวเตอร์ทั่วไปหรือแม้แต่มือถือ. landscape กำลังเปลี่ยนจาก "ใหญ่เสมอดีกว่า" เป็น "พอดีกับงานดีกว่า."

โมเดลล่าสุดหลายตัวใช้เทคนิคที่เรียกว่า **mixture-of-experts (MoE)** ซึ่งทำให้ตัวเลข parameter รวมสูงมาก แต่ในการตอบแต่ละครั้งจะเปิดใช้แค่ส่วนหนึ่งของ parameter ทั้งหมด — เหมือนโรงพยาบาลที่มีแพทย์หลายสาขา แต่ผู้ป่วยแต่ละคนพบแค่แพทย์ที่เกี่ยวข้องกับอาการของตัวเอง. ทำให้ประหยัดพลังงานในการใช้งานจริงโดยไม่เสียความสามารถ.

## ทำไมเรื่องนี้สำคัญกับการใช้ AI ในชีวิตประจำวัน

อาจมีคำถามในหัวว่า "รู้เรื่อง neuron กับ weight แล้วได้ประโยชน์อะไรตอนใช้ ChatGPT จริง ๆ?"

คำตอบคือมันเปลี่ยนวิธีคิดเรื่อง AI โดยตรง.

**เรื่องที่ 1 — ทำไม AI ถึง hallucinate (ตอบผิดอย่างมั่นใจ)**

neural network ไม่ "รู้" ข้อเท็จจริง มันเดา pattern ที่น่าจะถูกต้องจาก training data. เมื่อถามเรื่องที่ไม่มีในข้อมูล training หรือถามรายละเอียดเล็ก ๆ ที่ไม่บ่อยพอ network ก็เดา pattern ที่ "ดูเหมือน" คำตอบ แต่อาจผิด. นี่ไม่ใช่ bug — นี่คือพฤติกรรมพื้นฐานของ function approximator ที่เรียนรู้จาก data.

**เรื่องที่ 2 — ทำไม prompt ที่ดีถึงให้คำตอบที่ดีกว่า**

เมื่อคุณพิมพ์ prompt เข้าไปใน LLM input นั้นกลายเป็นชุดตัวเลขที่ไหลผ่าน hidden layer ทุกชั้น. การ prompt ที่ชัดเจนและมีบริบทมากขึ้นทำให้ตัวเลขเหล่านั้น "นำพา" network ไปทาง pattern ที่ถูกต้องกว่า. ไม่ใช่เรื่องของการ "บอกให้ AI เข้าใจ" — แต่เป็นเรื่องของการให้ input ที่ทำให้ forward pass ไปในทิศที่ดี.

**เรื่องที่ 3 — ทำไมโมเดลต่างขนาดทำงานต่างกัน**

โมเดลเล็ก (เช่น Claude Haiku หรือ GPT-4o mini) มี parameter น้อยกว่า ตอบเร็วกว่า ถูกกว่า แต่จับ pattern ละเอียดได้น้อยกว่า. โมเดลใหญ่ (เช่น Claude Opus หรือ GPT-4) มี parameter มากกว่า ตอบช้ากว่า แพงกว่า แต่จับ pattern ซับซ้อนกว่าได้ดีกว่า. เลือกใช้ตามงาน — ไม่ใช่ใช้ใหญ่เสมอ.

**เรื่องที่ 4 — ทำไมโมเดลเดียวกันตอบต่างกันในแต่ละครั้ง**

ตอนที่ LLM เลือก token ถัดไป มันไม่ได้เลือกตัวที่ probability สูงสุดเสมอ. มันมี parameter ที่เรียกว่า "temperature" ที่ควบคุมว่าจะสุ่มมากแค่ไหน. temperature สูง = สุ่มมาก ตอบหลากหลาย สร้างสรรค์. temperature ต่ำ = เลือกตัวที่น่าจะเป็นมากที่สุด ตอบคาดเดาได้. นี่คือเหตุผลที่คำถามเดิมถามสองครั้งอาจได้คำตอบต่างกัน.

> 💡 **Key insight:** neural network เรียนรู้ว่าจะให้ output อะไรโดยอาศัย pattern ในข้อมูล training ทั้งหมด ไม่ได้ "ดึงข้อเท็จจริงออกมาจากฐานความรู้." เมื่อคุณเข้าใจว่ามันทำงานยังไง คุณก็เข้าใจทั้งความสามารถและข้อจำกัดของมันได้พร้อมกัน.

### ลองทำดู

ลองทดสอบขีดจำกัดของ pattern matching ใน ChatGPT โดยถามคำถามสองแบบ:

แบบที่ 1 — ถามเรื่องที่มีในข้อมูลบ่อย ๆ: "กรุงเทพเป็นเมืองหลวงของประเทศอะไร?"

แบบที่ 2 — ถามเรื่องที่เฉพาะมาก ๆ และอาจไม่มีในข้อมูล: "จำนวนพนักงานที่ทำงานในอาคาร X ในเดือนมีนาคม ปีที่แล้ว?"

สังเกตความแตกต่าง — แบบแรกมันตอบทันทีและถูกต้อง. แบบที่สองมันอาจเดาหรือบอกว่าไม่รู้ หรืออาจให้ตัวเลขที่ฟังดูสมเหตุสมผลแต่ไม่มีที่มา. ทั้งหมดนี้เป็นพฤติกรรมของ function approximator ที่เรียนรู้จาก pattern ไม่ใช่จากฐานข้อมูลที่ถูกต้องเสมอ.

## สิ่งที่ neural network เป็น และไม่เป็น

ก่อนจบบทนี้ ควรพูดตรง ๆ เรื่องที่มักเข้าใจผิดกัน เพราะ hype รอบ ๆ AI มักทำให้ภาพไม่ชัด.

**neural network เป็น:**

- **function approximator** ที่เรียนรู้จากข้อมูล ทฤษฎีบอกว่า network ที่ใหญ่พอสามารถแทนฟังก์ชันต่อเนื่องใดก็ได้ (universal approximation theorem) — แต่ "สามารถแทนได้" กับ "เรียนรู้ได้จริงในทางปฏิบัติ" เป็นคนละเรื่อง
- **เครื่องมือที่จับ pattern** ในข้อมูลที่มีโครงสร้างได้ดีมาก เช่น ภาพ ข้อความ และเสียง
- **ระบบวิศวกรรมที่ทำงานได้จริง** ในทางปฏิบัติ แม้ทฤษฎีจะยังอธิบายไม่ครบทุกพฤติกรรม — นักวิจัยยังงงกับหลายสิ่งที่โมเดลทำได้
- **ระบบที่ประสิทธิภาพขึ้นกับข้อมูล training** — โมเดลที่ train กับข้อมูลไทยน้อยก็จะตอบคำถามไทยได้แย่กว่า ไม่ว่า architecture จะดีแค่ไหน

**neural network ไม่ใช่:**

- **แบบจำลองของสมองมนุษย์** (ดูส่วนก่อนหน้า)
- **เครื่องที่ "เข้าใจ" ความหมาย** หรือมี consciousness หรือ intention
- **reasoning engine แบบ symbolic** ที่ทำตาม logic step-by-step แบบที่นักตรรกศาสตร์ทำ — แม้พฤติกรรมภายนอกจะดูคล้ายกันในบางกรณี
- **magic** — ทุกสิ่งที่มันทำอธิบายได้ด้วยคณิตศาสตร์เชิงสถิติในที่สุด แม้ explanation จะยาวและซับซ้อน

นักวิชาการบางคนเรียก neural network ว่า "fancy curve-fitter" — เป็นการอธิบายที่ตรงและมีประโยชน์ เพราะช่วยป้องกันไม่ให้เราคาดหวังเกินจริง. มันเก่งมากในการหา pattern ในข้อมูล — แต่นั่นไม่ใช่เรื่องเดียวกับการเข้าใจ หรือการให้เหตุผล หรือการรู้ว่าตัวเองทำอะไรอยู่.

นักวิจัยที่ทำงานด้าน **interpretability** กำลังพยายามอ่านว่า weight ในโมเดลหมายความว่าอะไร — ค้นหาว่า "ความรู้" ถูก encode อย่างไรในตัวเลขพันล้านตัว. ในปี 2026 เริ่มมีความเข้าใจบ้างแล้ว เช่น การพบว่า neuron บางกลุ่มตอบสนองกับ concept เฉพาะ แต่ยังห่างไกลจากการอ่านโมเดลได้ทั้งหมด.

ข้อเท็จจริงที่น่าคิดคือ: เราสร้าง neural network ขึ้นมา เราออกแบบ architecture เราเขียน code training แต่ค่า weight เฉพาะที่ทำให้มันทำงานได้ดีนั้น ไม่มีใครเขียนหรือออกแบบ. มันเกิดขึ้นจากกระบวนการทางคณิตศาสตร์ที่ทำซ้ำหลายพันล้านรอบ. ในแง่หนึ่ง เราสร้างระบบที่ "สอนตัวเอง" มากกว่าระบบที่เราสอน.

> 💡 **Key insight:** ความรู้ของ neural network ถูกกระจายอยู่ในทุก weight ทุกตัว ไม่มี neuron ตัวไหน "เก็บความรู้เรื่องประเทศไทย" ไว้โดยเฉพาะ. ถ้าลบ neuron หนึ่งตัวออก คุณจะไม่ทำให้โมเดลลืมข้อเท็จจริงข้อใดข้อหนึ่ง — แต่จะค่อย ๆ ทำให้คุณภาพโดยรวมลดลงเล็กน้อย. นี่คือจุดเริ่มต้นของสาขาวิจัยที่เรียกว่า interpretability ที่พยายามอ่านว่า network "รู้" อะไรอยู่ตรงไหน.

## สรุป — กลไกที่ควรจำ

neural network คือโครงสร้างคณิตศาสตร์ที่ประกอบจาก neuron จำนวนมาก แต่ละ neuron คูณ input ด้วย weight บวก bias แล้วผ่าน activation function ส่งต่อให้ layer ถัดไป.

สิ่งที่ทำให้ neural network ทรงพลังมีสามอย่าง:

**หนึ่ง — depth.** การซ้อน layer หลาย ๆ ชั้นทำให้ network เรียนรู้ abstraction ที่ซับซ้อนขึ้นเรื่อย ๆ จากสิ่งง่ายในชั้นแรก ๆ ไปสู่ pattern ซับซ้อนในชั้นลึก. ไม่มีชั้นไหนที่ "รู้" ทุกอย่าง — ความรู้เกิดจากการทำงานร่วมกันของทุกชั้น.

**สอง — non-linearity.** activation function (เช่น ReLU) คือสิ่งที่ทำให้ network ไม่ถูกจำกัดอยู่แค่สมการเส้นตรง. ถ้าไม่มี activation function ไม่ว่าจะลึกแค่ไหนก็เทียบเท่า layer เดียว. มี activation function — network สามารถแทน pattern ในโลกจริงได้ตามทฤษฎีแทบไม่จำกัด.

**สาม — scale.** parameter หลายร้อยล้านถึงหลายล้านล้านตัวที่ถูกปรับแต่งจากข้อมูลมหาศาล ทำให้ network จับ pattern ละเอียดได้มากขึ้นเรื่อย ๆ. scale นี้คือสิ่งที่เปลี่ยน network จาก "เล่น ๆ ในห้องแล็บ" เป็น "ระบบที่เปลี่ยนโลก."

ทุกสิ่งที่คุณเห็น AI ทำในปัจจุบัน ตั้งแต่ตอบคำถาม สรุปเอกสาร แปลภาษา รู้จำรูปภาพ ล้วนเกิดจากกลไกสามอย่างนี้ทำงานร่วมกันในสเกลที่ใหญ่มาก. ไม่มีอะไรพิเศษหรือลึกลับกว่านั้น.

สิ่งที่ยังเป็น open question แม้แต่ในวงการวิจัยเอง: ทำไม neural network ที่ scale ใหญ่พอถึงมีพฤติกรรมที่เรียกว่า "emergent" — ความสามารถที่ไม่มีในโมเดลเล็กแต่ปรากฏขึ้นมาเองในโมเดลใหญ่ โดยไม่มีใครโปรแกรมมันเข้าไป? ทำไม scale ถึง unlock สิ่งที่ดูเหมือน "เหตุผล" หรือ "ความเข้าใจ"? เรื่องเหล่านี้ยังถกเถียงกันอยู่ในปี 2026 และเป็นหนึ่งในคำถามที่น่าสนใจที่สุดในวิทยาการคอมพิวเตอร์ยุคนี้.

**สิ่งที่คุณควรบอกคนอื่นได้แล้วหลังจากบทนี้:**

- neural network คือชุดของ neuron ที่แต่ละตัวทำแค่คูณและบวก แต่เมื่อซ้อนกันหลายชั้นกับ activation function มันจับ pattern ซับซ้อนได้
- activation function คือสิ่งที่ทำให้ network ไม่ใช่สมการเส้นตรง — ขาดไปไม่ได้ และเหตุผลที่ XOR เคยพัง neural network ไป 20 ปี
- forward pass คือกระบวนการที่ input ไหลผ่านทุก layer ออกมาเป็น output — เกิดทุกครั้งที่คุณถาม ChatGPT ทีละ token
- training คือการปรับ weight พันล้านตัวซ้ำแล้วซ้ำเล่าด้วย gradient descent จนโมเดลเดาได้แม่น — ไม่ใช่การ "โปรแกรม" ความรู้เข้าไปตรง ๆ
- neural network เป็น function approximator ที่เรียนรู้ pattern จากข้อมูล ไม่ใช่แบบจำลองของสมอง และไม่มี consciousness
- transformer คือ architecture ที่ LLM ทุกตัวใช้ ต่างจาก MLP และ CNN ตรงที่ใช้ attention แทน recurrence

**สิ่งที่บทนี้ไม่ได้ลงลึก (เพราะมีบทเฉพาะสำหรับมัน):**

- กลไก attention ใน transformer ทำงานอย่างไรในระดับ operation — เรื่องของบทขั้นสูง
- backpropagation ทำงานอย่างไรในทางคณิตศาสตร์ (chain rule, gradient flow) — เรื่องของบท how-models-learn
- เทคนิค training จริง เช่น learning rate scheduling, regularization, RLHF — เรื่องของบทที่ลึกกว่า

ตอนนี้คุณรู้แล้วว่า neural network คืออะไรในระดับโครงสร้าง รู้ว่า neuron ทำอะไร รู้ว่า activation function สำคัญอย่างไร รู้ว่า layer ซ้อนกันสร้าง abstraction อย่างไร รู้ว่าความรู้ถูก encode ไว้ใน weight อย่างไร และรู้ว่า architecture ต่าง ๆ ถูกออกแบบมาเพื่อชนิดข้อมูลต่าง ๆ.

## คำถามที่ถามบ่อย

**"neural network กับ deep learning ต่างกันยังไง?"**

ไม่ต่างกัน — deep learning คือ neural network ที่มีหลาย hidden layer. คำว่า "deep" มาจากความลึกของ layer. ทุก deep learning model คือ neural network แต่ไม่ใช่ทุก neural network ที่เป็น deep learning (network 1-2 layer ไม่เรียกว่า deep).

**"neural network กับ machine learning ต่างกันยังไง?"**

machine learning เป็นหมวดใหญ่กว่า — รวมทุกเทคนิคที่ให้คอมพิวเตอร์เรียนรู้จากข้อมูลโดยไม่ต้องโปรแกรมทุกกฎ. neural network เป็นหนึ่งใน tool ของ machine learning. มี algorithm อื่น ๆ เช่น decision tree, random forest, SVM ที่ก็เป็น machine learning แต่ไม่ใช่ neural network.

ในปัจจุบันเมื่อพูดถึง AI ส่วนใหญ่หมายถึง neural network และ deep learning เพราะ performance ดีกว่า algorithm อื่น ๆ ในงานที่ซับซ้อน แต่ algorithm เก่า ๆ ยังใช้อยู่มากสำหรับงานที่ข้อมูลน้อยหรือต้องการ interpretability สูง.

**"ถ้า neural network แค่คูณและบวก ทำไมถึงฉลาดแบบนี้?"**

นี่คือคำถามที่ดีที่สุด. คำตอบคือ scale และ data. เมื่อ network มี parameter หลายพันล้านตัวและถูก train กับข้อมูล text เกือบทั้งอินเทอร์เน็ต pattern ที่มันจับได้นั้นซับซ้อนมาก. มันไม่ได้ "ฉลาด" ในแบบที่มนุษย์ฉลาด — แต่มันเก่งมากในการจับ statistical pattern ของภาษาและโลก ซึ่งในทางปฏิบัติดูเหมือนฉลาด.

Richard Feynman นักฟิสิกส์ชื่อดังเคยบอกว่า "What I cannot create, I do not understand." น่าสนใจว่าในทางกลับกันก็เป็นจริง — AI สร้างภาษาที่ฟังดูเข้าใจได้โดยไม่ต้องเข้าใจจริง ๆ. นี่คือหัวใจของการถกเถียงว่า AI "เข้าใจ" หรือแค่ "เลียนแบบ" ที่ยังไม่มีคำตอบที่ทุกฝ่ายยอมรับ.

**"ฉันควรเรียนรู้ neural network ให้ลึกกว่านี้ไหม?"**

ขึ้นอยู่กับเป้าหมาย. ถ้าต้องการ ใช้ AI ได้ดี — ความเข้าใจระดับนี้เพียงพอแล้ว. ถ้าต้องการ สร้าง AI application หรือ fine-tune โมเดล — ควรเรียนรู้เพิ่มเติมเรื่อง training และ architecture. ถ้าต้องการ วิจัย AI — ต้องลงลึกไปถึง backpropagation, optimization theory, และ architecture design. บทความในชุดนี้ครอบคลุมระดับแรกสองระดับ.

**"neural network รู้ว่าตัวเองทำอะไรอยู่ไหม?"**

ไม่. neural network ไม่มี self-awareness ไม่มี intention และไม่ "รู้สึก" อะไร. มันเป็นฟังก์ชันคณิตศาสตร์ที่รับ input แล้วให้ output. ที่ดูเหมือนมีความเข้าใจเป็นเพราะ pattern ที่มันจับจาก training data ซับซ้อนพอที่จะ "ดูเหมือน" เข้าใจ แต่ไม่เหมือนจริง.

คำถามว่า AI มี consciousness หรือไม่เป็นคำถามเชิงปรัชญาที่แยกออกจากคำถามทางวิศวกรรมว่ามันทำงานอย่างไร. เราอธิบายกลไกได้ชัดเจน แต่คำถามเรื่อง experience ยังเป็นเรื่องที่นักปรัชญาและนักวิทยาศาสตร์ถกเถียงกัน. สำหรับการใช้งานจริง คำตอบที่ปลอดภัยที่สุดคือ: treat AI เป็นเครื่องมือที่ทรงพลัง ไม่ใช่เพื่อนหรือสิ่งมีชีวิต.

---

บทถัดไปจะต่อยอดจากโครงสร้าง neural network ที่เพิ่งเรียน ไปสู่ **deep learning** — ทำไมการซ้อน neuron หลายชั้นลึก ๆ ถึงทำให้เกิดความสามารถที่ network ตื้น ๆ ทำไม่ได้เลย. และเมื่อข้ามไปถึงบท [LLM คืออะไร](/foundations/what-is-an-llm) ที่จะมาถึง คุณจะเห็นว่า transformer ซึ่งเป็น neural network ชนิดพิเศษที่ขับเคลื่อน ChatGPT และ Claude ทุกตัว ทำให้ทุก token "มองเห็น" ทุก token อื่น ๆ พร้อมกันได้อย่างไร. มันคือก้าวที่ทำให้ AI จากยุค rule-based กลายเป็น AI ที่คุณใช้อยู่ทุกวัน.

ถ้ามีสิ่งหนึ่งที่อยากให้จำจากบทนี้คือ: neural network เป็นเครื่องมือ ไม่ใช่เวทมนต์. มันทรงพลังมาก แต่ทรงพลังในแบบที่อธิบายได้ด้วยคณิตศาสตร์และสถิติ. ทุกครั้งที่คุณเจอข่าวเรื่อง AI ที่ "ทำได้เหนือมนุษย์" หรือ "ฉลาดกว่าคน" — กลับมาที่พื้นฐาน: มันคือ function approximator ที่เรียนรู้ pattern จาก data. ไม่มากไม่น้อยกว่านั้น — แต่นั่นก็เพียงพอที่จะเปลี่ยนโลกได้.

และนั่นคือสิ่งที่น่าทึ่งที่สุดของ neural network — ไม่ใช่ว่ามัน "ฉลาด" แต่ว่ามันแสดงให้เห็นว่าพฤติกรรมที่ดูเหมือนฉลาดสามารถเกิดขึ้นได้จากกลไกที่เรียบง่าย เมื่อ scale ใหญ่พอ. นั่นเป็นข้อเท็จจริงที่ยังทำให้นักวิทยาศาสตร์และนักปรัชญาต้องครุ่นคิดในปี 2026 และคงจะต่อไปอีกนาน.

คุณมีพื้นฐานแล้ว. บทต่อไปรอคุณอยู่.

ถ้าอยากทดสอบว่าเข้าใจจริง ๆ ลองอธิบายให้เพื่อนฟังว่า "neural network คืออะไร" ในสองนาที ไม่ใช้คำว่า 'เหมือนสมอง'. ถ้าทำได้ บทนี้ทำงานสำเร็จแล้ว.
