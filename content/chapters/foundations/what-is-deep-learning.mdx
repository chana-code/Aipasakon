# Deep Learning คืออะไร — เหตุที่ ChatGPT ทำสิ่งที่ไม่มีใครเขียน code สั่งได้

ลองนึกดูว่าถ้าจะสอนให้คอมพิวเตอร์แยกรูปแมวออกจากรูปสุนัข คุณจะเขียน rule อย่างไร. หูแหลมกว่า? จมูกสั้นกว่า? แต่สุนัขบางสายพันธุ์ก็มีหูแหลม แมวบางตัวก็มีจมูกยาว. ยิ่งเขียน rule ยิ่งมีข้อยกเว้น — จนถึงจุดที่นักวิทยาศาสตร์คอมพิวเตอร์หลายคนยอมรับว่า "เขียน rule ไม่ได้จริง ๆ".

แล้ว ChatGPT แยกได้ยังไง? แล้ว Midjourney วาดรูปแมวออกมาได้ยังไง?

คำตอบอยู่ที่ **deep learning** — เทคนิคที่หยุดพยายามเขียน rule แล้วปล่อยให้โมเดลค้นพบ rule เองจากข้อมูล. และนี่คือเหตุที่ AI ในปี 2026 ทำได้สิ่งที่ไม่มีใครเขียน code สั่งตรง ๆ ได้.

บทนี้จะอธิบายว่า deep learning คืออะไร ทำงานยังไงในระดับที่เข้าใจได้จริง (ไม่ใช่แค่นิยาม) และทำไมมันถึงเปลี่ยน AI ทั้งหมดตั้งแต่ปี 2012 จนถึง ChatGPT ที่คุณใช้อยู่วันนี้. บทนี้จะเป็นพื้นฐานที่ทำให้บท [LLM คืออะไร](/foundations/what-is-an-llm) ที่จะมาถึงเข้าใจง่ายขึ้นมาก.

## Deep Learning อยู่ตรงไหนใน AI

จากแผนภาพ 4 วงกลมในบท [AI คืออะไร](/foundations/what-is-ai) (AI ⊃ ML ⊃ Deep Learning ⊃ LLM) — deep learning คือวงที่สามจากนอก. สิ่งที่แยก deep learning ออกจาก ML ชั้นนอกคือข้อนี้: ML ทั่วไปยังต้องให้มนุษย์เตรียม "feature" ให้ก่อน เช่น บอกว่า "ให้ดูขนาดหู, สีตา, ความยาวจมูก" — แล้วโมเดลค่อยเรียนว่า feature ไหนสำคัญกว่า. Deep learning ข้ามขั้นตอนนั้น: ส่งข้อมูลดิบเข้าไปตรง ๆ แล้วโมเดลคิดออกเองว่าต้องดูอะไร.

และถ้ายิ่งลึกเข้าไปอีก **LLM** (เช่น ChatGPT) ก็อยู่ในวง DL — เป็น deep learning ที่ train บน text จำนวนมหาศาล. นั่นคือตัวอย่างของ deep learning ที่คุณรู้จักดีที่สุดในชีวิตประจำวัน และเราจะเจาะลึกมันในบท [LLM คืออะไร](/foundations/what-is-an-llm) ที่จะมาถึง.

เหตุที่ต้องรู้ตำแหน่งนี้ในชีวิตจริง: เวลาอ่านข่าวหรือประชุมกับทีม IT แล้วมีคนพูดว่า "เราจะใช้ AI ทำนี่" — บางทีพวกเขาหมายถึง rule-based system ธรรมดา บางทีหมายถึง classical ML บางทีหมายถึง deep learning ที่ใช้ GPU หลายตัว. งบประมาณ เวลา และ data ที่ต้องการต่างกันมาก. รู้ว่า deep learning อยู่วงไหนจะช่วยให้คุณถามคำถามที่ถูกต้อง.

> 💡 **Key insight:** เวลาอ่านข่าว AI แล้วเห็นคำว่า "AI", "ML", "deep learning" — ทั้งสามนี้ไม่ใช่คำเดียวกัน. แต่ในปี 2026 สิ่งที่ครองข่าว AI เกือบทั้งหมดคือ deep learning ทั้งนั้น — ChatGPT, Midjourney, Gemini, Stable Diffusion ล้วนอยู่ในวง DL. เมื่อคนพูดว่า "AI" ในปี 2026 พวกเขามักหมายถึง deep learning โดยไม่รู้ตัว.

## วิธีคิดเก่าที่ตันมาตลอด

เพื่อเข้าใจว่า deep learning เปลี่ยนอะไร ต้องเข้าใจก่อนว่าวิธีคิดเก่าทำงานยังไง — และทำไมมันถึงตันในงานบางประเภท.

ก่อน deep learning ยุค (ก่อน ~2012) AI ส่วนใหญ่ทำงานแบบ "ใช้ rule และ feature ที่มนุษย์คิด". วงการ AI ใช้เวลาหลายสิบปีพยายาม "เข้ารหัส" ความรู้มนุษย์ลงไปในระบบ — เรียก expert system. ผลคือทำงานได้ดีในขอบเขตที่แคบมาก แต่พอ input เปลี่ยนนิดเดียว ระบบพัง.

ตัวอย่างที่ดี: ระบบ AI เก่าในปี 1990 ที่ไม่ถูกต้อง (เช่น Deep Blue ที่เล่นหมากรุก) ทำงานได้เฉพาะในเกมหมากรุกเท่านั้น — ถ้าเปลี่ยนเป็นหมากฮอส มันทำไม่ได้เลยแม้แต่ตัวเดียว. rule ที่เขียนไว้เป็นของหมากรุกโดยเฉพาะ. deep learning ที่ train กับ data หลากหลายสามารถ generalize ได้ดีกว่ามาก — เช่น model ที่ train กับรูปทั่วไปสามารถนำมา fine-tune สำหรับงานใหม่ได้โดยใช้ data น้อย.

ลองนึกถึงงาน **แยกเสียงพูดเป็นข้อความ** (speech recognition) ในยุคก่อน deep learning. นักวิทยาศาสตร์ต้องทำแบบนี้:

1. บันทึกเสียงพูดมาเป็นไฟล์
2. เขียน code สกัด "feature" ออกมาจากเสียง — เช่น ความถี่เสียง, การขึ้นลงของน้ำเสียง, ช่วงเงียบระหว่างคำ
3. ป้อน feature เหล่านั้นให้โมเดลเรียนว่า pattern ไหนตรงกับเสียง "ก" เสียง "ข" เสียง "ค"

ขั้นที่ 2 คือ **feature engineering** — และมันคือฝันร้ายของวงการ.

เพราะเสียงพูดของคนสองคนที่พูดคำเดียวกันไม่เหมือนกัน. ผู้หญิงกับผู้ชายเสียงต่างกัน. คนแก่กับคนหนุ่มต่างกัน. สำเนียงเหนือกับสำเนียงใต้ต่างกัน. คนเป็นหวัดกับคนปกติต่างกัน. ทุกครั้งที่เจอข้อยกเว้น ต้องกลับไปเขียน feature ใหม่ — หรือแก้ feature เดิม.

งาน vision (แยกรูปวัตถุ) ก็เหมือนกัน. ก่อน deep learning วิศวกรต้อง hand-craft feature เช่น SIFT และ HOG — algorithm ที่คนเขียนเพื่อตรวจจับ edge และ texture ในรูปภาพ. ใช้เวลาหลายปีในการ refine feature พวกนี้. และยังไม่ดีพอ.

งาน NLP (ประมวลผลภาษา) ก็เหมือนกัน. ก่อน deep learning ต้องสร้าง bag-of-words, TF-IDF, n-gram — เป็น feature ที่คนคิดขึ้นมาเพื่อ represent ความหมายของข้อความ. แต่ feature พวกนี้จับ nuance ของภาษาได้น้อยมาก.

**มนุษย์เป็นคอขวด** — ต้องมีผู้เชี่ยวชาญมาคิด feature ก่อน โมเดลถึงจะทำงานได้. และในงานที่ complex เช่น "ทำความเข้าใจความหมายของประโยค" ไม่มีผู้เชี่ยวชาญคนไหนบอกได้ว่า feature ของ "ความหมาย" คืออะไร. ความหมายของภาษาไม่ใช่สิ่งที่สามารถเขียนเป็น rule ได้ — มันเกิดจาก pattern ของการใช้ภาษาในหลายล้าน context.

deep learning พลิกสิ่งนี้กลับหัว.

สังเกตได้ว่า bottleneck ของ classical ML อยู่ที่ขั้นตอนที่ต้องใช้ human intelligence มาก ๆ — feature engineering. และมนุษย์ก็มีข้อจำกัด: เราเห็น feature ที่ตาเห็น feature ที่มีชื่อในภาษา feature ที่เคยคิดถึง. แต่ pattern ที่จริง ๆ แล้วสำคัญในรูป X-ray อาจเป็น combination ของ pixel สีเทาหลายพันจุดในรูปแบบที่ไม่มีชื่อ ไม่มีคำอธิบาย ไม่มีใครเคยนึกถึง. deep learning สามารถค้นพบ pattern นั้นได้ถ้า data มีมากพอ.

> 💡 **Key insight:** feature engineering คือ "ความรู้ที่มนุษย์รู้แล้ว" — representation learning คือ "ความรู้ที่ data บอกว่ามีอยู่แต่มนุษย์ไม่รู้ชื่อ". ในงานที่ซับซ้อน เช่น ภาษา, รูปภาพ, เสียง — ความรู้ประเภทที่สองมีมากกว่าประเภทแรกมาก. นั่นคือเหตุที่ deep learning ทำงานได้ดีกว่าในงานเหล่านี้.

## หัวใจของ deep learning: โมเดลเรียน feature เอง

แทนที่จะบอกว่า "ให้ดู feature เหล่านี้" — deep learning บอกว่า "เอา data ดิบมาเลย แล้วหา pattern เอง".

กลับไปที่งานแยกแมวกับสุนัข:

**วิธีเก่า (classical ML):** วิศวกรนั่งคิดว่า feature ที่แยกแมวกับสุนัขได้คืออะไร — เขียน code สกัด feature เหล่านั้นออกมาก่อน แล้วส่งเข้าโมเดล. โมเดลแค่เรียนว่าจะ weight feature เหล่านั้นอย่างไร.

**วิธีใหม่ (deep learning):** ส่งรูปดิบ (pixel ล้วน ๆ) เข้าโมเดลตรง ๆ. โมเดลค้นพบเองว่าต้องดูอะไร ผ่านกระบวนการ training.

และนี่ไม่ใช่แค่การประหยัดแรง — feature ที่โมเดลค้นพบเองมักดีกว่าที่มนุษย์คิดด้วย เพราะมันจับ pattern ที่มนุษย์ไม่เคยนึกถึง ไม่มีชื่อเรียก และไม่สามารถเขียนเป็น rule ได้.

ผลที่ตามมาจากการที่โมเดลค้นพบ feature เองคือ: ChatGPT รู้ว่า "อาหารร้านนี้อร่อยมาก ไปกินอีก 3 ปีหลังได้" หมายถึงอาหารแย่มาก — ทั้งที่ไม่มีใครเขียน rule ว่า "ถ้าบอกว่าจะไปกิน 3 ปีหลัง = ไม่ชอบ". มันเรียน pattern ของ sarcasm จาก context ที่คล้ายกันหลายพันครั้งใน training data. ไม่มีนักภาษาศาสตร์คนไหนสอน — แต่โมเดลจับได้.

ความสามารถนี้มีชื่อในวงการว่า **representation learning** — โมเดลเรียนรู้การ "แทนค่า" ข้อมูลด้วยตัวเอง. และมันคือเหตุที่ deep learning ปฏิวัติวงการ AI ทั้งหมด.

ลองเปรียบกับการเรียนภาษาใหม่. วิธีเก่า (classical ML) เหมือนการเรียนภาษาจากตำรา — ใครบางคนสรุป grammar ให้ คุณท่องจำกฎ 200 ข้อ แล้วพยายามประกอบประโยค. วิธี deep learning เหมือนการอยู่ในประเทศนั้น 5 ปี — ไม่มีใคร "สอน" grammar ตรง ๆ แต่คุณซึมซับ pattern ของภาษาจาก context หมื่น ๆ ครั้ง. ผลที่ได้ต่างกัน: คนที่อยู่ในประเทศมาพูดได้เป็นธรรมชาติกว่า จับ nuance ได้มากกว่า แม้จะไม่ได้ท่อง rule ตรง ๆ.

analogy นี้ใช้ได้ที่ว่า "เรียนจาก experience มากกว่า rule" — ส่วนที่ต่างคือคนเรียนภาษาใช้เวลา 5 ปี แต่ deep learning ใช้เวลาหลายวันหลายสัปดาห์ใน compute cluster ที่ประมวลผลข้อความหลาย trillion ตัวอักษร.

### กลไกที่ทำให้โมเดลเรียนรู้: ผิดก็แก้ ผิดก็แก้

ถ้าสงสัยว่า "โมเดลเรียนรู้ feature เองได้ยังไง" คำตอบสั้น ๆ คือ: ผ่านกระบวนการที่ทำซ้ำหลายล้านครั้ง — ดู input, เดาผลลัพธ์, เช็คว่าผิดหรือถูก, แล้วปรับตัวเลขใน network ให้เดาได้ถูกขึ้น.

ลองนึกภาพแบบนี้: เด็กเล็กที่กำลังเรียนรู้ว่าอะไรคือ "แมว". พ่อแม่แสดงรูปให้ดู บอกว่า "นี่แมว" "นี่ไม่ใช่แมว". ทีแรกเดาสุ่ม. แต่ทุกครั้งที่เดาผิด สมองปรับ weight ของ "สัญญาณ" ต่าง ๆ ที่เห็น — ค่อย ๆ เรียนรู้ว่า feature ไหนสำคัญ.

deep learning ทำแบบเดียวกัน แต่ systematic กว่ามาก: มีสูตรคำนวณว่า "ผิดมากแค่ไหน" (เรียกว่า loss function) และมีอัลกอริธึมที่บอกว่าต้องปรับ parameter ตัวไหนมากแค่ไหนเพื่อให้ผิดน้อยลง (เรียกว่า gradient descent). รายละเอียดของ gradient descent อยู่ในบทเรื่อง [how models learn](/foundations/how-models-learn) — ตอนนี้รู้แค่นี้ก็เพียงพอแล้ว.

สิ่งที่ทำให้ deep learning ทรงพลังคือ: กระบวนการ "ผิดก็แก้" นี้ทำงานผ่าน layer ทั้งหมดพร้อมกัน. ผลคือทุก layer เรียนรู้ feature ที่ช่วยให้ผลลัพธ์สุดท้ายถูกต้องขึ้น — ไม่ใช่แค่ layer สุดท้าย. นั่นคือเหตุที่ layer ต้น ๆ เรียน edge, layer กลางเรียน part, layer ลึกเรียน object — ทุกอย่างเกิดขึ้นโดยไม่มีใคร "สั่ง".

ตัวเลขที่น่าประทับใจ: ในการ train GPT-3 โมเดลดู text ราว 300 พันล้าน token. สมมติคนอ่านปกติอ่านได้ 250 คำต่อนาที 8 ชั่วโมงต่อวัน — ต้องใช้เวลากว่า 1,000 ปีในการอ่าน text ปริมาณเท่ากัน. GPT-3 train เสร็จใน 30-40 วันด้วย GPU หลายพันตัว. การเห็น pattern ซ้ำ ๆ ในข้อมูลมหาศาลนั้นคือสิ่งที่ทำให้ representation learning ทำงานได้จริง.

> 💡 **Key insight:** representation learning ไม่ใช่แค่ทำให้งานง่ายขึ้น — มันเปิดประตูให้กับงานที่ "เขียน feature ไม่ได้เลย" เช่น ทำความเข้าใจความหมายของประโยค หรือสร้างรูปจาก text หรือแปลภาษาที่ไม่เคยเห็น. ถ้าไม่มี representation learning ก็ไม่มี ChatGPT — และไม่มีทาง build มันจากการเขียน rule ได้.

## "Deep" หมายถึงอะไรจริง ๆ

คำว่า "deep" ใน deep learning หมายถึง **จำนวนชั้น (layer) ที่ซ้อนกันใน neural network**. ไม่ใช่ "ลึก" ในความหมายว่า "ฉลาดมาก" หรือ "ซับซ้อนมาก" แต่หมายถึงตัวเลขตรง ๆ — มี layer กี่ชั้น.

neural network แบบเก่าที่ใช้กันก่อนปี 2006 มักมีแค่ 1-2 layer. deep network ในยุค 2010s เริ่มมี 10, 50, 100 layer ขึ้นไป. GPT ในปี 2026 มีประมาณ 100 transformer block ซ้อนกัน — แต่ละ block คือชุดของ layer.

ลองดูตัวเลขจริงเพื่อให้เห็นภาพ:

| โมเดล | ปี | จำนวน layer | งานที่ทำ |
|-------|-----|------------|---------|
| Shallow neural network | ก่อน 2006 | 1-2 hidden layer | งานง่าย ๆ เท่านั้น |
| AlexNet | 2012 | 8 layer | แยกประเภทรูปภาพ |
| VGG-16 | 2014 | 16 layer | vision ที่แม่นขึ้น |
| ResNet-152 | 2015 | 152 layer | ต่ำกว่า human level |
| GPT-3 | 2020 | 96 transformer block | ภาษาธรรมชาติ |
| GPT-4 (ประมาณ) | 2023 | ไม่เปิดเผย | ใช้งานทั่วไปปัจจุบัน |

ตัวเลขนี้แสดงให้เห็นว่า "deep" ไม่ใช่คำอุปมาอุปไมย — มันคือตัวเลขที่เพิ่มขึ้นจริง และผลที่ได้ก็ดีขึ้นตาม (จนถึงจุดหนึ่ง).

แต่ทำไมต้องลึก? ทำไม 1-2 layer ไม่พอ?

เพราะแต่ละ layer **สร้างความเข้าใจบนสิ่งที่ layer ก่อนเรียนไว้**. มันเหมือนการสร้างบ้าน — ต้องวางรากฐานก่อน แล้วค่อยสร้างผนัง แล้วค่อยวางหลังคา. ถ้าพยายามวางหลังคาโดยไม่มีผนัง มันจะพัง. แต่ละ layer คือชั้นใหม่ที่ต่อยอดจากชั้นก่อน.

ถ้ามีแค่ 1-2 layer โมเดลสามารถเรียนได้แค่ pattern ง่าย ๆ เช่น "ถ้า pixel ตรงนี้สว่างและตรงนั้นมืด น่าจะเป็นแมว" — ซึ่งจริง ๆ แล้วทำงานไม่ได้กับรูปหลายล้านรูปที่หลากหลาย. layer ที่ลึกขึ้นทำให้โมเดลสร้าง representation ที่ซับซ้อนและ abstract ขึ้นได้.

มีทฤษฎีทางคณิตศาสตร์ที่พิสูจน์ได้ว่า neural network ที่มี 1 hidden layer พอใหญ่สามารถ approximate function ใดก็ได้ (universal approximation theorem). ฟังดูแปลก — ถ้าแค่ชั้นเดียวก็ทำได้ทุกอย่างแล้ว ทำไมต้องลึก? คำตอบคือ "พอใหญ่" ในที่นี้หมายถึงกว้างมหาศาล — จำนวน neuron ต่อ layer ที่ต้องมีมากจนไม่ practical. deep network ใช้ layer น้อยกว่าแต่ลึกกว่า ซึ่ง parameter-efficient กว่ามากและ train ได้จริงในทางปฏิบัติ.

### ดูจากรูปภาพ: จาก pixel สู่ "แมว"

ในโมเดลที่ train จากรูปสัตว์ ถ้า visualize ว่าแต่ละ layer "เห็น" อะไร จะเห็น pattern ที่ชัดมาก:

- **layer 1-3:** เห็นแค่ขอบ (edge) — เส้นแนวตั้ง แนวนอน ทแยง — และ blob ของสี. ทุก layer แรก ๆ ทุก CNN ทุกตัวจะเรียน pattern แบบนี้
- **layer 10-15:** เริ่มเห็น texture — ขนสั้น ขนยาว ผิวเรียบ ลายทาง. ยังไม่รู้ว่านี่คืออะไร แค่รู้ว่า pattern นี้มีอยู่
- **layer 25-30:** เริ่มเห็น "ส่วน" ของสัตว์ — หู ตา จมูก ขา. ตรงนี้เองที่ feature ของแมวกับสุนัขเริ่มต่างกันชัดขึ้น
- **layer 45+:** เห็น "หน้าแมว" หรือ "หน้าหมา" ทั้งหน้า. layer นี้คือ representation สุดท้ายที่ใช้ตัดสินใจว่ารูปนี้เป็นอะไร

ไม่มีใครบอกโมเดลว่า "ให้ดู edge ก่อน แล้วค่อยประกอบเป็น part แล้วค่อยประกอบเป็นใบหน้า" — มันเรียนลำดับนี้เอง ผ่าน training กับรูปสัตว์หลายล้านรูป.

นี่คือ **hierarchical representation** — การเรียนรู้แบบมีลำดับชั้นจากง่ายไปซับซ้อน. และมันเกิดขึ้นได้เพราะ network ลึกพอที่จะมี "ชั้น" ให้ pattern ประกอบสะสมกันได้.

มีจุดที่น่าสังเกต: layer ต้น ๆ ทุก CNN ที่ train กับ data ต่าง ๆ กัน ล้วนเรียน edge detector คล้าย ๆ กัน. ไม่ว่าจะ train กับรูปแมว รูปรถ หรือรูปดอกไม้ — layer แรกออกมาหน้าตาคล้ายกันเกือบทุกโมเดล. นี่บอกว่า edge คือ feature พื้นฐานที่สุดในการเข้าใจรูปภาพ ไม่ว่าจะ task ไหน — โมเดลค้นพบเรื่องนี้เองทุกครั้ง.

ในทางกลับกัน layer ลึก ๆ จะต่างกันมากระหว่างโมเดล ขึ้นอยู่กับว่า train กับ data อะไร. โมเดลที่ train กับรูปแมวจะมี layer ลึกที่ "หน้าตา" ต่างจากโมเดลที่ train กับรูปรถ — เพราะ high-level feature ของแมวกับรถไม่มีอะไรเหมือนกันเลย. นี่แสดงให้เห็นว่าการเรียนรู้เกิดขึ้นจริงในระดับ semantic ตาม task ที่ train.

### เช่นเดียวกันใน LLM ที่ train จาก text

hierarchical representation ไม่ได้เกิดแค่กับรูปภาพ. ใน LLM อย่าง ChatGPT ที่ train จาก text จำนวนมหาศาล รูปแบบเดียวกันเกิดขึ้นในระดับภาษา:

- **layer ต้น ๆ** จับ pattern ระดับตัวอักษรและคำ — เช่น "คำนี้มักอยู่ต่อกับคำไหน" หรือ "ตัวอักษรไหนมักตามหลังตัวอักษรไหน"
- **layer กลาง ๆ** เริ่มจับโครงสร้างประโยค — ประธาน กริยา กรรมอยู่ตรงไหน ประโยคนี้เป็นคำถามหรือคำสั่ง
- **layer ลึก ๆ** จับความหมาย context ระดับย่อหน้าและเอกสาร — "ประโยคนี้พูดถึงเรื่องอะไร" หรือ "คนนี้พูดในน้ำเสียงแบบไหน"

ChatGPT ไม่มีใครสอนว่า grammar ภาษาไทยคืออะไร ว่า subject คืออะไร object คืออะไร. มันเรียน structure ของภาษาเองจาก text หลาย trillion token. นั่นคือ representation learning ในระดับภาษา — เหมือนกันกับที่ CNN เรียนจาก pixel.

น่าสังเกตว่า LLM ใน text ก็มี pattern เดียวกันกับ CNN ใน image — layer ต้น ๆ เรียน feature ง่าย ๆ ที่ใช้ได้กับทุกงาน, layer ลึกเรียน feature เฉพาะเจาะจงกับ task. ความรู้เรื่อง hierarchical representation ที่คุณเรียนจาก CNN ในบทนี้จึงใช้ได้กับ LLM โดยตรง.

### ลองทำดู

เปิด ChatGPT แล้วลองถามว่า:

"ถ้าฉันให้รูปหมา 1 ล้านรูปกับรูปแมว 1 ล้านรูปให้ AI ดู แต่ไม่บอกว่ารูปไหนเป็นอะไร AI จะเรียนรู้ได้ไหมว่ารูปไหนเป็นหมา รูปไหนเป็นแมว"

ดูว่า ChatGPT ตอบอะไร แล้วลองถามต่อว่า "แล้วถ้าให้แค่ 10 รูป จะยังได้ผลไหม"

คุณจะเห็นว่ามันอธิบายเรื่อง data ที่ต้องการได้ดีทีเดียว เพราะมันเองก็เป็นผลผลิตของ deep learning ที่ต้องการ data จำนวนมาก.

## ปี 2012 — วันที่โลกเปลี่ยน

แนวคิดของ neural network ที่มีหลาย layer ไม่ใช่เรื่องใหม่. มีมาตั้งแต่ทศวรรษ 1960. backpropagation ซึ่งเป็นอัลกอริธึมหลักที่ใช้ train deep network ก็มีตั้งแต่ทศวรรษ 1980.

แต่จนถึงปี 2010 — deep learning ยังไม่ work ในทางปฏิบัติ. Network ลึก ๆ ที่มีหลาย layer กลับ train ได้ไม่ดี ผลออกมาด้อยกว่า classical ML ที่มีคนเตรียม feature ไว้ให้ดี ๆ.

แล้วอะไรเปลี่ยน?

### ปัจจัยที่ 1 — Data ระดับอินเทอร์เน็ต

ปี 2009-2012 ImageNet ถูกสร้างขึ้น — รูปภาพมี label กว่า 14 ล้านรูป คนนั่ง label ทีละรูปทีละรูป จัดหมวดหมู่เป็นพัน ๆ ประเภท. ก่อนหน้านี้ dataset ขนาดนี้ไม่มีอยู่. dataset ที่ใหญ่ที่สุดในยุคก่อนมักมีแค่หลักหมื่นรูป.

deep network ที่มี parameter เป็นล้านตัว "หิว" data — ถ้า data ไม่มากพอ มันจะจำ training data มากเกินไปแต่ไม่ generalize กับรูปใหม่ที่ไม่เคยเห็น. เหมือนนักเรียนที่ท่องจำข้อสอบเก่าจนขึ้นใจ แต่พอเจอข้อสอบใหม่ที่เปลี่ยนคำถามนิดเดียว ตอบไม่ได้.

ตัวอย่างที่ชัด: ถ้าจะ train โมเดลให้รู้จัก "แมว" คุณต้องให้มันเห็นแมวในมุมต่าง ๆ แสงต่าง ๆ สายพันธุ์ต่าง ๆ พื้นหลังต่าง ๆ. ถ้ามีแค่รูปแมวสีส้มนอนบนโซฟา 100 รูป โมเดลอาจ "เรียน" ว่า "แมว = พื้นหลังโซฟา" ไม่ใช่ feature ของแมวจริง ๆ. Data หลายล้านรูปที่หลากหลายทำให้โมเดลต้องหา feature ที่ generalize ได้จริง.

ในฝั่งภาษา text บนอินเทอร์เน็ตทั้งหมด — Wikipedia, หนังสือดิจิทัล, บทสนทนาออนไลน์ — กลายเป็น training data ที่มีปริมาณหลาย trillion token. นี่คือ data ที่ทำให้ LLM เป็นไปได้ในทศวรรษถัดมา.

### ปัจจัยที่ 2 — GPU

GPU (การ์ดจอ) ถูกออกแบบมาเพื่อเร่ง graphics rendering — แต่กลับเหมาะมากกับ matrix multiplication ซึ่งเป็นการคำนวณหลักของ neural network.

ทำไม GPU เหมาะกับ neural network? เพราะทั้งสองอย่างต้องการการคำนวณแบบเดียวกัน: คำนวณหลายร้อยหลายพัน operation พร้อมกันพร้อมกัน (parallel computation). GPU ออกแบบมาคำนวณแบบนั้นตั้งแต่ต้น. ซีพียูทั่วไปทำได้ แต่ทำได้ทีละไม่กี่ operation.

ลองนึกภาพโรงงานผลิตสินค้า: ซีพียูเหมือนคนงานเก่ง 8 คนทำได้ทุกอย่างแต่ทำทีละชิ้น. GPU เหมือนคนงาน 10,000 คนที่ทำงานได้แค่อย่างเดียว (คูณตัวเลข) แต่ทำพร้อมกันได้ทั้งหมด. สำหรับงาน neural network ที่ต้องคูณตัวเลขซ้ำ ๆ เป็นล้านล้านครั้ง โรงงานแบบที่สองเร็วกว่ามาก.

ราคา compute ต่อหน่วยลดลงราว 100 เท่าในทศวรรษ 2010s. Training ที่เคยใช้เวลาเป็นเดือนบนซีพียูลดเหลือหลายวันบน GPU. นี่เปิดให้ทดลองโมเดลได้เร็วขึ้น ทำซ้ำได้มากขึ้น และสร้าง network ที่ใหญ่ขึ้นได้.

ในปี 2026 NVIDIA ยังเป็นบริษัท semiconductor ที่ market cap สูงที่สุดในโลกช่วงหนึ่ง — ส่วนใหญ่เพราะ GPU ของพวกเขาเป็นสิ่งที่ทุกบริษัท AI ต้องการ. สิ่งที่เริ่มจากการ train AlexNet บน GPU 2 ตัวในปี 2012 กลายเป็น industry ขนาดใหญ่ที่ขับเคลื่อนเศรษฐกิจโลก. ทั้งหมดนี้มาจากการค้นพบว่า GPU ที่ออกแบบมาวาดภาพในเกม สามารถ train neural network ได้เร็วกว่าซีพียูหลายร้อยเท่า.

### ปัจจัยที่ 3 — Algorithm เล็ก ๆ ที่เปลี่ยนทุกอย่าง

มีเทคนิคหลายอย่างที่ดูเล็กน้อยแต่สำคัญมาก — แต่ละอย่างแก้ปัญหาที่ทำให้ deep network ก่อนหน้านี้ train ไม่ได้:

**ReLU activation** — ก่อนหน้านี้ neural network ใช้ sigmoid หรือ tanh เป็น activation function. ปัญหาคือ gradient ค่อย ๆ หายไปเมื่อผ่านหลาย layer (vanishing gradient) จนโมเดลไม่อัปเดต weight ในชั้นต้น ๆ ได้จริง. ReLU แก้ปัญหานี้ด้วยสมการที่เรียบง่ายมาก: ถ้าตัวเลขเป็นบวก ส่งผ่านตามเดิม ถ้าลบ เปลี่ยนเป็น 0. แค่นั้น. แต่มันทำให้ gradient ไหลผ่านหลาย layer ได้โดยไม่หาย.

**Dropout** — เทคนิคที่ระหว่าง training ปิด neuron สุ่ม ๆ ราว 50% ทุก step. ฟังดูแปลก แต่ผลคือโมเดลไม่สามารถ "พึ่ง" neuron ใดตัวเดียว ทำให้ต้องเรียน pattern ที่ robust มากขึ้น และลด overfitting.

**Batch normalization** — ทำให้ distribution ของตัวเลขที่ผ่านแต่ละ layer มี mean และ variance ที่สม่ำเสมอ. ผลคือ training stable มากขึ้นแม้ network จะลึก.

**Weight initialization ที่เหมาะสม** — network ที่เริ่มต้น weight ผิดวิธีจะมี gradient ที่ระเบิดหรือหายตั้งแต่แรก. Xavier init และ He init แก้จุดนี้.

แต่ละอย่างฟังดูเทคนิคเล็กน้อย แต่รวมกันแล้วทำให้ network 100+ layer train ได้ ในขณะที่ก่อนหน้านี้ network 5 layer ยังหาคำตอบไม่เจอ.

เหมือนการขับรถขึ้นเขาสูง — ถ้าขาดแค่น้ำมันเพียงสิ่งเดียวก็ไปต่อไม่ได้. algorithm พวกนี้เหมือนน้ำมันที่ขาดไม่ได้ แต่ไม่ใช่สิ่งที่ทำให้รถวิ่งได้เอง — ต้องมีทั้ง data, compute, และ architecture ร่วมกัน.

เปรียบเทียบนี้ใช้ได้ที่ว่า "ต้องการหลายสิ่งพร้อมกัน" — ส่วนที่ต่างคือรถขาดน้ำมันก็หยุดทันที แต่ deep network ที่ขาด algorithm พวกนี้จะ train ได้บ้าง แค่ได้ผลแย่กว่ามากและ network ลึก ๆ จะ train ไม่ได้เลย.

### ปัจจัยที่ 4 — AlexNet: จุดพิสูจน์ที่โลกจำได้

ปี 2012 Krizhevsky, Sutskever และ Hinton ส่ง CNN ลึก (ที่ต่อมาเรียกว่า AlexNet) เข้าแข่ง ImageNet Challenge — การแข่งขันที่วัดว่าโมเดลไหนแยกประเภทรูปภาพได้แม่นกว่ากัน. โจทย์คือจัดหมวดหมู่รูปภาพ 150,000 รูปให้ถูกต้องใน 1,000 หมวด มีทีมจากมหาวิทยาลัยชั้นนำทั่วโลกเข้าร่วมทุกปี.

ก่อนหน้านั้น top-5 error rate ของผู้นำอยู่ที่ประมาณ 26% — ปีที่ผ่านมาใครทำได้ 25% ก็ถือว่าเก่งมาก. การปรับปรุงปีละ 1-2% คือ norm ของวงการ. ทุกคนใช้ hand-crafted feature เป็น standard.

AlexNet ทำได้ **15.3%** — ลดลงกว่า 10 percentage point ในปีเดียว. อันดับ 2 อยู่ที่ 26.2%.

ไม่มีใครในวงการเคยเห็นการกระโดดขนาดนี้มาก่อน. ไม่ใช่แค่ชนะ — แต่ชนะอย่างถล่มทลายจนทุกคนต้องหยุดและถามว่า "เกิดอะไรขึ้น". Hinton พูดในภายหลังว่า "ทันใดนั้นทุกคนก็ฟังเรา".

เพื่อให้เห็นภาพชัดขึ้น: ในวงการวิทยาศาสตร์ถ้าผลทดลองชนะค่า baseline มา 2-3% นั่นถือเป็น breakthrough ที่ตีพิมพ์ได้ใน top journal. AlexNet ชนะมา 11%. ในโลกที่ทุกคนคาดว่าจะเห็น "ปรับปรุงเล็กน้อย" กลับเห็นการก้าวกระโดดที่ไม่มีใครเคยจินตนาการ.

ปี 2013-2014 ทุก team ใน competition หันมาใช้ deep learning. ภายในปี 2015 ResNet ทำ error rate ต่ำกว่า human level ที่ 5% — machine เก่งกว่ามนุษย์ในการแยกประเภทรูปแล้ว. วงการ computer vision เปลี่ยน paradigm ภายใน 3 ปี.

แล้ว deep learning ก็ขยายไป NLP, speech, generative model — จนถึง LLM ที่คุณใช้อยู่ทุกวันนี้.

ย้อนกลับมามองประวัติศาสตร์: Geoffrey Hinton ที่ส่ง AlexNet เข้าแข่งในปี 2012 เป็นคนเดียวกับที่ได้รับ Nobel Prize สาขาฟิสิกส์ในปี 2024 จากงาน neural network ของเขา. John Hopfield ได้รับ Nobel Prize เดียวกัน. บางทีโลกใช้เวลา 12 ปีในการตระหนักว่าเกิดอะไรขึ้นในปีนั้น.

เรื่องน่าสังเกตอีกอย่าง: Hinton ลาออกจาก Google ในปี 2023 เพื่อพูดถึงความเสี่ยงของ AI ได้อย่างอิสระ. คนที่สร้างรากฐานของ deep learning และเห็นว่ามันเป็นไปได้ก่อนใคร ก็เป็นคนที่เตือนเรื่อง risk ของ AI จริงจังที่สุดคนหนึ่ง. deep learning ทรงพลังมาก — และ power นั้นมาพร้อม responsibility ที่วงการทั้งหมดกำลังคิดหาวิธีรับมือ.

> 💡 **Key insight:** deep learning ไม่ใช่เทคโนโลยีที่คิดขึ้นมาใหม่ในปี 2012 — แนวคิดมีมาตั้งแต่ทศวรรษ 1960. สิ่งที่เกิดขึ้นในปี 2012 คือ **ปัจจัย 4 อย่างมาบรรจบกันพอดี** จนทำให้แนวคิดเก่าที่เคยไม่ work กลายเป็น work. บ่อยครั้งในเทคโนโลยี ไม่ใช่ "idea ใหม่" แต่ "เงื่อนไขที่สุกงอม" ต่างหากที่ทำให้อะไรบางอย่างเกิดขึ้นได้.

## deep learning เก่งอะไร และไม่เก่งอะไร

มีความเข้าใจผิดที่พบบ่อยมาก: "AI ปฏิวัติโลกแล้ว ดังนั้น deep learning ดีกว่า classical ML ในทุกงาน"

นั่นไม่ถูก.

มีงานที่ classical ML ยังชนะอยู่ในปี 2026 — และนักวิทยาศาสตร์ข้อมูลที่เก่งรู้ว่าเมื่อไหรควรใช้อะไร.

ลองนึกแบบนี้: deep learning ปฏิวัติโดยการ "เปิดประตู" งานที่ก่อนหน้านี้ทำไม่ได้เลย เช่น สร้างรูปจาก text, ตอบคำถามในภาษาธรรมชาติ. แต่งานที่ classical ML ทำได้อยู่แล้วก่อนปี 2012 — เช่น ทำนาย churn จาก tabular data — classical ML ยังทำได้ดีพอ ๆ กัน หรือดีกว่าในหลายกรณี. deep learning ไม่ได้ make classical ML obsolete — มันแค่เพิ่ม tool ใหม่ที่ทรงพลังมากในงานบางประเภทเข้ามาใน toolkit.

### deep learning เป็นทางเลือกที่ดี เมื่อ

งานที่เกี่ยวกับ **การรับรู้** (perception) — รูปภาพ เสียง ภาษา. เหล่านี้คืองานที่ input มีโครงสร้างซับซ้อนและมีมิติสูงมาก เขียน feature ด้วยมือไม่ได้จริง ๆ. รูปภาพขนาด 512x512 pixel มี 262,144 มิติ — มนุษย์บอกไม่ได้ว่า pixel ไหนสำคัญ.

งานที่มี **data จำนวนมาก** — หลายแสนถึงหลายล้านตัวอย่าง. deep network มี parameter เป็นล้าน ต้องการ data มากพอถึงจะ generalize ได้โดยไม่ overfit.

งาน **generative** — สร้างรูป สร้างเสียง สร้างข้อความ. classical ML แทบไม่สามารถทำได้เลย. deep learning ทำได้ และทำได้ดีมาก.

**end-to-end learning** — อยากให้ระบบเรียนทุก step จาก raw input ถึง output โดยไม่ต้องออกแบบ pipeline แยกย่อย. deep learning ทำได้ดีกว่า classical ML มาก.

### classical ML (decision tree, XGBoost, etc.) ยังเหนือกว่าใน

**Tabular data** — ข้อมูลในรูปแบบตาราง เช่น ข้อมูลธุรกรรมลูกค้า รายงานการเงิน ข้อมูล ERP. ปัจจุบัน XGBoost และ LightGBM ยังชนะ deep learning ใน Kaggle competition ประเภทนี้บ่อยมาก. ในปี 2026 นี้ยังเป็นความจริง.

**Data ขนาดเล็ก** — ถ้ามีข้อมูลแค่ไม่กี่พันตัวอย่าง deep network จะ overfit — คือจำ training data มากเกินไปแต่ไม่ generalize กับข้อมูลใหม่. classical ML ที่มี regularization ดีให้ผลที่น่าเชื่อถือกว่า.

**งานที่ต้องการ interpretability** — ถ้าต้องอธิบายให้ผู้บริหารหรือหน่วยงานกำกับดูแลว่าโมเดลตัดสินใจอย่างไร decision tree อ่านออกตรง ๆ. deep network เป็น "กล่องดำ" ที่แม้แต่คนสร้างก็อธิบาย logic ภายในได้ยาก.

**ระบบที่ compute จำกัด** — อุปกรณ์ฝังตัว (embedded device) ที่ไม่มี GPU. classical ML inference เร็วกว่าและกินทรัพยากรน้อยกว่ามาก.

### ดูจากตัวอย่างจริง

สมมติบริษัทมีข้อมูลธุรกรรมลูกค้า 50,000 รายการ 30 column (อายุ รายได้ ประวัติซื้อ ฯลฯ) และต้องการทำนายว่าใครจะหยุดใช้บริการ (churn prediction).

ทีม data science ทดลองทั้งสองวิธี:

ลอง **deep neural network** ก่อน: accuracy ประมาณ 78% แต่ overfit ง่าย ต้องปรับ hyperparameter หนัก training ใช้เวลานาน และเมื่อผู้บริหารถามว่า "ทำไมถึงทำนายว่าลูกค้าคนนี้จะ churn" ตอบไม่ได้.

แล้วลอง **XGBoost** (classical ML): accuracy ประมาณ 82% train ใน 30 วินาที บอกได้เลยว่า feature ไหนสำคัญที่สุด (เช่น "ไม่ได้ใช้บริการ 60+ วัน" และ "มี complaint ในช่วง 3 เดือนล่าสุด") และผู้บริหารเข้าใจได้ทันที.

ในงานนี้ classical ML ชนะทั้งความแม่น ความเร็ว และความใช้งานได้จริง. เหตุผลชัดเจน:

1. Data 50,000 รายการไม่ใหญ่พอสำหรับ deep network ที่มี parameter เป็นล้าน
2. Data เป็น tabular ที่ feature มีความหมายชัดอยู่แล้ว ไม่มีอะไรที่ deep learning ต้อง "ค้นพบเอง"
3. ต้องการ interpretability เพื่ออธิบายให้ business stakeholder

นี่คือเหตุที่ Kaggle competition ที่เป็น tabular data ส่วนใหญ่ XGBoost และ LightGBM ยังครองอยู่จนปัจจุบัน.

ตรงนี้เป็นจุดที่คนทำงานกับ AI จริง ๆ ต้องเข้าใจ: deep learning ดีที่สุดในงาน perception และ generative แต่ในงาน structured data ที่ธุรกิจส่วนใหญ่มีอยู่ (ตาราง Excel, ฐานข้อมูลลูกค้า, ข้อมูล transaction) classical ML ยังเป็นทางเลือกที่เหมาะสมกว่าในหลายกรณี. เลือกเครื่องมือตาม task ไม่ใช่ตาม hype.

## Architecture ต่าง ๆ ใน deep learning

ภายใต้ร่มของ deep learning ยังมี architecture หลายแบบที่เหมาะกับ data ต่างกัน. ตรงนี้ไม่ต้องท่องจำ — แค่รู้ว่ามีอะไรบ้างและใช้กับอะไร:

**MLP (Multi-Layer Perceptron)** — โครงสร้างพื้นฐานที่สุด. layer ซ้อนกัน แต่ละ neuron เชื่อมกับทุก neuron ใน layer ถัดไป. เป็นรากฐานของทุก architecture. ทุกอย่างที่คุณเรียนในบทนี้เรื่อง "layer ซ้อนกัน" คือ MLP ในรูปแบบพื้นฐาน.

**CNN (Convolutional Neural Network)** — สำหรับรูปภาพและข้อมูลที่มี spatial structure. แทนที่จะเชื่อมทุก pixel กับทุก neuron ใช้ convolution ที่ดูแค่ local region ขนาดเล็ก แล้ว slide ไปทั่วรูป. ทำให้ประหยัด parameter และจับ pattern ที่เหมือนกันในที่ต่างกันของรูปได้. AlexNet ที่เปลี่ยนโลกในปี 2012 คือ CNN.

**RNN / LSTM** — สำหรับ sequence data ที่ลำดับสำคัญ เช่น ข้อความ เสียง. เคยเป็น default ของ NLP ก่อน transformer. ปัญหาคือ "ความจำระยะสั้น" — ประมวลผลทีละคำเรียงกัน พอถึงคำที่ 200 context ที่ต้นประโยคอาจ "จางไป" แล้ว.

**Transformer** — architecture ที่ "ชนะทุกอย่าง" ตั้งแต่ปี 2017. ใช้ attention mechanism ที่มองทุกคำในประโยคพร้อมกัน ไม่ใช่เรียงลำดับ — ทำให้จับ relationship ระหว่างคำที่อยู่ไกล ๆ กันได้ดีกว่า RNN มาก. นี่คือ architecture ของ ChatGPT, Claude, Gemini ทุกตัว.

**Diffusion model** — สำหรับ image generation. เรียนรู้วิธี "ค่อย ๆ เอา noise ออก" จากรูป. เป็นแกนของ Stable Diffusion, DALL-E, Midjourney. แนวคิดต่างจาก CNN และ Transformer แต่ยังอยู่ในกลุ่ม deep learning.

ทุก architecture เหล่านี้คือ deep learning ทั้งหมด — ต่างกันที่วิธี connect layer และวิธีประมวลผล data ประเภทต่างกัน.

ไม่ต้องท่องจำรายละเอียดของแต่ละ architecture ตอนนี้. สิ่งที่ควรจำในระดับนี้คือ: "deep learning ไม่ใช่แค่สิ่งเดียว — มันเป็น family ของ approach ที่เหมาะกับ data ต่างกัน". เมื่อได้ยินชื่อ CNN หรือ Transformer คุณจะรู้ว่ามันอยู่ในตระกูล deep learning ทั้งคู่ แค่เหมาะกับงานต่างกัน.

ลองเปรียบกับเครื่องมือช่าง: ค้อน ไขควง เลื่อย เจาะ — ทั้งหมดนั้นคือ "เครื่องมือช่าง" แต่ใช้กับงานต่างกัน. คนที่รู้แค่ว่า "ต้องใช้เครื่องมือช่าง" แต่ไม่รู้ว่าอันไหนเหมาะกับงานไหน จะทำงานไม่ได้. CNN เหมือนเจาะเหมาะกับงานที่ต้องการ "เจาะ" pattern ใน spatial data. Transformer เหมือนเลื่อยที่ "ตัด" ผ่าน sequence ยาว ๆ และดู relationship ระหว่างส่วนต่าง ๆ.

analogy นี้ใช้ได้ที่ว่า "เลือกเครื่องมือตามงาน" — ส่วนที่ต่างคือเครื่องมือช่างจริง ๆ ทำงานต่างกันชัดเจน แต่ใน deep learning มีงานหลายอย่างที่ architecture หลาย ๆ แบบทำได้ดีพอ ๆ กัน การเลือกจึงต้องอาศัย experiment ด้วย.

ในปี 2026 Transformer กำลังขยายไปใช้กับงานที่เคยเป็นของ CNN ด้วย (Vision Transformer) — แสดงให้เห็นว่าเส้นแบ่งระหว่าง architecture ไม่ได้ตายตัว. บางที "เครื่องมือที่ดีพอ ๆ กัน" ก็ขึ้นอยู่กับว่า data มีมากแค่ไหน — CNN ทำงานได้ดีกว่า Transformer เมื่อ data น้อย แต่ Transformer แซงเมื่อ data มีมากพอ.

## ดูจากตัวอย่างจริง: deep learning ทำงานยังไงใน production

เพื่อให้เห็นภาพรวมชัดขึ้น ลองติดตาม 2 ตัวอย่างที่คนไทยน่าจะเจอได้ในชีวิตจริง.

### ตัวอย่างที่ 1 — ระบบ fraud detection ของธนาคาร

ธนาคารในไทยหลายแห่งใช้ ML (บางแห่งใช้ deep learning) ตรวจจับ transaction ผิดปกติ. ลองดูว่าวิธีทั้งสองต่างกันอย่างไร:

**วิธีเก่าก่อนมี ML:** วิศวกรเขียน rule เช่น "ถ้า transaction มากกว่า 50,000 บาทในต่างประเทศ ให้ block" หรือ "ถ้ามีการโอนเงินมากกว่า 3 ครั้งใน 1 ชั่วโมง ให้ flag". Rule พวกนี้เขียนโดยคนที่รู้ pattern ของ fraud ที่เคยเกิดขึ้น. ปัญหาคือ fraud เปลี่ยน pattern ตลอด — พอ block rule หนึ่งก็หาช่องอื่น.

**วิธี ML (classical):** ใช้ข้อมูล transaction ประวัติ label ไว้ว่าอันไหน fraud อันไหนปกติ train XGBoost ให้หา feature ที่แยกได้. โมเดลอาจค้นพบว่า "transaction ที่มาจาก device ใหม่ + เวลากลางดึก + ร้านที่ไม่เคยไป + จำนวนผิดปกติ" = สัญญาณ fraud. ไม่มีใคร "บอก" โมเดลว่าให้ดูสี่ variable นี้ — มันค้นพบเอง.

**วิธี deep learning:** ถ้า data มากพอ (transaction หลายสิบล้านรายการ) deep network สามารถจับ pattern ที่ซับซ้อนกว่า เช่น sequence ของ transaction ก่อนหน้า 30 วัน, การเปลี่ยนพฤติกรรมอย่างกะทันหัน, หรือ pattern ที่คล้ายกับ fraud case ที่เคยเกิดขึ้นแต่ในรูปแบบที่ต่างกันนิดหน่อย.

แต่ในงานนี้ธนาคารหลายแห่งยังเลือกใช้ classical ML ก่อน เพราะ data ต้องการ label, ต้องการ interpretability เพื่ออธิบายให้ลูกค้าได้ว่าทำไมถึง block, และ regulatory compliance ต้องการเหตุผล.

### ตัวอย่างที่ 2 — แอปตรวจโรคจากรูปภาพ

นี่คืองานที่ deep learning ชนะชัดเจน. สมมติโรงพยาบาลต้องการระบบ screen รูป X-ray ว่ามีสัญญาณ pneumonia หรือไม่.

**วิธีเก่า:** radiologist คนหนึ่งต้องนั่งดูรูปทีละรูป. ถ้าอยากทำ automation ด้วย classical ML ต้องหา radiologist มา "บอก" ว่า feature ของ pneumonia คืออะไรในรูป X-ray — แต่ผู้เชี่ยวชาญอธิบายได้ยากมาก เพราะมันเป็น visual intuition ที่สะสมมาจากประสบการณ์หลายปี ไม่ใช่ rule ที่ออกมาเป็นภาษา.

**วิธี deep learning:** ให้ CNN ดูรูป X-ray ที่มี label "มี pneumonia" กับ "ไม่มี" หลายแสนรูป. โมเดลค้นพบ pattern เองว่า pixel pattern ไหนสัมพันธ์กับ pneumonia. ผลที่ได้: โมเดล deep learning ที่ train ดี ๆ มีความแม่นในการ screen เทียบได้กับ radiologist ระดับกลาง-ดีในบางกรณี.

จุดที่น่าสังเกต: มนุษย์ไม่สามารถอธิบายได้ว่า "โมเดลดู pattern ไหน" ในรูป X-ray — มันเรียนรู้ feature ที่ไม่มีชื่อในตำราแพทย์. ซึ่งนั่นคือพลังของ representation learning และในเวลาเดียวกันก็คือความท้าทายของ interpretability ใน clinical setting.

ในปี 2026 มี deep learning model ที่ผ่าน FDA approval สำหรับการ screen มะเร็งบางชนิดในสหรัฐอเมริกาแล้ว. ยังไม่ได้ replace radiologist แต่ทำงานเป็น second opinion ที่ช่วยลด workload และจับ case ที่อาจ miss ได้. นี่คือตัวอย่างของ deep learning ที่ไม่ใช่แค่ hype แต่ deployed จริงในระบบสาธารณสุข.

## deep learning กับ LLM — เส้นที่ต่อกัน

เมื่อถึงบท LLM ที่จะมาถึง ทุกอย่างจะเริ่มเชื่อมกัน — และพื้นฐาน deep learning ในบทนี้คือกุญแจสำคัญ.

LLM คือ **transformer ขนาดมหึมา train บน text จำนวนมหาศาล** — transformer คือ deep learning architecture ที่ชนะทุกอย่างตั้งแต่ปี 2017. ดังนั้น LLM คือ deep learning โดยตรง.

เรื่องที่คุณเรียนในบทนี้ใช้ได้กับ LLM ทั้งหมด:

**Representation learning ในระดับภาษา** — ChatGPT ไม่ได้มีใครสอนว่า grammar ภาษาไทยคืออะไร ว่า subject คืออะไร object คืออะไร ว่า "แต่" เชื่อมสองประโยคอย่างไร. มันเรียน structure ของภาษาเองจาก text หลาย trillion token. feature ที่มันค้นพบไม่มีชื่อในตำราภาษาศาสตร์ แต่กลับให้ผลที่ดีกว่าวิธีเก่า.

**Hierarchical feature ใน text** — layer ต้น ๆ ของ ChatGPT เรียน pattern ระดับ token (คำ, subword). layer กลาง เรียน syntax. layer ลึก เรียน semantic และ world knowledge — ว่าปารีสอยู่ในฝรั่งเศส ว่าน้ำต้มเดือดที่ 100 องศา ว่าประธานาธิบดีสหรัฐฯ คือใคร ณ ตอน training.

**ปัจจัย 4 อย่างยังทำงานอยู่** — เหตุที่ ChatGPT ดีขึ้นจาก GPT-2 ไปเป็น GPT-4 ไปเป็น GPT-4o คือปัจจัยเดียวกันกับที่ทำให้ AlexNet ชนะในปี 2012 — data มากขึ้น, compute มากขึ้น (เพิ่มขึ้นหลายร้อยเท่า), algorithm ดีขึ้น. ไม่มี "magic" — มีแค่ scaling.

ลองดูตัวเลขให้เห็นภาพ: GPT-2 ที่ OpenAI ปล่อยในปี 2019 มี 1.5 พันล้าน parameter, train บน text 40 GB. GPT-3 ในปี 2020 มี 175 พันล้าน parameter, train บน text 570 GB. GPT-4 ในปี 2023 ใหญ่กว่านั้นอีกมาก (OpenAI ไม่เปิดเผยตัวเลข). ทุก version ที่ดีขึ้นคือการเพิ่ม data + compute ในสัดส่วนที่เหมาะสม — ไม่ใช่การคิด "idea ใหม่" ที่แตกต่างจากรุ่นก่อน.

นี่คือเหตุที่ AI ในปี 2026 ยังดีขึ้นเรื่อย ๆ โดยไม่ต้องรอ "breakthrough ใหม่" — แค่ scale ปัจจัยเดิมให้ใหญ่ขึ้น. และนั่นก็คือเหตุที่ค่าใช้จ่ายในการ train model ระดับ frontier ปีนี้สูงกว่า budget งบวิจัยของมหาวิทยาลัยทั้งประเทศในบางกรณี.

> 💡 **Key insight:** ChatGPT ไม่ได้เกิดขึ้นจากความคิดใหม่ที่แปลกประหลาด — มันเป็นผลลัพธ์ตามธรรมชาติของ deep learning ที่ scale ขึ้นด้วยปัจจัยเดิมสี่ข้อ. ทุก concept ที่คุณเรียนในบทนี้ ถ้าเอาไปบวกกับ data ระดับอินเทอร์เน็ต กับ compute ระดับ data center ก็ได้ LLM. ไม่มีขั้นตอนลึกลับอื่น.

## ความเชื่อผิด ๆ ที่ควรรู้

การเข้าใจ misconception พวกนี้สำคัญมากสำหรับคนที่ต้องตัดสินใจเรื่อง AI ในองค์กร หรือแค่อยากอ่านข่าว AI แล้วไม่หลงทาง.

**"deep learning เข้าใจข้อมูลเหมือนมนุษย์"**

ไม่ใช่. การที่ layer ระดับสูงตอบสนองกับ concept abstract เช่น "หน้าแมว" ไม่ได้แปลว่ามัน "เข้าใจ" ในแบบที่มนุษย์เข้าใจ. มันคือ statistical correlation — weight ที่ activate พร้อมกันเมื่อเจอ pixel pattern ที่มนุษย์เรียกว่า "หน้าแมว". ไม่มีความหมาย ไม่มีความรู้สึก ไม่มี consciousness. เปรียบเทียบนี้ใช้ได้ที่ว่า "ผลออกมาคล้ายกับที่มนุษย์จะทำ" — ส่วนที่ต่างคือกระบวนการข้างในไม่มีอะไรเหมือนกันเลย.

**"ยิ่ง layer ลึกยิ่งดีเสมอ"**

ไม่ใช่. network ที่ลึกเกินไปโดยไม่มีเทคนิคพิเศษ เช่น residual connection จะ train ไม่ได้เพราะ gradient หายก่อนถึง layer ต้น ๆ. ResNet ในปี 2015 พิสูจน์ว่า "ลึกขึ้นได้ถ้ามี trick ที่เหมาะ" — แต่ไม่ใช่ "ลึกไปเรื่อย ๆ ยิ่งดีขึ้นเรื่อย ๆ". ในทางปฏิบัติต้อง balance กับ data ที่มี, compute ที่ใช้ได้, และ task ที่ต้องการ.

**"deep learning เพิ่งถูกคิดค้นไม่นานมานี้"**

แนวคิดมีมาตั้งแต่ทศวรรษ 1960-1980. สิ่งที่ "ใหม่" ในปี 2012 คือเงื่อนไขที่สุกงอมพร้อมกัน — ไม่ใช่ idea ใหม่. เข้าใจเรื่องนี้ถูกต้องแล้ว คุณจะคาดการณ์ได้ดีขึ้นว่า "breakthrough ต่อไป" จะมาจากอะไร — มักจะมาจาก data ใหม่, compute ใหม่, หรือ algorithm ใหม่ที่แก้ปัญหา training — ไม่ใช่แค่ "คิด idea ใหม่".

**"deep learning และ AI หมายถึงสิ่งเดียวกัน"**

AI ใหญ่กว่า. มีทั้งระบบ rule-based, expert system, classical ML และ deep learning อยู่ในนั้น. ในข่าวปี 2026 เมื่อพูดถึง "AI" มักหมายถึง deep learning โดยนัย แต่นั่นไม่ได้แปลว่าทุก AI เป็น deep learning. และระบบที่ใช้ rule-based หรือ classical ML ยังมีอยู่จำนวนมากในธุรกิจจริง ๆ ที่ไม่เป็นข่าว.

**"AI ที่ใช้ deep learning รู้ทุกอย่างเพราะมันเรียนรู้จาก data ทั้งหมด"**

ไม่ใช่. โมเดลรู้เฉพาะสิ่งที่อยู่ใน training data ณ ช่วงเวลาที่ train. ChatGPT ที่ train จนถึงต้นปี 2024 ไม่รู้เหตุการณ์ที่เกิดขึ้นหลังจากนั้น. และแม้แต่สิ่งที่อยู่ใน training data ก็ไม่ได้หมายความว่าโมเดลจำได้ทุกอย่าง หรือจะเอามาใช้ได้ถูกต้องทุกครั้ง. deep learning เรียน pattern — ไม่ได้จำข้อเท็จจริงแบบ database.

**"โมเดลที่ train ด้วย deep learning จะ bias ตามข้อมูลที่ได้"**

นี่ถูก และเป็นเรื่องสำคัญมากในปี 2026. เพราะโมเดลเรียน pattern จาก data — ถ้า data มี bias (เช่น มีรูปแพทย์ที่เป็นผู้ชายมากกว่าผู้หญิง) โมเดลก็จะ "คิด" ว่าแพทย์คือผู้ชาย. นี่เป็นเหตุที่วงการ AI มีสาขา Responsible AI และ Fairness ที่ทำงานกับปัญหานี้โดยตรง. deep learning ทรงพลังมาก แต่มันสะท้อน data ที่ได้ — ไม่ใช่ความจริงที่เป็น objective.

> 💡 **Key insight:** misconception ที่อันตรายที่สุดไม่ใช่เรื่องเทคนิค แต่คือการคิดว่า "AI บอกมา ต้องถูก". deep learning เรียนจาก data ที่มนุษย์สร้าง ซึ่งเต็มไปด้วย bias, ความผิดพลาด, และมุมมองเฉพาะ. เมื่อเข้าใจนี้ คุณจะใช้ AI อย่าง critical มากขึ้น — ไม่ใช่เชื่อทุกอย่าง แต่ตรวจสอบในสิ่งที่สำคัญ.

## ความรู้นี้เปลี่ยนวิธีที่คุณใช้ AI ได้อย่างไร

ถ้าอ่านมาถึงจุดนี้แล้วรู้สึกว่า "น่าสนใจ แต่ฉันจะเอาไปทำอะไรได้บ้าง" — นี่คือ 3 วิธีที่ความเข้าใจ deep learning ช่วยได้ในชีวิตจริง:

**1. เข้าใจว่าทำไม ChatGPT ผิดในบางเรื่อง**

เพราะ ChatGPT เรียน pattern จาก training data — ถ้าข้อเท็จจริงบางอย่างหายากใน training data หรือ pattern ที่ต้องการ "reasoning" แบบที่ไม่ใช่ pattern matching มันจะสุ่มเดาหรือ hallucinate ออกมา. เมื่อรู้แบบนี้ คุณจะ:
- ระวังเรื่องที่ต้องการความแม่นสูง (ตัวเลข วันที่ ชื่อ)
- รู้ว่าควรเช็ค source เองสำหรับข้อมูล critical
- ไม่ประหลาดใจเมื่อ AI ตอบผิดในเรื่องที่ดูเหมือนง่าย

**2. เข้าใจว่าทำไมบาง task ต้องการ context ที่ดี**

deep learning เรียนจาก context — ยิ่งให้ context ที่ชัดและตรงประเด็น โมเดลยิ่ง "จับ pattern" ที่ถูกได้ดีขึ้น. เมื่อรู้ว่ามันทำงานโดยการ match pattern นี่คือเหตุที่การเขียน prompt ที่ดีสำคัญ — คุณกำลัง "เซ็ต context" ให้โมเดลหา pattern ที่ถูกต้อง.

**3. ตั้งคำถามกับ vendor AI ได้ตรงขึ้น**

เมื่อบริษัท AI มาเสนอว่า "โมเดลของเราแม่น 95%" — คุณสามารถถามได้ว่า:
- "train กับ data แบบไหน size เท่าไหร่"
- "test กับ data ที่ต่างจาก training ไหม หรือ test กับ distribution เดิม"
- "ถ้าเจอ case ที่ไม่เคยอยู่ใน training โมเดลจะทำอะไร"
- "ถ้า data ในอนาคต drift ไปจาก training โมเดลจะยังทำงานได้ไหม"

คำถามพวกนี้เกิดจากการเข้าใจว่า deep learning เรียนจาก data — และ performance บน test set ที่คล้าย training data ไม่ได้การันตีว่าจะดีกับ real-world case. ในปี 2026 มีระบบ AI หลายตัวที่ "แม่น 99%" ใน lab แต่พัง dramatically ใน production เพราะ data จริงต่างจาก training data มากกว่าที่คาด.

## คำถามที่ได้ยินบ่อยเกี่ยวกับ deep learning

**Q: ถ้าจะใช้ deep learning ต้องเป็นโปรแกรมเมอร์ไหม?**

ขึ้นอยู่กับว่า "ใช้" แบบไหน. ถ้า "ใช้" หมายถึงพิมพ์ใน ChatGPT — ไม่ต้องเขียนโค้ดเลย. ถ้า "ใช้" หมายถึงนำ deep learning ไป solve ปัญหาธุรกิจของบริษัท — ต้องการคนที่มีทักษะเขียนโค้ดและเข้าใจ framework เช่น PyTorch หรือ TensorFlow. แต่ถ้า "ใช้" หมายถึงเป็นผู้จัดการที่ต้องตัดสินใจว่าจะลงทุนกับ deep learning หรือเปล่า — เข้าใจ concept ที่อยู่ในบทนี้ก็เพียงพอแล้ว.

**Q: deep learning กับ generative AI คืออะไรเดียวกันไหม?**

ไม่ใช่อย่างเดียวกัน แต่ generative AI เป็น subset ของ deep learning. Generative AI (เช่น ChatGPT สร้างข้อความ, Midjourney สร้างรูป) ใช้ deep learning architecture เป็นพื้นฐาน. deep learning ยังครอบคลุมงาน discriminative ด้วย เช่น "รูปนี้เป็นแมวหรือสุนัข" ซึ่งไม่ได้ "สร้าง" อะไร.

**Q: ทำไม deep learning ถึงต้องการ GPU ไม่ใช่ซีพียูปกติ?**

GPU ออกแบบมาเพื่อทำ matrix multiplication หลายพันครั้งพร้อมกัน — ซึ่งตรงกับ operation หลักของ neural network. ซีพียูทำได้ แต่ทีละน้อย GPU เป็นร้อยเป็นพันเท่า. ตัวอย่าง: การคูณเมทริกซ์ขนาด 1,000 x 1,000 บนซีพียูใช้เวลาเป็นนาที บน GPU เป็นมิลลิวินาที. deep learning ต้องทำ operation แบบนี้หลายพันล้านครั้งต่อ training step — GPU จึงเป็นสิ่งที่ขาดไม่ได้.

**Q: เราจะรู้ได้ยังไงว่า deep learning โมเดล "ดี" แค่ไหน?**

วัดผ่าน metric บน test data — ชุดข้อมูลที่โมเดลไม่เคยเห็นระหว่าง training. ถ้าโมเดลทำงานได้ดีกับ test data แสดงว่า generalize ได้. metric ที่ใช้ขึ้นอยู่กับ task เช่น accuracy สำหรับ classification, BLEU score สำหรับ translation, FID score สำหรับ image generation. ความสำเร็จของ AlexNet ในปี 2012 ก็วัดแบบนี้ — top-5 error rate บน test set ของ ImageNet.

**Q: deep learning จะถูกแทนที่ด้วยอะไรในอนาคต?**

ไม่มีใครรู้แน่ชัด. ในปี 2026 deep learning ยังเป็น dominant paradigm อยู่. มีงานวิจัยที่พยายามสร้าง approach ใหม่ที่ใช้ data น้อยกว่า, อธิบายตัวเองได้ดีกว่า, หรือ generalize ได้ดีกว่า — แต่ยังไม่มีอะไรที่ "เอาชนะ" deep learning ได้ในงาน perception และ language. ถ้าจะเปลี่ยน น่าจะเหมือนที่ deep learning เปลี่ยน classical ML — ค่อย ๆ แทนที่ในบางงาน ไม่ใช่หายไปพร้อมกันทั้งหมด.

**Q: ถ้าต้องการเรียนรู้เพิ่มเติมเกี่ยวกับ deep learning ควรเริ่มจากที่ไหน?**

บทถัดไปในหลักสูตรนี้คือ [LLM คืออะไร](/foundations/what-is-an-llm) ซึ่งจะนำทุกอย่างในบทนี้ไปประกอบเป็น ChatGPT ที่คุณใช้จริง. ถ้าต้องการ hands-on มากขึ้น Andrej Karpathy มี video series ชื่อ "Neural Networks: Zero to Hero" ที่สร้าง neural network จาก scratch ด้วย Python ซึ่งเห็นภาพ mechanism จริง ๆ ได้ดีมาก. สำหรับคนที่ต้องการ survey กว้าง ๆ Andrew Ng Deep Learning Specialization บน Coursera ครอบคลุม CNN, RNN, sequence model อย่างเป็นระบบ.

**Q: ทำไมบาง AI บริษัทไทยใช้ชื่อว่า "deep learning" บางบริษัทใช้ "machine learning" บางบริษัทใช้ "AI"?**

เพราะแต่ละคำมี marketing connotation ต่างกัน. "AI" ฟังดูน่าประทับใจที่สุด. "deep learning" ฟังดูเทคนิคที่สุด. "machine learning" ฟังดู practical. ในทางเทคนิค ถ้าผลิตภัณฑ์ใช้ neural network ที่มีหลาย layer คือ deep learning ซึ่งก็คือ ML ซึ่งก็คือ AI — ทั้งสามชื่อถูกทั้งนั้น. ปัญหาคือบางครั้งบริษัทใช้คำว่า "AI" กับ system ที่จริง ๆ แล้วเป็น rule-based ธรรมดา — ไม่มี ML ไม่มี deep learning เลย. การรู้ความแตกต่างทำให้คุณ "ถามต่อ" ได้ว่า "ใช้ algorithm อะไร train ด้วย data ประเภทไหน" แทนที่จะเชื่อ label ที่บริษัทติดมาให้.

## สรุป — สิ่งที่ควรจำจากบทนี้

deep learning คือ machine learning ที่ให้โมเดลเรียน feature เอง ไม่ต้องให้มนุษย์เตรียม feature. ทำได้โดยใช้ neural network ที่มี layer ซ้อนกันหลายชั้น — แต่ละชั้นสร้าง representation บนสิ่งที่ชั้นก่อนเรียนไว้.

ความลึก (depth) ทำให้เกิด hierarchical representation — จากง่ายไปซับซ้อน. จาก edge ไปเป็น texture ไปเป็น part ไปเป็น object ในรูปภาพ. จาก token ไปเป็น syntax ไปเป็น meaning ในภาษา. สิ่งที่ทำให้ลำดับนี้เกิดขึ้นได้คือความลึกของ network นั่นเอง.

deep learning ไม่ได้ดีกว่า classical ML ในทุกงาน — แต่มันเปิดประตูให้กับงานที่ classical ML ทำไม่ได้เลย เช่นสร้างรูปจาก text, ตอบคำถามในภาษาธรรมชาติ, แปลเสียงพูดเป็นข้อความ. งานเหล่านี้เป็นไปไม่ได้ในยุคก่อน deep learning ไม่ว่าจะมีคนเก่งขนาดไหนมาเขียน feature.

ปัจจัย 4 อย่างที่ทำให้ deep learning ทำงานได้จริงในปี 2012 — data, compute (GPU), algorithm, และ AlexNet เป็นจุดพิสูจน์ — ยังเป็นปัจจัยเดิมที่ขับเคลื่อน AI ในปี 2026 ทุกอย่างที่เห็น.

และทุกอย่างที่เกิดขึ้นใน AI ตั้งแต่ปี 2012 — image recognition, speech recognition, machine translation, และ LLM ทั้งหมด — ล้วนอยู่บนรากฐานของ deep learning.

ถ้าจะสรุปด้วยประโยคเดียว: deep learning คือการ shift ความรับผิดชอบในการค้นพบ feature จากมนุษย์ไปให้โมเดล — และ shift นั้นเปลี่ยนทุกอย่างที่เกี่ยวกับ AI ในปัจจุบัน.

ใน 10 ปีที่ผ่านมา deep learning เดินทางจาก AlexNet ในห้องแล็บ University of Toronto ไปสู่ ChatGPT ในมือของคนหลายร้อยล้านคนทั่วโลก. journey นั้นไม่ได้เกิดจาก magic — แต่จาก data ที่มากขึ้น compute ที่ถูกลง algorithm ที่ดีขึ้น และ architecture ที่ scale ได้. concept ทุกอย่างในบทนี้อธิบาย journey นั้นได้ครบ.

ถ้าจะมี "AI winter" (ยุคที่ AI ถูก abandon) อีกครั้งในอนาคต มันจะไม่ได้เกิดเพราะ idea หมด — แต่เพราะปัจจัยอย่างใดอย่างหนึ่งใน 4 ข้อชะลอตัว. ถ้าจะมี "AI spring" ต่อ ก็จะมาจากการที่ปัจจัยเหล่านั้นก้าวกระโดดอีกครั้ง. นี่คือ mental model ที่ช่วยให้คาดการณ์ทิศทาง AI ได้ดีกว่าการฟัง hype.

**สิ่งที่ควรเอาไปใช้ได้ทันที:**

ถ้าคุณทำงานในองค์กรที่กำลังคิดจะใช้ AI กับข้อมูลตาราง (ธุรกรรม ลูกค้า การเงิน) — ลองพิจารณา classical ML ก่อน. ถ้าคุณต้องการประมวลผลรูปภาพ เสียง หรือข้อความ — deep learning น่าจะเป็นจุดเริ่มต้น. ถ้าคุณใช้ ChatGPT อยู่ทุกวัน — ตอนนี้คุณรู้แล้วว่ามันเป็น deep learning ที่ scale ขึ้นด้วยปัจจัยเดียวกับที่ทำให้ AlexNet ชนะในปี 2012.

เรื่อง deep learning มี depth ที่ลงลึกได้อีกมาก — ตั้งแต่ mathematics ของ gradient descent, ไปจนถึง architecture design, scaling laws, และ emergent behavior ของโมเดลขนาดใหญ่. แต่ concept ที่คุณเรียนในบทนี้คือรากฐานที่ทุกอย่างต่อยอดจาก: representation learning, hierarchical feature, depth ที่หมายถึงจำนวน layer จริง ๆ, และปัจจัย 4 ข้อที่ทำให้มันทำงานได้จริงในปี 2012.

ความเข้าใจนี้จะทำให้คุณอ่านข่าว AI ได้ฉลาดขึ้น ตั้งคำถามกับ vendor AI ได้ตรงขึ้น และตัดสินใจเรื่อง AI ในการทำงานได้ดีขึ้น. และเมื่อคุณใช้ ChatGPT พรุ่งนี้ คุณจะรู้ว่าหลังหน้าจอนั้นมี deep network หลาย layer กำลังทำ hierarchical representation learning ด้วยปัจจัย 4 อย่างที่มารวมตัวกันในปี 2012 — และมันไม่ใช่ magic เลย.

**concept ที่คุณเรียนในบทนี้ เชื่อมกับบทอื่น ๆ อย่างไร:**

บท [Machine Learning คืออะไร](/foundations/what-is-machine-learning) อธิบาย paradigm กว้าง ๆ ของ ML ที่ deep learning อยู่ใน. เรื่อง [How Models Learn](/foundations/how-models-learn) อธิบาย gradient descent — กลไก training ที่ deep network ใช้เรียน feature ทุกชั้น. และบท [Neural Network คืออะไร](/foundations/what-is-a-neural-network) ที่อ่านมาแล้ว ก็คือโครงสร้างจริง ๆ ของสิ่งที่ทำให้ deep learning ทำงานได้.

ถ้าอ่านทั้งสี่บทนี้ครบ คุณจะมีภาพรวมที่แม่นมากของว่า AI ในปี 2026 ทำงานอย่างไรในระดับกลไก — ไม่ใช่แค่รู้ชื่อ แต่รู้ว่าเกิดอะไรขึ้นจริง. และนั่นคือความต่างระหว่างคนที่ "ใช้ AI" กับคนที่ "เข้าใจ AI" — ซึ่งในปี 2026 ความต่างนั้นมีค่ามากขึ้นทุกวัน.

ในบทถัดไป เราจะนำ deep learning ที่เพิ่งเข้าใจ ไปประกอบเป็น **LLM** — โมเดลที่อยู่หลัง ChatGPT และ Claude. คุณจะเห็นว่า deep network ที่ train บน text จำนวนมหาศาล กลายเป็น "เครื่องเดาคำถัดไป" ที่ทรงพลังได้อย่างไร และทำไมมันถึงดูเหมือนเข้าใจภาษาคน.

ถ้าตอนนี้คุณยังสงสัยว่า "แต่ deep network พวกนี้กลายเป็น ChatGPT ที่คุยกับเราได้ยังไงกันแน่" — นั่นคือ exactly คำถามที่บทถัดไปจะตอบ. ความสงสัยนั้นเป็นสัญญาณที่ดีว่าคุณเข้าใจบทนี้แล้วจริง ๆ และพร้อมสำหรับขั้นต่อไป.

เรื่อง deep learning มีความลึกที่ไม่มีที่สิ้นสุด — ตั้งแต่ mathematics ของ optimization, ไปจนถึงการอ่าน paper งานวิจัย, ถึง implementation จริงใน PyTorch. แต่ภาพรวมที่คุณได้จากบทนี้ — ว่า deep learning คืออะไร ทำงานอย่างไรในหลักการ ดีที่ไหน ไม่ดีที่ไหน และมาจากไหน — เป็นรากฐานที่ทุกความรู้ต่อยอดจาก. ถ้าจะจำแค่อย่างเดียวจากบทนี้ ให้จำว่า deep learning คือ machine learning ที่เรียน feature เอง ผ่าน layer ที่ลึก ด้วยกระบวนการ training ที่ทำซ้ำหลายล้านครั้ง — ทุกอย่างที่เหลือเป็นรายละเอียดบน concept นี้.

สิ่งที่ทำให้บทนี้สำคัญในบริบทของหลักสูตรนี้ทั้งหมด: deep learning คือรากฐานของ AI ทุกอย่างที่คุณใช้ในปี 2026. เมื่อเข้าใจว่ามันเรียน feature เองและทำงานผ่าน hierarchy ของ layer คุณจะเข้าใจ LLM, image AI, และทุก AI tool ที่จะมาในอนาคตได้ดีขึ้นโดยอัตโนมัติ — เพราะพวกมันทั้งหมดอยู่บน foundation เดียวกัน.
