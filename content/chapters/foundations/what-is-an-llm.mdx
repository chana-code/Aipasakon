# LLM คืออะไร — เปิดกล่องดำข้างหลัง ChatGPT

เปิด ChatGPT พิมพ์ "ช่วยสรุปอีเมลนี้ให้หน่อย" ได้คำตอบภาษาไทยอ่านรู้เรื่องภายใน 3 วินาที. รู้สึกเหมือนมีคนฉลาดนั่งอยู่อีกฝั่ง พร้อมช่วยเสมอ.

แต่ข้างในนั้นไม่มีคนนั่งอยู่. ไม่มีอะไร "คิด" ในแบบที่คนคิด.

สิ่งที่อยู่หลัง ChatGPT, Claude, Gemini และ AI ทุกตัวที่คุณคุยด้วย คือ **สมการทางคณิตศาสตร์ขนาดใหญ่มาก** ที่ทำสิ่งเดียวซ้ำแล้วซ้ำเล่า: รับข้อความเข้ามา คำนวณ แล้วบอกว่าคำถัดไปน่าจะเป็นอะไร. แค่นั้น.

แต่ "แค่นั้น" ไม่ได้แปลว่าง่าย. สมการนี้มีตัวเลขหลายพันล้านตัว รันอยู่บนคอมพิวเตอร์ระดับศูนย์ข้อมูลที่ใช้ไฟฟ้าเท่าเมืองเล็กทั้งเมือง และถูก "สอน" ด้วยข้อความจากอินเทอร์เน็ตเกือบทั้งหมด.

สมการนี้เรียกว่า **LLM** ย่อมาจาก **Large Language Model** -- โมเดลภาษาขนาดใหญ่. ทุกบริการ AI ที่คุณน่าจะรู้จัก -- ChatGPT ของ OpenAI, Claude ของ Anthropic, Gemini ของ Google -- ล้วนมี LLM เป็นแกนกลาง.

ถ้าคุณเคยสงสัยว่า "ทำไม ChatGPT ตอบบางอย่างได้ดีมาก แต่ตอบบางอย่างผิดอย่างน่าตกใจ" -- คำตอบอยู่ในกลไกของ LLM. และเมื่อคุณเข้าใจกลไกนี้ คุณจะเลิกประหลาดใจ.

ในบทนี้ เราจะเปิดกล่องดำทีละชั้น ตั้งแต่ "ข้อความกลายเป็นตัวเลขได้ยังไง" ไปจนถึง "สมการนี้ถูกสอนมายังไง" จนคุณเข้าใจกลไกข้างในจริง ๆ -- ไม่ใช่แค่รู้ว่า "มันเดาคำต่อไป" แต่รู้ว่า **เดายังไง** และ **เรียนรู้ที่จะเดาได้ยังไง**.

## ข้อความกลายเป็นตัวเลขได้ยังไง

ลองนึกถึงตอนที่คุณพิมพ์ข้อความลงใน ChatGPT. คุณเห็นตัวหนังสือ -- ตัวอักษรไทย คำภาษาอังกฤษ ตัวเลข เครื่องหมายวรรคตอน. แต่คอมพิวเตอร์ไม่เข้าใจตัวหนังสือ. มันเข้าใจแค่ตัวเลข.

ดังนั้นก่อนที่ LLM จะทำอะไรกับข้อความของคุณได้ มันต้องแปลงข้อความเป็นตัวเลขก่อน. และวิธีที่มันแปลง ส่งผลโดยตรงต่อคุณภาพของคำตอบที่คุณได้กลับมา.

กระบวนการนี้มีสองขั้น. ทั้งสองขั้นเกิดขึ้นก่อนที่ LLM จะเริ่มทำงานจริง -- เป็นขั้นเตรียม input.

### ขั้นที่ 1 -- ตัดข้อความเป็นชิ้น ๆ

ลองนึกว่าคุณมีประโยค "วันนี้อากาศดี" -- โมเดลจะตัดมันออกเป็นชิ้นเล็ก ๆ เช่น `["วัน", "นี้", "อากาศ", "ดี"]`. แต่ละชิ้นเรียกว่า **token**.

ภาษาไทยใช้ token มากกว่าภาษาอังกฤษประมาณ 2-3 เท่า ซึ่งส่งผลต่อทั้งค่าใช้จ่ายและคุณภาพ (ดูรายละเอียดและตารางเปรียบเทียบในบท [LLM ตอบคำถามยังไง](/using-ai/how-llms-respond))

### ขั้นที่ 2 -- แต่ละ token กลายเป็นรายการตัวเลข

ขั้นนี้สำคัญมาก: แต่ละ token ไม่ได้ถูกแทนด้วยตัวเลขตัวเดียว. มันถูกแปลงเป็น **รายการตัวเลขยาว ๆ** -- เช่น 4,096 ตัวเลข.

ทำไมต้อง 4,096 ตัว? ทำไมตัวเดียวไม่พอ?

ทำไมต้องตัวเลขหลายตัว? เพราะตัวเลขตัวเดียวจับความหมายไม่ได้.

ลองคิดแบบนี้: ถ้าจะอธิบาย "คน" คนหนึ่งด้วยตัวเลขตัวเดียว เช่น "คนหมายเลข 5234" -- ตัวเลขนั้นบอกอะไรไม่ได้เลย. จะเปรียบเทียบกับ "คนหมายเลข 8821" ก็ไม่รู้ว่าเหมือนหรือต่างกันยังไง.

แต่ถ้าอธิบายคนแต่ละคนด้วยตัวเลขหลายตัว:

- ส่วนสูง = 170
- น้ำหนัก = 65
- อายุ = 30
- ระดับความสุข = 0.8

ทีนี้คุณเปรียบเทียบได้แล้ว: "คนนี้สูงเท่ากัน หนักใกล้กัน อายุต่างกันเยอะ" -- ตัวเลขหลายตัวจับ "คุณสมบัติ" ได้หลายมิติ.

LLM ทำเหมือนกันกับคำ. แต่ละ token ถูกอธิบายด้วยตัวเลข 4,096 ตัว. แต่ละตัวเลขจับ "คุณสมบัติ" บางอย่างของคำนั้น -- ไม่ใช่คุณสมบัติที่มนุษย์ตั้งชื่อได้ง่าย ๆ อย่าง "เป็นคำนาม" หรือ "เป็นคำเกี่ยวกับอาหาร" แต่เป็นคุณสมบัติทางคณิตศาสตร์ที่โมเดลค้นพบเองตอน training.

ตัวเลขเหล่านี้เรียกว่า **embedding** -- เป็นการ "ฝัง" ความหมายของคำลงในตัวเลข.

(จำนวนตัวเลขต่อ token ต่างกันไปตาม LLM แต่ละรุ่น -- GPT-4 ใช้ราว 12,000 ตัว, Claude อาจใช้จำนวนอื่น. เราใช้ 4,096 เป็นตัวอย่างเพื่อให้เห็นภาพ.)

> **Key insight:** คำที่มีความหมายคล้ายกันจะมี embedding ที่คล้ายกัน. "แมว" กับ "สุนัข" จะมีตัวเลขใกล้กัน มากกว่า "แมว" กับ "รถยนต์". เหมือนคำทุกคำถูกวางไว้บน "แผนที่ความหมาย" -- คำที่เกี่ยวข้องกันจะอยู่ใกล้กันบนแผนที่. แค่แผนที่นี้ไม่ได้มี 2 มิติเหมือนแผนที่ทั่วไป -- มันมี 4,096 มิติ ซึ่งมนุษย์จินตนาการไม่ออก แต่คณิตศาสตร์คำนวณได้.

## เกิดอะไรขึ้นข้างในสมการ

ตอนนี้ข้อความของคุณกลายเป็นตัวเลขแล้ว. ลองจินตนาการ: ประโยค "ช่วยสรุปอีเมลนี้ให้หน่อย" ถูกตัดเป็น token แล้วแต่ละ token กลายเป็นรายการตัวเลข 4,096 ตัว. ทั้งประโยคจึงกลายเป็น "ตาราง" ของตัวเลข -- แต่ละแถวคือ token หนึ่ง แต่ละคอลัมน์คือมิติหนึ่งของ embedding.

ขั้นตอนถัดไปคือหัวใจของ LLM: ตัวเลขเหล่านี้ถูกส่งผ่าน **ชั้น (layer)** ของสมการซ้ำแล้วซ้ำเล่า.

### แต่ละชั้นทำอะไร

แต่ละชั้นทำสิ่งที่ฟังดูธรรมดามาก: **รับตัวเลขเข้ามา คูณด้วย parameter บวกกัน ส่งตัวเลขชุดใหม่ออกไป**.

ฟังดูง่ายเกินไป? จริง ๆ แล้วแค่นั้นจริง ๆ -- คูณกับบวก. แต่สิ่งที่ทำให้มันทรงพลังคือ:

- LLM มี **หลายสิบถึงหลายร้อยชั้น** ต่อกันเป็นแถว
- แต่ละชั้นมี parameter (ตัวคูณ) อยู่หลายล้านตัว
- ผลลัพธ์ของชั้นหนึ่งเป็น input ของชั้นถัดไป

นึกถึงโรงงานทำขนม:

- สถานีที่ 1: รับแป้งดิบ ผสมน้ำ ส่งต่อ
- สถานีที่ 2: รับแป้งผสมน้ำ นวด ส่งต่อ
- สถานีที่ 3: รับแป้งที่นวดแล้ว ปั้นรูป ส่งต่อ
- ...สิบกว่าสถานีต่อมา...
- สถานีสุดท้าย: ขนมสำเร็จรูปพร้อมใส่กล่อง

แต่ละสถานีทำงานง่าย ๆ แต่เมื่อต่อกันหลายสิบสถานี วัตถุดิบ (แป้ง) กลายเป็นของสำเร็จรูป (ขนม) ได้. ไม่มีสถานีไหนสถานีเดียวที่ "ทำขนม" -- ขนมเกิดจากการทำงานร่วมกันของทุกสถานี.

(เปรียบเทียบนี้ใช้ไม่ได้ตรงไหน: โรงงานจริงมีคนออกแบบว่าแต่ละสถานีทำอะไร. ใน LLM ไม่มีใครออกแบบว่าแต่ละชั้นจับ pattern อะไร -- โมเดล "ค้นพบ" เองตอน training ว่าแต่ละชั้นควรทำอะไรถึงจะเดาคำถัดไปได้แม่นที่สุด.)

### จาก pattern ง่ายสู่ pattern ซับซ้อน

LLM มักมี 80-120 ชั้น. นักวิจัยพบว่าชั้นต่าง ๆ จับ pattern ในระดับที่ต่างกัน:

- **ชั้นแรก ๆ** จับ pattern ง่าย ๆ: "คำนี้เป็นคำนาม" "คำสองคำนี้อยู่ติดกันบ่อย" "ตรงนี้น่าจะเป็นจุดจบประโยค"
- **ชั้นกลาง ๆ** จับ pattern ซับซ้อนขึ้น: "ประโยคนี้กำลังถามคำถาม" "บริบทกำลังพูดถึงอาหาร" "น้ำเสียงของข้อความนี้เป็นทางการ"
- **ชั้นท้าย ๆ** จับ pattern ระดับสูง: "จากทุกอย่างที่อ่านมา คำต่อไปควรเป็นคำที่เกี่ยวกับ___" "คำตอบของคำถามนี้น่าจะเป็นชื่อสถานที่"

ไม่มีใครเขียนกฎพวกนี้ให้. โมเดลค้นพบเองจาก training -- และนี่คือส่วนที่น่าทึ่งที่สุดของ LLM.

ถ้าคุณเคยได้ยินคำว่า **transformer** -- นั่นคือชื่อของสถาปัตยกรรม (แบบแปลนของสมการ) ที่ LLM ทุกตัวในปัจจุบันใช้. Transformer ถูกคิดค้นโดยทีมวิจัยของ Google ในปี 2017 และมันเปลี่ยนวงการ AI ไปตลอดกาล.

ชื่อเต็มของ paper คือ "Attention Is All You Need" -- เพราะกลไกหลักของ transformer คือการที่แต่ละ token "มองไปหา" token อื่น ๆ ทั้งหมดในประโยค เพื่อตัดสินใจว่าควรให้น้ำหนักกับคำไหนมากน้อยแค่ไหน.

ตัวอย่าง: ในประโยค "แมวที่นั่งอยู่บนโต๊ะกินปลา" เมื่อ LLM ต้องเดาว่าคำหลัง "กิน" คืออะไร มันต้อง "มองกลับ" ไปที่คำว่า "แมว" (ไม่ใช่ "โต๊ะ") เพื่อรู้ว่าใครเป็นคนกิน. กลไกนี้เรียกว่า **attention** -- มันช่วยให้ LLM เข้าใจความสัมพันธ์ระหว่างคำที่อยู่ห่างกันในประโยคได้. รายละเอียดเชิงลึกของ attention mechanism เป็นเรื่องของระดับที่สูงขึ้นไป แต่แค่รู้ว่ามันมีอยู่ก็พอสำหรับตอนนี้.

### ผลลัพธ์: probability ของทุกคำที่เป็นไปได้

หลังจากตัวเลขผ่านทุกชั้นแล้ว ชั้นสุดท้ายให้ผลลัพธ์ออกมาเป็น **รายการ probability ของทุก token ที่เป็นไปได้ในภาษา**. เช่น ถ้า input คือ "อากาศร้อน ควรดื่ม":

| token ถัดไป | probability |
|------------|-------------|
| น้ำ | 35% |
| ชา | 15% |
| กาแฟ | 12% |
| น้ำแข็ง | 8% |
| น้ำผลไม้ | 6% |
| ... | ... |
| (ทุก token อื่น ๆ อีกหลายหมื่นตัว) | (รวมกัน 24%) |

โมเดลเลือก token หนึ่งจากรายการนี้ (มีหลายวิธี -- เลือกตัวที่ probability สูงสุด สุ่มตาม probability หรือวิธีอื่น). แล้วใส่ token นั้นต่อท้าย input:

```
"อากาศร้อน ควรดื่ม" → เลือก "น้ำ"
→ input ใหม่: "อากาศร้อน ควรดื่มน้ำ" → เลือก "เย็น"
→ input ใหม่: "อากาศร้อน ควรดื่มน้ำเย็น" → เลือก "ๆ"
→ ...ต่อไปเรื่อย ๆ จนจบประโยค
```

นี่คือเหตุผลที่ output ของ ChatGPT ออกมา **ทีละคำแบบ stream** -- เพราะมันเดาทีละ token จริง ๆ. ทุกครั้งที่คุณเห็นข้อความค่อย ๆ ปรากฏบนหน้าจอ นั่นคือโมเดลกำลังวนลูป: ส่งตัวเลขผ่านทุกชั้น ได้ probability เลือก token ต่อท้าย วนใหม่.

หลายคนเข้าใจว่า AI "คิด" เหมือนมนุษย์ -- มีกระบวนการให้เหตุผล ชั่งน้ำหนัก ตัดสินใจ. จริง ๆ แล้วสิ่งที่เกิดขึ้นข้างในคือการคูณและบวกตัวเลขผ่านหลายร้อยชั้น แล้วได้ probability ออกมา. ผลลัพธ์ "ดูเหมือนคิด" เพราะ parameter ถูกปรับมาจนจับ pattern ภาษาได้ดีมาก -- แต่กระบวนการข้างในไม่เหมือนกับการให้เหตุผลของมนุษย์.

> **Key insight:** LLM ไม่ได้ "คิดคำตอบทั้งหมดแล้วค่อยพิมพ์ออกมา" -- มันสร้างข้อความทีละคำ จากซ้ายไปขวา โดยแต่ละคำขึ้นกับคำก่อนหน้าทั้งหมด. มันไม่มีแผนว่าจะพูดอะไรในท้ายประโยค ตอนที่มันเพิ่งเริ่มต้นประโยค.

## สมการนี้ถูกสอนมายังไง

ตอนนี้คุณรู้แล้วว่า LLM ทำงานยังไง: รับ text แปลงเป็นตัวเลข ส่งผ่านหลายชั้นของการคูณและบวก ได้ probability ของคำถัดไป.

แต่ตอนเริ่มต้น parameter ทุกตัว (ตัวเลขหลายพันล้านตัวที่อยู่ในทุกชั้น) ถูกตั้งค่าแบบ **สุ่ม**. สมการยังไม่รู้อะไรเลย. ถ้าใส่ "อากาศร้อน ควรดื่ม" เข้าไป มันอาจให้ probability สูงสุดกับคำว่า "ม้า" -- ไม่มีเหตุผลอะไรทั้งสิ้น.

กระบวนการที่เปลี่ยนสมการสุ่ม ๆ นี้ให้กลายเป็นสมการที่เดาคำถัดไปได้แม่น เรียกว่า **training**. กลไกข้างในเป็นแบบนี้:

### วงจร training -- ทำซ้ำหลายล้านล้านรอบ

**ขั้นที่ 1: ให้โจทย์.**
หยิบประโยคจากข้อมูลมาหนึ่งประโยค เช่น "แมวชอบกินปลา" แล้วปิดคำสุดท้ายไว้:

> โจทย์: "แมวชอบกิน___"
> คำตอบจริง: "ปลา"

**ขั้นที่ 2: ให้โมเดลเดา.**
ส่ง "แมวชอบกิน" เข้าไปในสมการ. สมการคำนวณ probability ของทุก token. สมมติว่ามันให้:
- "ข้าว" 22%
- "ปลา" 3%
- "ม้า" 0.5%

โมเดลให้ probability แค่ 3% กับคำตอบที่ถูก ("ปลา") -- แปลว่ามันยังเดาไม่ค่อยเก่ง.

**ขั้นที่ 3: วัดว่าผิดแค่ไหน.**
ระบบคำนวณค่าที่เรียกว่า **loss** -- ตัวเลขที่บอกว่า "โมเดลผิดมากแค่ไหนในรอบนี้." ถ้าโมเดลให้ probability สูงกับคำตอบที่ถูก loss จะต่ำ (ดี). ถ้าให้ probability ต่ำกับคำตอบที่ถูก loss จะสูง (ไม่ดี).

**ขั้นที่ 4-5: คำนวณว่าต้องปรับ parameter ตัวไหน แล้วปรับ.**

ตรงนี้คือกลไกที่เราเจาะลึกไปแล้วในบท [โมเดลเรียนรู้ยังไง](/foundations/how-models-learn) -- ทบทวนสั้น ๆ: แคลคูลัสคำนวณได้ว่าต้องขยับ parameter แต่ละตัวขึ้นหรือลงเพื่อให้ loss ลดลง (คำนวณพร้อมกันทุกตัวในรอบเดียว) แล้วขยับทุกตัว **นิดเดียว** ไปในทิศนั้น. กระบวนการนี้เรียกว่า **gradient descent** -- เหมือนคนที่ยืนบนภูเขาในหมอก ค่อย ๆ ก้าวไปทางที่พื้นลาดลงทีละก้าวจนถึงจุดต่ำสุด. ต่างกันแค่ LLM "เดิน" ในพื้นที่ที่มีหลายพันล้านมิติ มิติละ parameter หนึ่งตัว.

**แล้ววนซ้ำ.** หยิบประโยคใหม่ ให้เดา วัด loss คำนวณ gradient ปรับ parameter. ทำแบบนี้ **หลายล้านล้านรอบ** กับข้อความหลายล้านล้าน token.

### ผลลัพธ์: จากตัวเลขสุ่ม สู่ "ความรู้"

หลัง training หลายล้านล้านรอบ parameter ที่เริ่มจากตัวเลขสุ่ม ค่อย ๆ ถูกปรับจนทั้งระบบเดาคำถัดไปได้แม่นยำ.

ไม่มีใครเขียนกฎว่า "ถ้าเจอคำว่า 'แมวชอบกิน' ให้ตอบ 'ปลา'". ไม่มีใครป้อนฐานข้อมูลข้อเท็จจริง. โมเดล **ค้นพบ pattern เอง** จากการปรับตัวเลขหลายพันล้านตัว ทีละนิด ๆ หลายล้านล้านรอบ.

"ความรู้" ของ LLM จึงไม่ได้อยู่ในรูปแบบที่มนุษย์อ่านได้ -- มันฝังอยู่ใน parameter. ตัวเลข 175,000 ล้านตัวใน GPT-3 เรียงต่อกันเป็นไฟล์ขนาดหลายร้อย GB -- นั่นคือ "ทุกอย่างที่โมเดลรู้" ทั้งหมด. ไม่มีอะไรอื่น.

หลายคนคิดว่า LLM มี "ฐานข้อมูลคำตอบ" ซ่อนอยู่ข้างใน -- มี database แยก มีตาราง มีไฟล์ข้อมูล. จริง ๆ แล้วไม่มี. มันสร้างคำตอบขึ้นใหม่ทุกครั้งจากการคำนวณ. ทุกอย่างที่มันตอบออกมา เกิดจาก parameter ล้วน ๆ.

> **Key insight:** LLM ไม่ได้ "จำ" คำตอบเหมือนคุณจำเบอร์โทร. มันจับ pattern ว่า "หลังคำแบบนี้ คำแบบไหนมักจะตามมา" แล้วเดาจาก pattern นั้น. ถ้า pattern ชัดและมีใน training data เยอะ มันเดาถูก. ถ้า pattern ไม่ชัด มันก็สร้างคำตอบที่ "ดูเหมือนคำตอบ" ออกมา -- และนั่นคือ hallucination.

### ลองทำดู

เปิด ChatGPT หรือ Claude แล้วลองถามคำถาม 2 แบบนี้ แล้วสังเกตผลลัพธ์:

1. **คำถามที่มี pattern ชัด:** "เขียนอีเมลสุภาพขอเลื่อนประชุม" -- ลองดูว่าคำตอบดีแค่ไหน (pattern อีเมลมีเยอะมากใน training data)

2. **คำถามที่ pattern ไม่ชัด:** "ร้านอาหารที่ดีที่สุดในซอยข้างบ้านฉันชื่ออะไร" -- ดูว่ามันตอบยังไง (ข้อมูลนี้แทบไม่มีใน training data)

สังเกตความแตกต่าง: คำถามแรกมันตอบได้ดีมาก คำถามที่สองมันอาจแต่งชื่อร้านขึ้นมาอย่างมั่นใจ. ทั้งสองคำตอบเกิดจากกลไกเดียวกัน -- เดาคำถัดไปจาก pattern.

ลองทำอีกอัน: ถามว่า "อธิบาย quantum physics ให้เด็ก 5 ขวบเข้าใจ" แล้วถามซ้ำว่า "อธิบาย quantum physics ให้วิศวกรเข้าใจ" -- สังเกตว่าคำตอบเปลี่ยนระดับความยากอย่างชัดเจน.

ทำไม? เพราะใน training data มี pattern ของการอธิบายแบบง่ายกับแบบยากต่างกัน -- ชั้นกลาง ๆ ของ LLM จับได้ว่า "context นี้ต้องการคำตอบระดับไหน" แล้วชั้นท้าย ๆ ก็ปรับ probability ตาม. ไม่มีใครเขียนกฎนี้ให้ -- LLM ค้นพบเองว่า "หลังคำว่า 'เด็ก 5 ขวบ' ควรใช้คำง่าย ๆ" จาก pattern ใน training data.

## สามขั้นจาก "สมการเปล่า" สู่ ChatGPT

ตอนนี้คุณเข้าใจแล้วว่า LLM เรียนรู้จากวงจร training -- เดา วัดความผิด ปรับ parameter วนซ้ำ. แต่เส้นทางจาก "สมการสุ่มที่ไม่รู้อะไรเลย" ไปจนถึง "ChatGPT ที่คุณคุยด้วยทุกวัน" ไม่ได้จบแค่นั้น.

กระบวนการ training ที่อธิบายไป -- อ่านข้อความแล้วเดาคำถัดไป -- เป็นแค่ขั้นแรกจากสามขั้น. และแต่ละขั้นเปลี่ยนพฤติกรรมของโมเดลอย่างชัดเจน:

### ขั้นที่ 1: Pretraining -- อ่านอินเทอร์เน็ตทั้งหมด

นำข้อความจำนวนมหาศาลมาให้โมเดลอ่าน -- บทความ เว็บไซต์ หนังสือ โค้ด บทสนทนา -- รวมกันหลายล้านล้าน token. ใช้วงจร training ที่อธิบายไว้ข้างบน วนซ้ำจนโมเดลเดาคำถัดไปเก่ง.

ขั้นนี้แพงที่สุด -- ใช้ GPU หลายพันตัวรันต่อเนื่องเป็นเดือน. ต้นทุนต่อรอบของ LLM ระดับ GPT-4 หรือ Claude อยู่ที่ราว $100M+ (ประมาณ 3,500 ล้านบาท). นี่คือเหตุผลที่มีบริษัทไม่กี่แห่งในโลกที่สร้าง LLM ระดับแนวหน้าได้ -- ต้องมีทั้งเงินทุน ทีมวิจัย และ GPU จำนวนมหาศาล.

ข้อมูลที่ใช้ใน pretraining มาจากหลายแหล่ง: Common Crawl (snapshot ของเว็บทั้งหมด) หนังสือดิจิทัล Wikipedia โค้ดจาก GitHub บทสนทนาจากฟอรัมต่าง ๆ บทความวิจัย และอื่น ๆ อีกมาก. ข้อมูลเหล่านี้ถูกกรองและทำความสะอาดก่อนใช้ -- เอาเนื้อหาซ้ำออก เอาขยะออก เอาเนื้อหาอันตรายออก. คุณภาพของกระบวนการทำความสะอาดข้อมูลส่งผลต่อคุณภาพของ LLM โดยตรง.

ผลลัพธ์คือ **base model** -- สมการที่เดาคำถัดไปเก่งมาก แต่ยังไม่ใช่ chatbot. ถ้าคุณพิมพ์ "กรุงเทพมหานครเป็นเมืองหลวงของ" มันจะต่อให้ว่า "ประเทศไทย" ได้ถูก. แต่ถ้าพิมพ์ "ช่วยสรุปอีเมลนี้ให้หน่อย" มันอาจต่อด้วย "...แล้วช่วยเช็คตารางประชุมด้วย" -- เพราะมันแค่ต่อข้อความให้ดูสมเหตุสมผล ไม่ได้เข้าใจว่าคุณกำลัง "สั่งงาน" มัน.

### ขั้นที่ 2: Fine-tuning -- สอนให้ทำตามคำสั่ง

หลัง pretraining ทีมวิศวกรนำตัวอย่างคำถาม-คำตอบที่มนุษย์เขียนมาสอนเพิ่ม:

> **คำถาม:** "สรุปบทความนี้ให้เหลือ 3 bullet"
> **คำตอบตัวอย่าง:** (บทสรุป 3 ข้อที่มนุษย์เขียนอย่างดี)

โมเดลเรียนรู้ว่า "เมื่อเจอ input ที่มีลักษณะเป็นคำสั่ง ควรตอบแบบทำตามคำสั่ง ไม่ใช่ต่อข้อความไปเรื่อย ๆ." ใช้ข้อมูลน้อยกว่า pretraining มาก -- หลักหมื่นถึงแสนตัวอย่าง -- แต่เปลี่ยนพฤติกรรมอย่างชัดเจน.

กระบวนการนี้เรียกว่า **Supervised Fine-Tuning (SFT)** -- "supervised" เพราะมีมนุษย์กำกับว่าคำตอบที่ดีหน้าตาเป็นแบบไหน.

สิ่งที่น่าสนใจคือ: SFT ไม่ได้ "ลบ" ความรู้จาก pretraining. มันแค่ปรับ parameter เพิ่มเติมให้โมเดลรู้ว่า **เมื่อไหร่ควรใช้ความรู้นั้นยังไง**. เหมือนคนที่อ่านหนังสือเยอะ (pretraining) แล้วไปฝึกงาน (SFT) จนรู้ว่าจะเอาความรู้มาใช้ในสถานการณ์จริงยังไง.

(เปรียบเทียบนี้ใช้ไม่ได้ตรงไหน: คนอ่านหนังสือแล้วจำได้ชัดเจนว่าอ่านอะไร. LLM ไม่ได้ "จำ" อะไรชัดเจน -- ข้อมูลทั้งหมดถูกกลืนหายไปใน parameter จนแยกไม่ออกว่าความรู้ชิ้นไหนมาจากข้อความไหน.)

### ขั้นที่ 3: RLHF -- สอนให้ตอบในแบบที่คนพอใจ

หลัง fine-tuning โมเดลตอบคำถามเป็นแล้ว แต่ "ตอบเป็น" ไม่เท่ากับ "ตอบดี." บางทีมันตอบยาวเกินไป บางทีตอบแบบก้าวร้าว บางทีแต่งเรื่องมาตอบอย่างมั่นใจ.

ขั้นสุดท้ายใช้มนุษย์มาให้คะแนน: แสดงคำตอบ 2 แบบสำหรับคำถามเดียวกัน แล้วถามว่า "แบบไหนดีกว่า?"

ตัวอย่าง:

> **คำถาม:** "อธิบายการทำงานของเครื่องยนต์"
>
> **คำตอบ A:** "เครื่องยนต์สันดาปภายในทำงานโดยใช้วัฏจักรสี่จังหวะ ประกอบด้วย ดูด อัด จุด คาย..." (ถูกต้องแต่แห้ง ๆ)
>
> **คำตอบ B:** "ลองจินตนาการว่าเครื่องยนต์เหมือนห้องเล็ก ๆ ที่ทำ 4 อย่างวนซ้ำ: ดูดอากาศเข้า บีบให้แน่น จุดไฟระเบิด แล้วไล่ควันออก..." (ถูกต้องและอ่านง่าย)
>
> **มนุษย์เลือก:** B ดีกว่า

โมเดลเรียนจากคะแนนเหล่านี้ว่าตอบแบบไหนคนชอบ ตอบแบบไหนคนไม่ชอบ -- แล้วปรับ parameter (ด้วย gradient descent เช่นเดียวกัน) ให้ตอบในแบบที่ได้คะแนนดี.

กระบวนการนี้เรียกว่า **RLHF -- Reinforcement Learning from Human Feedback**.

ถ้าข้ามขั้นที่ 2 กับ 3 ไปเลย จะเกิดอะไรขึ้น? คุณจะได้ base model ที่เดาคำถัดไปเก่ง แต่ใช้งานจริงไม่ค่อยได้. ลองจินตนาการ: คุณถามว่า "ช่วยสรุปข่าวนี้ให้หน่อย" แล้ว base model ตอบกลับมาว่า "...โดยสำนักข่าวรายงานเพิ่มเติมว่า นายกรัฐมนตรี..." -- มันไม่ได้สรุปอะไรให้ มันแค่ต่อข้อความไปเรื่อย ๆ เหมือนเครื่องพิมพ์ดีดอัตโนมัติ. ไม่ได้ "ตอบคำถาม" อย่างที่คุณคุ้นเคยจาก ChatGPT.

ความ "ฉลาด" ที่คุณเห็นในผลิตภัณฑ์จริงเกิดจากทั้งสามขั้นทำงานร่วมกัน. Pretraining ให้ความรู้. SFT ให้ทักษะการตอบคำถาม. RLHF ให้ความสามารถในการตอบแบบที่คนพอใจ.

**สรุปสามขั้น:**

```
สมการสุ่ม (ไม่รู้อะไรเลย)
  → Pretraining: อ่านอินเทอร์เน็ต → เดาคำถัดไปเก่ง (base model)
  → Fine-tuning (SFT): เรียนจากตัวอย่าง → ตอบคำถามเป็น
  → RLHF: เรียนจากคะแนนมนุษย์ → ตอบดี ตอบปลอดภัย
  = chat model ที่คุณใช้ (ChatGPT, Claude, Gemini)
```

สังเกตว่าทั้ง 3 ขั้นใช้กลไกพื้นฐานเดียวกัน: คำนวณ loss แล้วปรับ parameter ด้วย gradient descent. ต่างกันแค่ **ข้อมูลที่ใช้** และ **สิ่งที่วัดว่า "ดี" หรือ "ไม่ดี"**. Pretraining วัดว่าเดาคำถัดไปตรงไหม. SFT วัดว่าตอบตรงกับตัวอย่างที่มนุษย์เขียนไหม. RLHF วัดว่ามนุษย์ให้คะแนนดีไหม.

## จาก LLM สู่ผลิตภัณฑ์ที่คุณใช้

ถ้าอ่านมาถึงตรงนี้ คุณเข้าใจแล้วว่า LLM ถูกสร้างมายังไง -- จากสมการสุ่ม ผ่าน pretraining, fine-tuning, RLHF จนกลายเป็น chat model. แต่ยังมีอีกชั้นหนึ่งที่สำคัญมากและคนส่วนใหญ่มองข้าม.

สิ่งที่หลายคนไม่รู้: **ChatGPT ไม่ใช่แค่ LLM.**

หลายคนใช้ ChatGPT แล้วคิดว่า ChatGPT = LLM. จริง ๆ แล้ว ChatGPT เป็นผลิตภัณฑ์สำเร็จรูปที่ OpenAI ประกอบขึ้นจากหลายส่วน:

| ส่วนประกอบ | คืออะไร | ทำหน้าที่อะไร |
|-----------|---------|--------------|
| **LLM** (เช่น GPT-4) | สมการที่ผ่าน training มาแล้ว | เดาคำถัดไป -- แกนกลางของทุกอย่าง |
| **System prompt** | คำสั่งที่ OpenAI เขียนไว้เบื้องหลัง | กำหนดว่าโมเดลควรตอบแบบไหน ห้ามตอบแบบไหน |
| **เครื่องมือ (tools)** | เครื่องคิดเลข ค้นเว็บ สร้างรูป รันโค้ด | ทำสิ่งที่ LLM ทำเองไม่ได้ |
| **Safety filters** | ระบบกรอง | บล็อกคำตอบที่ไม่เหมาะสม |
| **Chat interface** | หน้าเว็บ / แอป | ให้คุณพิมพ์คุยกับมันได้ |
| **Memory / context** | ระบบจัดการบทสนทนา | จำบริบทในการสนทนา |

Claude ของ Anthropic ก็โครงสร้างเดียวกัน -- LLM คนละตัว แต่หลักการประกอบผลิตภัณฑ์คล้ายกัน. Gemini ของ Google ก็เช่นกัน.

ลองนึกเปรียบเทียบกับรถยนต์: เครื่องยนต์เหมือน LLM -- เป็นแกนกลางที่สร้างพลัง. แต่รถยนต์ไม่ใช่แค่เครื่องยนต์. มันมีพวงมาลัย (UI) เบรก (safety filter) GPS (tools) และระบบอื่น ๆ อีกมาก. รถ Toyota กับ Honda ใช้เครื่องยนต์ต่างกัน แต่มีส่วนประกอบอื่น ๆ คล้ายกัน -- เหมือน ChatGPT กับ Claude ที่ใช้ LLM ต่างกัน แต่โครงสร้างผลิตภัณฑ์คล้ายกัน.

(เปรียบเทียบนี้ใช้ไม่ได้ตรงไหน: เครื่องยนต์รถยนต์ทำงานตามกฎฟิสิกส์ที่คาดเดาได้ 100%. LLM มีการสุ่มอยู่ในกระบวนการ ทำให้ผลลัพธ์ไม่เหมือนกันทุกครั้ง.)

**ทำไมเรื่องนี้สำคัญ?**

เพราะเมื่อคุณเข้าใจว่าผลิตภัณฑ์ประกอบจากหลายส่วน คุณจะตัดสินได้ถูกที่:

- เวลา ChatGPT ค้นข้อมูลจากเว็บให้ -- ไม่ใช่ "LLM รู้เอง" มันใช้เครื่องมือค้นเว็บที่ OpenAI ต่อให้
- เวลามันคำนวณเลขถูก -- ไม่ใช่ "LLM เก่งเลข" มันส่งโจทย์ให้เครื่องคิดเลข
- เวลามันสร้างรูป -- ไม่ใช่ LLM วาดรูป มันส่งคำสั่งไปให้โมเดลสร้างรูปอีกตัว (เช่น DALL-E)
- เวลา Claude ปฏิเสธคำถามบางอย่าง -- อาจไม่ใช่ "LLM ไม่รู้" อาจเป็น safety filter ที่ Anthropic ตั้งไว้
- เวลา ChatGPT กับ Claude ตอบต่างกัน -- ไม่ใช่แค่ "LLM ต่างกัน" system prompt, tools, safety filter ก็ต่างกันด้วย

ตัวอย่างจริงในปี 2026: เมื่อ ChatGPT "สร้างรูป" ให้คุณ มันไม่ได้ใช้ LLM วาดรูป. ChatGPT ส่งคำอธิบายไปให้โมเดลสร้างรูปชื่อ DALL-E แล้ว DALL-E สร้างรูปกลับมา. LLM ทำหน้าที่แค่ "แปล" คำขอของคุณให้เป็นคำสั่งที่ DALL-E เข้าใจ. นี่คือตัวอย่างของ tools ที่ถูกต่อเข้ากับ LLM.

อีกตัวอย่าง: เมื่อ ChatGPT "รันโค้ด Python" ให้คุณ มันไม่ได้จำลองการรันโค้ดในหัว. มันเขียนโค้ดออกมา (ใช้ความสามารถของ LLM) แล้วส่งโค้ดนั้นไปรันจริงบนเซิร์ฟเวอร์ (ใช้ tools) แล้วเอาผลลัพธ์กลับมาแสดง.

> **Key insight:** เวลาเพื่อนบอกว่า "ChatGPT เก่งมาก มันค้นข้อมูลให้ฉันได้ด้วย" -- จริง ๆ แล้ว LLM ไม่ได้ค้นอะไร. ผลิตภัณฑ์ต่อเครื่องมือค้นเว็บให้ แล้ว LLM อ่านผลการค้นหาแล้วสรุปให้คุณ. การแยก "โมเดล" กับ "ผลิตภัณฑ์" ออกจากกันคือทักษะพื้นฐานในการเข้าใจ AI.

## ทำอะไรได้ดี ทำอะไรพลาด -- อธิบายจากกลไก

ตอนนี้คุณรู้กลไกข้างในแล้ว -- ลองใช้ความรู้นั้นอธิบายจุดแข็งและจุดอ่อนของ LLM ดู. ทุกข้อที่จะพูดถึง อธิบายได้จากสิ่งที่เรียนมาข้างบน.

สิ่งที่น่าสนใจคือ ทุกจุดแข็งและจุดอ่อนสามารถอธิบายกลับไปที่กลไกเดียวกัน: next-token prediction จาก pattern ใน training data. ไม่ต้องท่องจำว่า "AI ทำอะไรได้ ทำอะไรไม่ได้" -- แค่ถามตัวเองว่า "งานนี้มี pattern ชัดใน training data ไหม?" ก็พอจะคาดเดาได้แล้ว.

### ตอบอีเมลเก่ง -- เพราะ pattern ชัด

LLM เคยอ่านอีเมลมาหลายล้านฉบับตอน pretraining. pattern ของอีเมลสุภาพชัดมาก: เปิดด้วยคำทักทาย ระบุเหตุผล เสนอทางเลือก ปิดด้วยคำลงท้ายสุภาพ. เมื่อ parameter ถูกปรับมาจาก pattern ซ้ำ ๆ จำนวนมหาศาล ชั้นต่าง ๆ ของสมการจับ pattern นี้ได้แม่น. probability ของคำที่ "ถูก" ในบริบทอีเมลจึงสูง ผลลัพธ์ออกมาดี.

หลักการเดียวกันใช้ได้กับงานอื่น ๆ ที่ LLM ทำได้ดี: สรุปเนื้อหา (pattern ของบทสรุปชัดเจน) แปลภาษา (มี pattern คู่ภาษาเยอะมากใน training data) เขียนโค้ด (โค้ดบน GitHub มีจำนวนมหาศาล). ทุกงานที่ LLM ทำได้ดี ล้วนเป็นงานที่มี pattern ชัดเจนและมีตัวอย่างเยอะใน training data.

### ตอบเลขผิด -- เพราะสมการนี้ไม่ได้คำนวณ

ถ้าคุณถาม "7 x 23 = ?" สมการไม่ได้ทำการคูณ. มันเห็น pattern "7 x 23 =" แล้วดูว่าตัวเลขอะไรมี probability สูงสุดจาก training data. ถ้าเคยเห็น "7 x 23 = 161" มาก่อน ก็ตอบถูก. ถ้าไม่เคยเห็น ก็เดาจาก pattern ที่ใกล้เคียง -- อาจได้ 163 หรือ 167 หรือเลขที่ "ดูเหมือนคำตอบ" แต่ผิด.

ลองคิดแบบนี้: คุณเคยเห็นสมการ "7 x 23 = 161" หลายร้อยครั้ง คุณก็จำได้. แต่ถ้ามีคนถามว่า "4,567 x 8,912 = ?" คุณก็ต้องหยิบเครื่องคิดเลขเหมือนกัน -- เพราะคุณไม่ได้จำคำตอบนี้ไว้. LLM ก็เหมือนกัน แต่มันไม่มีเครื่องคิดเลขในตัว.

นี่คือเหตุผลที่ผลิตภัณฑ์รุ่นใหม่ต่อเครื่องคิดเลขให้ (tools ที่พูดถึงในหัวข้อก่อน) แทนที่จะให้ LLM เดาเอง. เมื่อ ChatGPT คำนวณเลขถูก ในปี 2026 นั่นมักเป็นเพราะมันส่งโจทย์ไปให้โปรแกรมคำนวณจริง ๆ แล้วเอาคำตอบมาตอบคุณ.

### ตอบมั่วอย่างมั่นใจ (hallucination)

บางครั้งคุณถาม ChatGPT ว่า "หนังสือเรื่อง X ใครเขียน" -- มันตอบชื่อคนมาแบบมั่นใจ คุณไปเช็คแล้วพบว่าไม่มีคนชื่อนั้นเขียนหนังสือเล่มนั้นเลย. AI ตอบมั่ว -- แต่มันไม่ได้ตั้งใจโกหก.

ทำไมเกิดขึ้น? เพราะตอน training โมเดลถูกลงโทษ (loss สูง) เมื่อเดาคำถัดไปไม่ตรง -- แต่ไม่เคยถูกลงโทษเมื่อ **เนื้อหาไม่เป็นจริง**. ดังนั้นเมื่อเจอคำถามที่ไม่มีคำตอบชัดเจนใน training data มันก็ยังสร้างข้อความที่ "ดูเหมือนคำตอบ" ออกมา -- เพราะนั่นคือสิ่งที่ parameter ถูกปรับมาให้ทำ. ปรากฏการณ์นี้ในวงการ AI เรียกว่า **hallucination** (แปลตรงตัวว่า "ภาพหลอน"). เป็นสิ่งที่เกิดขึ้นได้กับทุก LLM ทุกตัว ไม่ว่าจะใหญ่แค่ไหน.

สิ่งที่ทำให้ hallucination อันตรายคือ: คุณแยกไม่ออกจากคำตอบที่ถูก. LLM ตอบมั่วด้วยน้ำเสียงเดียวกับตอบถูก -- เพราะมันไม่รู้ว่าอะไรจริงอะไรไม่จริง. มันแค่เดาคำที่น่าจะตามมา. ดังนั้นกฎสำคัญสำหรับทุกคนที่ใช้ AI: ข้อเท็จจริงที่สำคัญ ต้องเช็คจากแหล่งอื่นเสมอ.

### ไม่รู้ข่าวล่าสุด -- เพราะ parameter ถูก "แช่แข็ง"

หลัง training เสร็จ parameter ไม่เปลี่ยนอีก. LLM "หยุดอ่าน" ณ วันที่ตัดข้อมูล (**training cutoff**). เหตุการณ์หลังวันนั้นไม่เคยถูกใช้ใน gradient descent. parameter ไม่เคยถูกปรับให้จับ pattern ของเหตุการณ์นั้น. โมเดลจึงไม่รู้.

เวลา ChatGPT ตอบข่าวล่าสุดได้ นั่นเพราะ tools ค้นเว็บ ไม่ใช่เพราะ LLM รู้เอง. เรื่อง training cutoff จะสำคัญมากเวลาคุณถามข้อมูลที่เปลี่ยนแปลงบ่อย -- ราคาหุ้น ผลกีฬา ข่าวการเมือง -- AI อาจตอบด้วยข้อมูลเก่าโดยไม่บอกคุณ.

### ถามเหมือนกัน ได้คำตอบต่างกัน

หลายคนเข้าใจว่าถ้าถามคำถามเดียวกัน ควรได้คำตอบเดียวกันเสมอ. แต่ไม่ใช่.

จำได้ไหมว่าชั้นสุดท้ายให้ probability ออกมา เช่น "น้ำ" 35%, "ชา" 15%? ถ้าโมเดลสุ่มเลือกตาม probability ครั้งแรกอาจได้ "น้ำ" ครั้งที่สองอาจได้ "ชา" -- แล้วคำที่ตามมาก็เปลี่ยนไปด้วย ทำให้คำตอบทั้งหมดต่างกัน. เหมือนแม่น้ำที่แยกเป็นสองทางตรงจุดหนึ่ง -- ถ้าเลี้ยวซ้ายตรงนั้น จะไปถึงทะเลคนละฝั่งกับเลี้ยวขวา.

ระดับความสุ่มนี้ปรับได้ด้วยค่าที่เรียกว่า **temperature** -- ยิ่ง temperature สูง ยิ่งสุ่มมาก คำตอบยิ่งหลากหลาย. ยิ่ง temperature ต่ำ ยิ่งเลือกคำที่ probability สูงสุดเสมอ คำตอบยิ่งคงที่.

ในทางปฏิบัติ: งานที่ต้องการความแม่นยำ (เช่น สรุปข้อมูล แปลเอกสาร) ควรใช้ temperature ต่ำ. งานที่ต้องการความสร้างสรรค์ (เช่น เขียนบทกวี brainstorm ไอเดีย) ควรใช้ temperature สูง. ผลิตภัณฑ์อย่าง ChatGPT ตั้ง temperature ไว้ที่ระดับกลาง ๆ เป็นค่า default แต่ถ้าคุณใช้ API หรือเครื่องมือบางตัว คุณปรับค่านี้ได้เอง.

## ตามรอยคำตอบจากต้นจนจบ

ถึงตอนนี้คุณรู้ทุกส่วนประกอบแล้ว:

- **Tokenization** -- ข้อความถูกตัดเป็นชิ้นเล็ก ๆ
- **Embedding** -- แต่ละชิ้นกลายเป็นรายการตัวเลขที่จับความหมาย
- **Layers** -- ตัวเลขถูกส่งผ่านหลายสิบถึงหลายร้อยชั้นของการคูณและบวก
- **Probability** -- ชั้นสุดท้ายให้ probability ของทุก token ที่เป็นไปได้
- **Training** -- วงจร เดา วัด loss คำนวณ gradient ปรับ parameter วนซ้ำ
- **Three-step pipeline** -- Pretraining, SFT, RLHF
- **Product vs Model** -- ChatGPT = LLM + tools + safety + UI

เพื่อให้ทุกอย่างเชื่อมกัน ลองตามรอยว่าเกิดอะไรขึ้นตั้งแต่คุณพิมพ์คำถามจนได้คำตอบ.

คุณพิมพ์ใน ChatGPT: "เขียนอีเมลขอเลื่อนนัด"

สิ่งที่เกิดขึ้นข้างใน ทีละขั้น:

1. **Tokenize:** "เขียนอีเมลขอเลื่อนนัด" ถูกตัดเป็น tokens เช่น `["เขียน", "อีเมล", "ขอ", "เลื่อน", "นัด"]`
2. **Embed:** แต่ละ token ถูกแปลงเป็นรายการตัวเลข 4,096 ตัว
3. **ผ่านชั้น:** ตัวเลขเหล่านี้ถูกคูณและบวกผ่าน ~100 ชั้น. ชั้นแรก ๆ จับว่า "นี่เป็นคำสั่ง" ชั้นกลาง ๆ จับว่า "เกี่ยวกับอีเมลและการเลื่อนนัด" ชั้นท้าย ๆ จับว่า "ควรเริ่มด้วยคำว่า..."
4. **Probability:** ชั้นสุดท้ายให้ "เรียน" 45%, "สวัสดี" 20%, ... -- เลือก "เรียน"
5. **วนซ้ำ:** ใส่ "เรียน" ต่อท้าย ส่งเข้าสมการใหม่ ได้ "คุณ" ต่อท้าย ส่งเข้าใหม่...
6. **Output ออกมาทีละคำ** จนได้อีเมลสมบูรณ์

ทั้งหมดนี้เกิดขึ้นภายในไม่กี่วินาที -- เพราะ GPU หลายพันตัวคำนวณพร้อมกัน.

ลองนึกดูว่าสมการนี้ต้อง "วิ่ง" ผ่านทุกชั้น (80-120 ชั้น) ทุกครั้งที่เลือก token ใหม่หนึ่งตัว. อีเมลสั้น ๆ 200 คำ = สมการต้องวิ่งผ่านทุกชั้น 200 รอบ. นั่นคือการคูณและบวกตัวเลขหลายพันล้านตัว 200 ครั้ง -- ภายในไม่กี่วินาที. นี่คือเหตุผลที่ AI ต้องใช้ GPU ราคาแพงและศูนย์ข้อมูลขนาดใหญ่ -- และเป็นเหตุผลที่บริการ AI ระดับดีไม่ฟรี (หรือถ้าฟรีก็มีข้อจำกัด).

> **Key insight:** ทุกครั้งที่คุณกด "ส่ง" ใน ChatGPT สมการหลายพันล้าน parameter ต้อง "วิ่ง" ทั้งหมดเพื่อสร้างคำตอบออกมาทีละ token. คำตอบยาว 500 คำ = สมการวิ่ง ~500 รอบ. นี่คือเหตุผลที่คำตอบยาวใช้เวลานานกว่า -- ไม่ใช่เพราะ AI "คิด" นาน แต่เพราะมันต้องคำนวณมากรอบกว่า.

### อีกตัวอย่าง: ทำไมคำถามบางคำถามเป็นกับดัก

คุณถาม ChatGPT: "CEO ของบริษัท Fairdee คือใคร?"

1. ข้อความถูก tokenize และ embed ตามปกติ
2. ผ่านทุกชั้น -- ชั้นท้าย ๆ จับ pattern ว่า "คำถามแบบนี้ต้องการชื่อคน" และ "หลัง 'CEO ของ [บริษัท] คือ' ควรมีชื่อ"
3. **ปัญหา:** ถ้า "Fairdee" ไม่ค่อยมีใน training data, embedding ของ "Fairdee" อาจใกล้กับบริษัทอื่นที่ชื่อคล้าย probability กระจายไปที่ชื่อคนหลายคนที่ "ดูเหมือน CEO" โมเดลเลือกชื่อหนึ่งมาอย่างมั่นใจ
4. **ผลลัพธ์:** ชื่อที่ฟังดูน่าเชื่อ แต่อาจเป็นชื่อที่แต่งขึ้นมา -- hallucination

LLM ไม่มี "ฐานข้อมูล CEO" ข้างใน. มันมีแค่ parameter ที่จับ pattern ว่า "หลังคำว่า CEO ของ [บริษัท] คือ ควรมีชื่อคน." ข้อเท็จจริงเฉพาะ (specific facts) ที่ไม่ได้อยู่ใน training data จึงเป็นจุดอ่อนร้ายแรง.

นี่คือเหตุผลที่เทคนิคอย่าง **RAG** (Retrieval-Augmented Generation) ถูกพัฒนาขึ้น -- แทนที่จะให้ LLM เดาจาก parameter อย่างเดียว ก็ไปค้นข้อมูลจากฐานข้อมูลที่เชื่อถือได้มาก่อน แล้วส่งข้อมูลนั้นเข้ามาพร้อมกับคำถาม ให้ LLM อ่านแล้วตอบ. วิธีนี้ลด hallucination ได้มาก เพราะ LLM มี "หลักฐาน" ให้อ้างอิงแทนที่จะเดาจาก pattern อย่างเดียว.

ในองค์กรขนาดใหญ่ RAG กลายเป็นวิธีมาตรฐานในการใช้ AI กับข้อมูลภายใน -- เช่น ให้ AI ตอบคำถามเกี่ยวกับนโยบายบริษัทโดยค้นจากเอกสารจริงก่อนตอบ. เรื่องนี้จะลงรายละเอียดในบทเรื่องเครื่องมือของ AI.

### ลองทำดู -- ทดสอบจุดอ่อน

ลองทดสอบ "จุดอ่อน" ของ LLM ด้วยตัวคุณเอง. ใช้เวลาไม่ถึง 5 นาที:

**ทดสอบที่ 1: LLM ไม่ได้คำนวณจริง**
1. เปิด ChatGPT หรือ Claude
2. ถามว่า "1234 x 5678 เท่ากับเท่าไหร่"
3. จดคำตอบไว้ แล้วเช็คกับเครื่องคิดเลข (คำตอบจริงคือ 7,006,652)
4. สังเกต: ถ้ามันตอบถูก อาจเป็นเพราะ tools ช่วยคำนวณ ไม่ใช่ LLM คำนวณเอง. ลองถามเลขที่แปลกกว่านี้ เช่น "4567.89 x 123.456"

**ทดสอบที่ 2: temperature ทำให้คำตอบไม่คงที่**
1. ถามว่า "เขียนประโยคเปิดสำหรับนิยายสยองขวัญ" 3 ครั้ง
2. สังเกตว่าได้คำตอบต่างกันทุกครั้ง -- เพราะ temperature ทำให้การเลือก token มีการสุ่ม

**ทดสอบที่ 3: hallucination กับข้อมูลเฉพาะ**
1. ถามคำถามส่วนตัวที่ไม่มีทางมีใน training data เช่น "แม่ของฉันชื่ออะไร"
2. ดูว่ามันจัดการยังไง -- LLM ที่ดี (หลัง RLHF) จะบอกว่าไม่รู้ แทนที่จะแต่งชื่อขึ้นมา
3. ลองถามอีกแบบ: "CEO ของบริษัท [ใส่ชื่อบริษัทเล็ก ๆ ที่คุณรู้จัก] คือใคร" -- ดูว่ามันตอบถูกหรือแต่งชื่อขึ้นมา

**ทดสอบที่ 4: ดูว่า "product" กับ "model" ต่างกัน**
1. ถาม ChatGPT เรื่องข่าวล่าสุด -- ถ้ามันตอบได้ แปลว่ามันใช้ tools ค้นเว็บ (ไม่ใช่ LLM รู้เอง)
2. ถาม Claude เรื่องเดียวกัน -- ดูว่าคำตอบต่างกันไหม ต่างเพราะอะไร (LLM ต่างกัน? tools ต่างกัน? safety filter ต่างกัน?)

หลังทำครบ 4 ทดสอบ คุณจะมีประสบการณ์ตรงกับกลไกทุกตัวที่อ่านมาในบทนี้. ลองจดผลลัพธ์ไว้ -- จะเป็นประโยชน์เมื่ออ่านบทต่อ ๆ ไป.

ถ้ามีเวลา ลองทดสอบเพิ่มอีกข้อ: ถามคำถามเดียวกันกับ ChatGPT และ Claude แล้วเปรียบเทียบคำตอบ. คุณจะเห็นว่า LLM ต่างตัวให้คำตอบต่างกัน -- เพราะ training data ต่างกัน parameter ต่างกัน และ RLHF ต่างกัน.

คุณจะเห็นกลไกทั้งหมดที่อ่านมาทำงานจริงตรงหน้า:

- **คำนวณเลขผิด** -- เพราะ LLM ไม่ได้คำนวณจริง มันเดาจาก pattern
- **คำตอบเปลี่ยนทั้งที่ถามเหมือนเดิม** -- เพราะ temperature ทำให้มีการสุ่ม
- **ตอบคำถามส่วนตัวไม่ได้** -- เพราะข้อมูลนี้ไม่มีใน parameter

การทดลองแบบนี้คือวิธีที่ดีที่สุดในการ "สัมผัส" กลไกที่อ่านมาทั้งบท. ทฤษฎีจะกลายเป็นสัญชาตญาณเมื่อคุณเห็นมันทำงานจริง.

## สิ่งที่ LLM ไม่ได้เป็น -- แก้ความเข้าใจผิดที่พบบ่อย

ก่อนจะสรุป มาพูดถึงสิ่งที่ LLM **ไม่ได้เป็น** -- เพราะความเข้าใจผิดเหล่านี้ส่งผลต่อวิธีที่คุณใช้ AI โดยตรง.

**LLM ไม่ใช่ search engine.** Google Search ไปค้นหน้าเว็บที่มีอยู่จริงแล้วแสดงลิงก์ให้คุณ -- มันรู้ว่าข้อมูลมาจากไหน. LLM สร้างข้อความขึ้นมาใหม่จาก pattern ที่เรียนมา -- มันไม่มีทางบอกได้ว่า "ข้อมูลนี้มาจากเว็บไหน" เพราะข้อมูลทั้งหมดถูกกลืนหายไปใน parameter แล้ว. เมื่อ ChatGPT "ค้นเว็บ" ได้ นั่นเป็นเพราะ tools ที่ต่อให้ -- ตัว LLM เองไม่ได้ค้นอะไร.

**LLM ไม่ใช่ database.** ไม่มีตาราง ไม่มีไฟล์ ไม่มีฐานข้อมูลข้อเท็จจริงข้างใน. ทุกอย่างที่มัน "รู้" ฝังอยู่ใน parameter -- ตัวเลขหลายพันล้านตัวที่ถูกปรับมาจาก training. มันจึงตอบบางอย่างถูก บางอย่างผิด และไม่มีทางรู้ล่วงหน้าว่าข้อไหนจะผิด.

**LLM ไม่ได้ "คิด" เหมือนคน.** คนให้เหตุผลได้ วางแผนได้ รู้ว่าตัวเองไม่รู้อะไร. LLM คำนวณ probability ของคำถัดไป. ผลลัพธ์ดูเหมือน "คิด" เพราะ parameter ถูกปรับมาจนจับ pattern ภาษาได้ดีมาก -- แต่กระบวนการข้างในต่างกันโดยพื้นฐาน. นี่ไม่ได้แปลว่า LLM ไม่มีประโยชน์ -- มันมีประโยชน์มหาศาล -- แค่ต้องเข้าใจว่ามันมีประโยชน์ **ในแบบที่ต่างจากคน**.

**LLM ไม่ใช่ "สิ่งมีชีวิตดิจิทัล" ที่กำลังฉลาดขึ้นเรื่อย ๆ.** LLM แต่ละตัวถูก train ครั้งเดียว (หรือหลายรอบแต่มีจุดจบ) แล้ว parameter ก็ถูกแช่แข็ง. มันไม่ได้ "เรียนรู้" จากบทสนทนาที่มีกับคุณ (ยกเว้นจะมีระบบ memory ที่ผลิตภัณฑ์ต่อให้ ซึ่งเป็นคนละเรื่องกับ training). ChatGPT เวอร์ชันใหม่ที่ดีขึ้น เกิดจากการที่ OpenAI train โมเดลใหม่ ไม่ใช่เพราะโมเดลเดิม "โตขึ้น."

ความเข้าใจผิดเหล่านี้ไม่ใช่แค่เรื่องวิชาการ -- มันส่งผลต่อการตัดสินใจในงานจริง:

- ถ้าคิดว่า LLM เป็น search engine → คุณจะไว้ใจข้อเท็จจริงที่มันบอกโดยไม่เช็ค → อาจเอาข้อมูลผิดไปใช้ในงาน
- ถ้าคิดว่ามันมี database ข้างใน → คุณจะคาดหวังว่ามันตอบข้อมูลเฉพาะได้ถูกเสมอ → ผิดหวังเมื่อมัน hallucinate
- ถ้าคิดว่ามัน "คิด" เหมือนคน → คุณจะไว้ใจเหตุผลที่มันให้โดยไม่ตั้งคำถาม → อาจตัดสินใจผิดตามไปด้วย
- ถ้าคิดว่ามันเรียนรู้จากคุณตลอดเวลา → คุณจะคาดหวังว่ามันจำสิ่งที่คุยไว้ข้ามบทสนทนา → สับสนเมื่อมันลืม

การเข้าใจว่า LLM เป็นอะไรจริง ๆ ไม่ได้ทำให้มันน่าใช้น้อยลง -- ตรงข้ามเลย. เมื่อคุณรู้ว่ามันเก่งตรงไหนและอ่อนตรงไหน คุณจะใช้มันได้มีประสิทธิภาพมากขึ้น: ให้มันทำงานที่มันเก่ง เช็คซ้ำในงานที่มันอาจพลาด และไม่ไว้ใจมันในงานที่มันทำไม่ได้จริง ๆ.

> **Key insight:** ขนาดของ LLM (จำนวน parameter) ช่วยให้เดาแม่นขึ้น แต่ไม่ใช่ทุกอย่าง. งานวิจัยที่มีชื่อเสียง (Chinchilla, 2022) แสดงว่าโมเดลเล็กที่ train ด้วยข้อมูลดีและมากพอ อาจดีกว่าโมเดลใหญ่ที่ train ด้วยข้อมูลน้อย. คุณภาพของ training data สำคัญพอ ๆ กับขนาดโมเดล.

## สรุป -- LLM ในหนึ่งย่อหน้า

ถ้าอ่านมาถึงตรงนี้ คุณรู้เรื่อง LLM มากกว่าคนส่วนใหญ่ที่ใช้ ChatGPT ทุกวันแล้ว. มาสรุปทั้งหมดให้กระชับ.

LLM คือสมการทางคณิตศาสตร์ขนาดใหญ่ที่รับข้อความ (ซึ่งถูกตัดเป็น **token** แล้วแปลงเป็น **embedding** ที่เป็นตัวเลข) เข้ามา ส่งผ่านหลายสิบถึงหลายร้อยชั้น (**layers**) ของการคูณและบวก แล้วให้ **probability** ว่าคำถัดไปน่าจะเป็นอะไร. สมการนี้ไม่ได้ถูก "เขียน" ขึ้นมา -- มันถูก "สอน" ด้วยการอ่านข้อความหลายล้านล้าน token เดาคำถัดไป วัดว่าผิดแค่ไหน (**loss**) แล้วปรับ **parameter** ทีละนิดไปในทิศทางที่ทำให้ loss ลดลง (**gradient descent**) วนซ้ำหลายล้านล้านรอบ.

ผลิตภัณฑ์ที่คุณใช้อย่าง ChatGPT หรือ Claude ไม่ใช่แค่ LLM เปล่า ๆ -- แต่เป็น LLM ที่ผ่าน 3 ขั้นของ training (pretraining, fine-tuning, RLHF) แล้วถูกห่อหุ้มด้วยเครื่องมือ ระบบกรองความปลอดภัย และหน้าเว็บอีกหลายชั้น.

**จุดแข็ง** ของ LLM เกิดจากการเห็น pattern ชัดเจนจาก training data -- งานเขียน งานสรุป งานแปลภาษา งานเขียนโค้ด. **จุดอ่อน** เกิดจากข้อจำกัดของกลไก -- ไม่ได้คำนวณจริง ไม่มีฐานข้อมูลข้อเท็จจริง ไม่รู้อะไรหลัง training cutoff. และ **ผลิตภัณฑ์** ที่คุณใช้ไม่ใช่แค่ LLM -- มันเป็น LLM ที่ถูกห่อหุ้มด้วย tools, safety filters, system prompt และ UI อีกหลายชั้น.

เมื่อคุณเข้าใจกลไกนี้ คุณจะเข้าใจ **ทำไม** AI ทำบางอย่างได้ดีและพลาดบางอย่าง -- ไม่ใช่แค่ท่องจำว่า "มันตอบเลขผิด" แต่อธิบายได้ว่า "เพราะมันไม่ได้คำนวณ มันเดาจาก pattern."

และที่สำคัญกว่านั้น: เมื่อรู้ว่ากลไกข้างในทำงานยังไง คุณจะตัดสินได้ว่างานไหนควรให้ AI ทำ งานไหนควรเช็คซ้ำ และงานไหนไม่ควรไว้ใจ AI เลย. นี่คือพื้นฐานของการ "ใช้ AI เป็น" -- ซึ่งเป็นสิ่งที่เราจะลงรายละเอียดกันในบทต่อ ๆ ไป.

ถ้าตอนนี้มีคนถามคุณว่า "ChatGPT คิดเองได้ไหม" คุณมีคำตอบที่ทั้งถูกต้องทางเทคนิคและคนทั่วไปเข้าใจ: "มันไม่ได้คิดแบบคน. มันเป็นสมการที่เดาคำถัดไปจาก pattern ที่เรียนมา. ผลลัพธ์ดูเหมือนคิด เพราะ pattern ที่มันจับได้ซับซ้อนมาก -- แต่ข้างในเป็นแค่การคูณและบวกตัวเลข."

ถ้ามีคนถามว่า "ทำไม ChatGPT ตอบเลขผิด" คุณอธิบายได้: "เพราะมันไม่ได้คำนวณ มันเดาจาก pattern ที่เคยเห็น."

ถ้ามีคนถามว่า "ทำไมมันตอบมั่วแต่ดูมั่นใจ" คุณอธิบายได้: "เพราะตอน training มันถูกปรับให้เดาคำที่น่าจะตามมา ไม่ได้ถูกปรับให้เช็คว่าเนื้อหาจริงหรือเปล่า."

ถ้ามีคนถามว่า "ChatGPT กับ Claude ต่างกันยังไง" คุณอธิบายได้: "LLM ต่างกัน system prompt ต่างกัน tools ต่างกัน safety filter ต่างกัน -- ทุกชั้นส่งผลต่อคำตอบ."

ถ้ามีคนถามว่า "AI จะมาแย่งงานไหม" คุณตอบได้จากกลไก: "LLM เก่งงานที่มี pattern ชัดใน training data. งานที่ต้องตัดสินใจจากบริบทที่ซับซ้อน ต้องรับผิดชอบผลลัพธ์ หรือต้องใช้ข้อมูลที่ไม่มีใน training data -- ยังต้องการคน."

นี่คือพลังของการเข้าใจกลไก -- คุณไม่ต้องท่องจำ คุณอธิบายได้จากหลักการ. และทุกบทต่อจากนี้ใน AI ภาษาคน จะสร้างอยู่บนพื้นฐานที่คุณเพิ่งเรียนมาในบทนี้ -- ดังนั้นถ้ามีส่วนไหนที่ยังไม่ชัด ลองกลับไปอ่านส่วนนั้นอีกครั้ง.

ถ้าคุณเคยอ่านบทก่อนหน้าเรื่อง machine learning และ neural network จะเห็นว่า LLM คือการเอาหลักการทั้งหมดนั้นมารวมกัน -- neural network ที่มีหลายร้อยชั้น train ด้วยข้อมูลภาษาจำนวนมหาศาล จนเกิดเป็น "เครื่องเดาคำถัดไป" ที่ทรงพลังจนดูเหมือนมีความคิดเป็นของตัวเอง. ถ้ายังไม่ได้อ่านบทเรื่อง neural network ก็ไม่เป็นไร -- บทนี้ออกแบบมาให้อ่านเข้าใจได้ด้วยตัวเอง แต่ถ้าอยากเข้าใจลึกขึ้นว่า "สมการหลายชั้น" ทำงานยังไงในระดับพื้นฐาน บทนั้นจะช่วยได้.

สิ่งสำคัญที่สุดที่ควรจำจากบทนี้:

1. LLM **เดาคำถัดไป** -- ไม่ได้คิด ไม่ได้ค้น ไม่ได้คำนวณ
2. ข้อความถูกแปลงเป็น **ตัวเลข** (token → embedding) ก่อนเข้าสมการ
3. สมการเรียนรู้จากการ **ปรับ parameter ทีละนิด** หลายล้านล้านรอบ (gradient descent)
4. จากสมการสุ่มสู่ chatbot ต้องผ่าน **3 ขั้น**: pretraining, SFT, RLHF
5. **ChatGPT ไม่ใช่แค่ LLM** -- มันเป็นผลิตภัณฑ์ที่ห่อหุ้ม LLM ด้วย tools และระบบอื่น
6. จุดแข็งจุดอ่อนทุกข้อ **อธิบายได้จากกลไก** -- ไม่ต้องท่องจำ

---

ในบทถัดไป เราจะซูมออกมาดูภาพกว้าง -- ใครทำ LLM ตัวไหน แต่ละตัวต่างกันยังไง ราคาเท่าไหร่ และภูมิทัศน์ AI ในปี 2026 หน้าตาเป็นแบบไหน. OpenAI, Anthropic, Google, Meta, Mistral และอีกหลายรายกำลังแข่งกันอย่างดุเดือด -- แต่ละรายมีจุดแข็งต่างกัน และราคาก็ต่างกันอย่างมาก ตั้งแต่ฟรีไปจนถึงเดือนละหลายพันบาท.

การรู้ว่า LLM ตัวไหนเหมาะกับงานไหน เป็นทักษะที่คุ้มค่าที่สุดที่คนทำงานยุค 2026 ควรมี. เพราะคุณเข้าใจกลไกพื้นฐานแล้ว -- ว่า LLM เดาคำถัดไปจาก pattern -- คุณจะตัดสินได้เร็วว่า "งานนี้ LLM ทำได้ดีไหม" โดยไม่ต้องลองทุกตัว.
