# เริ่มต้นใช้งานจริง ติดตั้งและรันครั้งแรก

มีสักครั้งไหมที่คุณเปิดหน้าดาวน์โหลด Claude Code ไว้ อ่านว่าต้องพิมพ์คำสั่งใน terminal ดำ ๆ แล้วใจฝ่อ ปิดแท็บไป คุณเข้าใจแล้วว่าเครื่องมือพวกนี้ทำอะไรได้ แต่พอจะลองจริง ก็มีเสียงในหัวว่า "เดี๋ยวพิมพ์ผิดแล้วมันพังแน่ ๆ"

สิ่งที่ทำให้คนหยุดอยู่ตรงนั้นไม่ใช่การใช้งาน แต่คือการเปิดใช้ครั้งแรก พอติดตั้งและ login ผ่านไปแล้วครั้งเดียว ที่เหลือทั้งหมดคือการพิมพ์คุยกับมันเป็นภาษาคนธรรมดา ซึ่งคุณทำเป็นอยู่แล้วตั้งแต่ใช้ ChatGPT บทนี้จะพาคุณข้ามจุดนั้นทีละ step ช้า ๆ แบบไม่ต้องกลัว

## ก่อนเริ่ม คุณมีทางเลือกในการเปิดใช้อยู่หลายทาง

หลายคนคิดว่า Claude Code มีหน้าตาเดียว คือจอดำ ๆ ใน terminal ความจริงคือมันเปิดใช้ได้หลายทาง และแต่ละทางเหมาะกับคนคนละแบบ คุณเลือกได้ว่าจะเริ่มจากทางที่สบายใจที่สุดก่อน

ทางแรกคือ **Terminal CLI** คือการพิมพ์คำสั่ง `claude` ในหน้าต่าง terminal ของเครื่องคุณ คำว่า terminal คือโปรแกรมที่ให้คุณพิมพ์คำสั่งสั่งงานเครื่องโดยตรง แทนการคลิกปุ่ม (CLI ย่อมาจาก Command Line Interface แปลว่า "หน้าจอที่สั่งงานด้วยการพิมพ์บรรทัดคำสั่ง") ทางนี้คือทางหลัก เบาที่สุด เร็วที่สุด แต่หน้าตาเปลือยที่สุด เหมาะกับคนที่อยากได้ของจริงเต็ม ๆ และไม่กลัวจอที่ไม่มีปุ่มให้คลิก

ทางที่สองคือ **Desktop app** คือแอปที่มีหน้าต่างหน้าตาเหมือนโปรแกรมทั่วไปบนเครื่อง มีแถบด้านข้างให้คลิก มีช่องแก้ไฟล์ มีปุ่ม ทำงานเบื้องหลังเหมือน CLI ทุกอย่าง แต่ห่อด้วยหน้าตาที่คุ้นตากว่า เหมาะกับคนที่ไม่อยากแตะ terminal เลย แอปนี้มีให้โหลดบน macOS และ Windows ส่วน Linux มีรุ่นเบต้าสำหรับ Ubuntu/Debian

ทางที่สามคือ **เว็บ** ที่ `claude.ai/code` คือเปิดผ่าน browser ไม่ต้องติดตั้งอะไรในเครื่องเลย เหมาะเวลาคุณอยู่หน้าเครื่องที่ลงโปรแกรมไม่ได้ หรืออยากลองเร็ว ๆ ทางนี้ใช้บัญชี subscription ของคุณเสมอ (เดี๋ยวจะอธิบายเรื่องบัญชีต่อ)

ทางที่สี่คือ **ส่วนเสริมในโปรแกรมเขียนโค้ด** อย่าง VS Code หรือ JetBrains คือถ้าคุณใช้โปรแกรมพวกนี้อยู่แล้ว มีส่วนเสริม (extension) ให้ติดตั้งเพื่อเรียก Claude Code ขึ้นมาในหน้าต่างเดียวกับงานที่ทำอยู่ ทางนี้เหมาะกับคนสายเทคที่นั่งอยู่ในโปรแกรมเขียนโค้ดทั้งวันอยู่แล้ว

> 💡 **ใจความสำคัญ:** ทั้งสี่ทางนี้ "เครื่องยนต์ข้างใน" เป็นตัวเดียวกัน คือ harness ที่ห่อ model เดียวกัน ต่างกันที่หน้าตาที่คุณเห็นและวิธีเปิด (บางทางมีลูกเล่นเพิ่มเล็กน้อย แต่งานหลักทำได้เหมือนกัน) ถ้าคุณไม่เคยแตะ terminal มาก่อน เริ่มจาก Desktop app หรือเว็บก่อนได้เลย ไม่ใช่เรื่องน่าอาย แล้วค่อยขยับมา CLI ทีหลังเมื่อพร้อม

## บัญชีแบบไหนถึงจะใช้ได้ subscription กับ API key

ก่อนติดตั้งต้องเคลียร์เรื่องบัญชีก่อน เพราะนี่คือจุดที่คนสับสนกันมากที่สุด Claude Code ไม่ได้ใช้ฟรีกับบัญชี Claude.ai ฟรี คุณต้องมีบัญชีแบบใดแบบหนึ่งที่จ่ายเงิน และมีสองวิธีจ่ายเงินที่ต่างกันโดยสิ้นเชิง

วิธีแรกคือ login ด้วย **subscription** คือบัญชีรายเดือนแบบเหมาจ่าย ได้แก่แพ็กเกจ Pro หรือ Max (และแบบทีมอย่าง Team กับ Enterprise) คุณจ่ายเป็นรายเดือนคงที่ แล้วใช้ได้ตามโควตาของแพ็กเกจนั้น เหมือนซื้อบุฟเฟต์รายเดือน สำหรับคนทั่วไปนี่คือวิธีที่ง่ายกว่า login ครั้งเดียวจบ ไม่ต้องมานั่งคิดว่าแต่ละครั้งจ่ายเท่าไหร่

วิธีที่สองคือใช้ **API key** จาก Claude Console คือกุญแจรหัสยาว ๆ ที่ผูกกับบัญชีแบบเติมเครดิตล่วงหน้า แล้วคิดเงินตามจริงทุกครั้งที่ใช้ ตามจำนวน token ที่ประมวลผล เหมือนเติมเงินมือถือแล้วจ่ายตามที่โทรจริง วิธีนี้เหมาะกับองค์กรหรือทีมที่อยากคุมงบแบบจ่ายตามใช้และต้องการระบบติดตามค่าใช้จ่ายแยกเป็นรายการ

> 💡 **ใจความสำคัญ:** ถ้าคุณเป็นคนทั่วไปที่อยากลองใช้ ให้เลือก subscription (Pro หรือ Max) จ่ายรายเดือนคงที่ login ครั้งเดียวแล้วลืมเรื่องเงินไปได้เลย API key เก็บไว้ตอนคุณต้องการคุมต้นทุนแบบจ่ายตามใช้ หรือมีระบบที่ต้องเรียกใช้แบบอัตโนมัติ ซึ่งเป็นเรื่องของผู้ใช้ระดับสูงขึ้นไป

มีจุดหนึ่งที่ทำคนสะดุดบ่อย ถ้าเครื่องคุณเคยตั้งค่า environment variable ชื่อ `ANTHROPIC_API_KEY` ค้างไว้ มันจะถูกหยิบไปใช้แทน subscription ของคุณ ถ้าวันหนึ่ง login ด้วย subscription แล้วมันแปลก ๆ ให้ลบหรือยกเลิกตัวแปรนั้นแล้ว login ใหม่

ตรงนี้คุณแค่ต้องรู้ว่ามีสองวิธีจ่าย และเลือกให้ถูกกับตัวเอง ส่วนราคาแต่ละแพ็กเกจเป็นเงินบาท และวิธีคิดต้นทุนละเอียด ๆ อยู่ในบท [ราคาและการคิดเงิน](/pro-usage/token-optimization)

## ติดตั้งบน macOS แบบไม่ต้องกลัว terminal

ทีนี้มาลงมือจริง ถ้าคุณเลือกทาง Desktop app ข้ามไปอ่านหัวข้อถัดไปได้เลย ส่วนหัวข้อนี้คือทาง CLI ซึ่งฟังดูน่ากลัวกว่าที่เป็นจริงมาก

บนเครื่อง Mac เปิดโปรแกรมที่ชื่อ **Terminal** ก่อน วิธีเปิดคือกด `Command` ค้างแล้วเคาะ `Space` จะมีช่องค้นหาเด้งขึ้นมา พิมพ์คำว่า terminal แล้วกด Enter จะได้หน้าต่างสีเรียบ ๆ ที่มีเคอร์เซอร์กระพริบรออยู่ นี่แหละคือ terminal มันก็แค่ช่องให้พิมพ์คำสั่ง ไม่มีอะไรในนั้นจะระเบิด

จากนั้นพิมพ์คำสั่งนี้ลงไปแล้วกด Enter

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

คำสั่งนี้คือการบอกเครื่องว่า "ไปโหลดตัวติดตั้งจากเว็บทางการของ Claude มา แล้วรันให้หน่อย" คุณไม่ต้องเข้าใจทุกตัวอักษร แค่พิมพ์ให้ตรง (หรือก๊อปวาง) แล้วกด Enter เครื่องจะขึ้นข้อความวิ่ง ๆ สักพักแล้วบอกว่าติดตั้งเสร็จ ตัวติดตั้งแบบนี้จะคอยอัปเดตตัวเองเป็นเวอร์ชันใหม่ให้อัตโนมัติเบื้องหลัง คุณไม่ต้องมาคอยอัปเดตเอง

ถ้าคุณคุ้นกับโปรแกรมที่ชื่อ Homebrew อยู่แล้ว (เป็นตัวช่วยลงโปรแกรมบน Mac) จะใช้คำสั่ง `brew install --cask claude-code` แทนก็ได้ แต่ถ้าคุณไม่รู้ว่า Homebrew คืออะไร ไม่ต้องสนใจบรรทัดนี้ ใช้คำสั่ง `curl` ข้างบนพอ

## ติดตั้งบน Windows

บน Windows ขั้นตอนคล้ายกัน แต่มีรายละเอียดต้องระวังนิดหนึ่ง เพราะ Windows มีหน้าต่างพิมพ์คำสั่งอยู่สองแบบที่หน้าตาคล้ายกันแต่ใช้คำสั่งคนละชุด

แบบแรกชื่อ **PowerShell** ถ้าคุณเปิดอันนี้ บรรทัดบนสุดจะขึ้นต้นด้วย `PS C:\` ให้พิมพ์คำสั่งนี้

```powershell
irm https://claude.ai/install.ps1 | iex
```

แบบที่สองชื่อ **CMD** หรือ Command Prompt ถ้าเปิดอันนี้ บรรทัดจะขึ้น `C:\` เฉย ๆ ไม่มี `PS` นำหน้า ให้ใช้คำสั่งคนละแบบ

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

วิธีง่าย ๆ ที่จะรู้ว่าคุณอยู่หน้าต่างไหน คือดูตัวอักษรหน้าบรรทัด มี `PS` นำคือ PowerShell ไม่มีคือ CMD ถ้าพิมพ์ผิดหน้าต่าง เครื่องจะฟ้อง error ทันที ไม่ได้พังอะไร แค่สลับไปอีกหน้าต่างแล้วใช้คำสั่งที่ถูกแทน

มีของอีกชิ้นที่ควรลงบน Windows คือโปรแกรมชื่อ **Git for Windows** (โหลดได้ที่ git-scm.com) มันไม่ใช่ของบังคับ แต่มันมาพร้อม Git Bash ซึ่งทำให้ Claude Code ใช้เครื่องมือ Bash ได้ ถ้าไม่ลง มันจะถอยไปใช้ PowerShell แทน ซึ่งทำงานได้น้อยกว่า ถ้ายังไม่อยากยุ่ง ข้ามไปก่อนได้ ลงทีหลังก็ได้

ถ้าคุณอยากเลี่ยง terminal บน Windows ไปเลย ทาง Desktop app คือคำตอบ โหลดตัวติดตั้งจากเว็บทางการของ Claude เปิดไฟล์ ติดตั้งเหมือนโปรแกรมทั่วไป แล้วกดเปิด ครั้งแรกที่เปิดแท็บ Code บน Windows มันจะขอให้คุณลง Git for Windows ก่อนหนึ่งครั้ง ลงแล้วเปิดแอปใหม่ก็ใช้ได้

> 💡 **ใจความสำคัญ:** บน Windows กับดักที่เจอบ่อยที่สุดคือพิมพ์คำสั่ง PowerShell ในหน้าต่าง CMD หรือสลับกัน วิธีจำคือ มี `PS` นำหน้าบรรทัด ใช้คำสั่ง `irm` ไม่มี `PS` ใช้คำสั่ง `curl` แค่นี้พอ

### ลองทำดู เช็กว่าติดตั้งสำเร็จไหม

ก่อนจะไป login ลองพิสูจน์ก่อนว่าติดตั้งสำเร็จแล้วจริง เปิด terminal พิมพ์

```bash
claude --version
```

ถ้ามันขึ้นเลขเวอร์ชันออกมา แปลว่าติดตั้งสำเร็จเรียบร้อย ถ้ามันฟ้องว่า `command not found` หรือหาคำสั่งไม่เจอ แปลว่าติดตั้งยังไม่ครบ ลองรันคำสั่งติดตั้งซ้ำอีกรอบ หรือดูทางแก้ใน [ภาคผนวก B](/in-practice/troubleshooting)

อีกคำสั่งที่มีประโยชน์มากคือ

```bash
claude doctor
```

คำสั่งนี้เหมือนพาเครื่องไปหาหมอ มันจะตรวจสภาพการติดตั้งและการตั้งค่าของคุณทั้งหมด แล้วบอกว่าตรงไหนมีปัญหา ถ้าวันไหนรู้สึกว่าอะไรแปลก ๆ สั่ง `claude doctor` ดูก่อนเป็นอันดับแรก

## login และยืนยันตัวตนครั้งแรก

ติดตั้งเสร็จแล้ว ทีนี้ถึงตอน login ซึ่งง่ายกว่าที่คิดมาก ในหน้าต่าง terminal พิมพ์คำเดียว

```bash
claude
```

แล้วกด Enter ครั้งแรกที่รัน Claude Code จะเปิดหน้าต่าง browser ขึ้นมาให้คุณ login ด้วยบัญชี Claude ของคุณเอง คุณ login ในเว็บตามปกติเหมือน login เข้าเว็บทั่วไป พอเสร็จ มันจะวิ่งกลับมาที่ terminal เอง

ถ้า browser ไม่เด้งขึ้นมาเอง ในหน้าจอจะมีบอกให้กดปุ่ม `c` เพื่อก๊อปลิงก์ login แล้วเอาไปวางใน browser เอง และถ้าหลัง login เว็บโชว์รหัสชุดหนึ่งแทนที่จะวิ่งกลับ ก็ก๊อปรหัสนั้นมาวางในช่องที่ terminal ถามว่า `Paste code here if prompted` กรณีนี้เจอบ่อยเวลาทำงานผ่านเครื่องระยะไกล แต่กับเครื่องส่วนตัวทั่วไปมักวิ่งกลับให้เองอัตโนมัติ

login เสร็จครั้งเดียว เครื่องจะจำรหัสยืนยันตัวตนของคุณไว้ ครั้งต่อ ๆ ไปเปิด `claude` ก็ใช้ได้เลยไม่ต้อง login ซ้ำ ถ้าอยากสลับบัญชีหรือ login ใหม่ ให้พิมพ์ `/login` ระหว่างที่ Claude Code เปิดอยู่ ถ้าอยากออกจากบัญชีใช้ `/logout`

รหัสยืนยันตัวตนนี้ถูกเก็บอย่างปลอดภัยในเครื่องคุณ บน Mac เก็บใน Keychain ที่เข้ารหัสไว้ บน Windows เก็บในไฟล์ที่ล็อกให้เฉพาะบัญชีผู้ใช้ของคุณเข้าถึงได้ คุณไม่ต้องจัดการอะไรเอง

## session แรกของคุณ เปิดโปรเจกต์ พิมพ์สิ่งที่อยากได้ ดูมันทำงาน

ลองเดินทีละ step ช้า ๆ

**step 1 เลือกโฟลเดอร์งาน** Claude Code ทำงานกับ "โฟลเดอร์" หนึ่งโฟลเดอร์ที่คุณเปิดมันขึ้นมา มันจะเห็นและแก้ไฟล์ในโฟลเดอร์นั้นได้ ถ้าคุณยังไม่มีโปรเจกต์อะไร สร้างโฟลเดอร์เปล่า ๆ ขึ้นมาสักอันเพื่อทดลองก่อนก็ได้ คุณย้ายเข้าไปในโฟลเดอร์ที่อยากทำงานด้วยคำสั่ง `cd` ตามด้วยที่อยู่โฟลเดอร์ เช่น

```bash
cd /path/to/your/project
claude
```

คำว่า `cd` ย่อมาจาก change directory แปลว่า "ย้ายเข้าไปในโฟลเดอร์นั้น" ส่วน `/path/to/your/project` คือที่อยู่ของโฟลเดอร์คุณ (ถ้าใช้ Desktop app ตรงนี้ง่ายกว่า คุณแค่คลิกเลือกโฟลเดอร์จากหน้าต่าง ไม่ต้องพิมพ์ที่อยู่เอง)

**step 2 รัน** พิมพ์ `claude` กด Enter คุณจะเห็นหน้าจอ Claude Code ขึ้นมา ด้านบนบอกเวอร์ชัน บอก model ที่ใช้ และบอกว่าตอนนี้อยู่ในโฟลเดอร์ไหน ด้านล่างมีช่องว่างให้คุณพิมพ์ ถ้าอยากดูว่าพิมพ์คำสั่งอะไรได้บ้าง พิมพ์ `/help`

**step 3 พิมพ์สิ่งที่อยากได้เป็นภาษาคน** นี่คือหัวใจ คุณไม่ต้องเขียนคำสั่งลึกลับอะไร พิมพ์เหมือนคุยกับเพื่อนที่เก่งเรื่องนี้ ลองเริ่มจากคำถามทำความรู้จักโปรเจกต์ก่อน เช่น

```text
โปรเจกต์นี้ทำอะไร
```

Claude Code จะไปอ่านไฟล์ในโฟลเดอร์ของคุณเองตามที่มันต้องการ แล้วสรุปกลับมา คุณไม่ต้องคอยป้อนไฟล์ให้มันทีละไฟล์ จุดนี้ต่างจาก ChatGPT บนเว็บที่คุณต้องก๊อปข้อความไปแปะ ตรงนี้มันเอื้อมไปอ่านไฟล์ในเครื่องเองได้ (เพราะมันคือ harness ที่มีเครื่องมืออ่านไฟล์ ถ้าลืมว่า harness คือตัวห่อ model ที่ให้มันลงมือทำงานได้ ย้อนอ่าน [ส่วนที่ไม่มีใครพูดถึง](/products/the-harness))

**step 4 ลองให้มันแก้อะไรสักอย่าง** ลองสั่งงานที่มีผลกับไฟล์จริง เช่น

```text
เพิ่มฟังก์ชัน hello world ลงในไฟล์หลักให้หน่อย
```

จุดสำคัญที่ต้องเห็นด้วยตาคือ Claude Code จะไม่แก้ไฟล์ของคุณเงียบ ๆ มันทำตามลำดับนี้

1. หาว่าควรแก้ไฟล์ไหน
2. โชว์ให้คุณดูก่อนว่าจะแก้อะไรบ้าง
3. ถามขออนุญาตคุณ
4. ค่อยลงมือแก้หลังคุณกดอนุญาต

คุณกดอนุมัติทีละครั้งได้ หรือจะเปิดโหมดให้มันทำต่อเนื่องในรอบนั้นก็ได้ การที่มันขออนุญาตก่อนทุกครั้งนี้คือกลไกความปลอดภัยที่สำคัญ รายละเอียดเรื่องโหมดอนุญาตต่าง ๆ อยู่ในบท [ความปลอดภัยและความเป็นส่วนตัว](/pro-usage/judgment-safety)

### ลองทำดู ออกคำสั่งแรกที่ปลอดภัยที่สุด

ถ้ายังกลัวว่ามันจะไปแก้ของพัง ให้เริ่มจากคำสั่งที่แค่ "อ่าน" ไม่ "แก้" สร้างโฟลเดอร์เปล่า ๆ ขึ้นมาอันหนึ่ง เปิด Claude Code ในนั้น แล้วพิมพ์

```text
อธิบายให้หน่อยว่าโฟลเดอร์นี้มีอะไรอยู่บ้าง และถ้าฉันอยากเริ่มทำเว็บง่าย ๆ ควรสร้างไฟล์อะไรก่อน
```

มันจะตอบโดยยังไม่แตะไฟล์อะไร คุณจะเห็นว่ามันมองเห็นโฟลเดอร์ของคุณจริง และคุยกับมันได้เป็นภาษาคนล้วน ๆ พอชินกับจังหวะนี้แล้ว ค่อยขยับไปสั่งให้มันสร้างหรือแก้ไฟล์จริง พอถึงตรงนี้ คุณก็เปิดใช้ Claude Code เป็นแล้วจริง ๆ

## เมื่อมันไม่ยอมติดตั้ง ปัญหาที่เจอบ่อย

ไม่ใช่ทุกคนจะติดตั้งผ่านในครั้งแรก และนั่นปกติมาก นี่คืออาการที่เจอบ่อยที่สุด พร้อมว่าต้นตอคืออะไร ส่วนวิธีแก้ทีละ step เก็บไว้รวมที่เดียวใน [ภาคผนวก B การแก้ปัญหา](/in-practice/troubleshooting) เพื่อไม่ให้บทนี้ยาวเกินไป

อาการแรก พิมพ์ `claude` แล้วเครื่องฟ้อง `command not found` แปลว่าติดตั้งยังไม่ครบ หรือเครื่องยังหาที่อยู่ของโปรแกรมไม่เจอ บางทีแค่ปิดเปิด terminal ใหม่ก็หาเจอ

อาการที่สอง บน Windows ฟ้องว่า `'irm' is not recognized` หรือ `The token '&&' is not a valid statement separator` นี่คืออาการพิมพ์คำสั่งผิดหน้าต่างที่พูดไปแล้ว สลับระหว่าง PowerShell กับ CMD ให้ตรงกับคำสั่ง

อาการที่สาม login แล้ว browser ไม่เด้ง หรือวนกลับมาไม่ได้ ใช้วิธีก๊อปลิงก์หรือก๊อปรหัสที่อธิบายในหัวข้อ login ข้างบน

อาการที่สี่ login ด้วย subscription แล้วมันใช้ของผิดบัญชี หรือฟ้องเรื่องสิทธิ์ มักเป็นเพราะมี API key เก่าค้างอยู่ในเครื่องและถูกหยิบไปใช้ก่อน subscription

เคล็ดลับเดียวที่ควรจำคือ ถ้าอะไรแปลก สั่ง `claude doctor` ก่อนเสมอ มันจะบอกคุณเองว่าปัญหาอยู่ตรงไหน แล้วค่อยเปิด [ภาคผนวก B](/in-practice/troubleshooting) ตามอาการ

## สรุป จุดที่ยากที่สุดอยู่ข้างหลังคุณแล้ว

ทั้งบทนี้มีใจความเดียว สิ่งที่ทำให้คนไม่กล้าเริ่มไม่ใช่ความยากของการใช้งาน แต่คือการเปิดใช้ครั้งแรก ติดตั้งและ login ครั้งเดียว แล้วที่เหลือคือการพิมพ์บอกสิ่งที่อยากได้เป็นภาษาคน ซึ่งคุณทำเป็นอยู่แล้ว ก้าวเดียวนั้นคุณเพิ่งข้ามมา

ถ้าวันไหนมีอะไรสะดุด สั่ง `claude doctor` ก่อนเสมอ มันจะบอกเองว่าปัญหาอยู่ตรงไหน

แต่พอเริ่มใช้จริง คุณจะเจอด่านถัดไปที่ไม่เกี่ยวกับการติดตั้งเลย คือเอกสารทางการของมันที่เต็มไปด้วยศัพท์อย่าง agent, MCP, hooks อ่านแล้วเหมือนจมน้ำ บทต่อไปจะพาคุณอ่านเอกสารพวกนั้นโดยไม่จมน้ำ จับศัพท์แต่ละตัวโยงกลับไปหาสิ่งที่คุณเข้าใจแล้วในบทก่อน ๆ
